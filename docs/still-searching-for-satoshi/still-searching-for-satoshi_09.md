# 4. 比特币

现在我们可以开始梳理比特币的运作方式了。前几章内容让我们更好地理解了多种多样的技术以及虚拟货币的试验，正是这些试验使得比特币得以在这个特定时间点出现。在深入探究使其成为可能的技术细节之前，我们先简要勾勒一下比特币的一些关键组成部分。

简单来说，比特币是一个提供名为 `bitcoin`（比特币）的虚拟货币的系统，用户无需通过银行、公司或其他组织等任何中央中介机构即可进行交易。比特币可以精确到小数点后八位（相比之下，普通货币通常只能精确到小数点后两位）。其最小单位称为 `satoshi`（聪），类似于美分与美元的关系。

要开始使用 `bitcoin`，用户需要拥有一个数字钱包。这个钱包包含用户可以用来与其他用户交换 `bitcoin` 的公开标识符。它的功能类似于我们熟知的账户，用于持有 `bitcoin`，即你可以从中支取总额的 `bitcoin`。只有两种方式可以将 `bitcoin` 存入钱包：一种是通过称为“挖矿”的过程创建 `bitcoin`（我们稍后会讲到），另一种更常见的方式是让别人将 `bitcoin` 转账到你的钱包。后一种方式可以是作为商品或服务的付款，也可以通过购买来实现。

不过，仍然只有少数人将 `bitcoin` 作为支付手段。当你购买 `bitcoin` 时，你实际上是将美元等法定货币转移给某个中间人，该中间人钱包里有 `bitcoin`，再将其转入你的钱包。这可以通过某个比特币交易所或任何钱包里有余额的人来完成。另一种方式是通过其他加密货币购买 `bitcoin`，这也很常见，因为 `bitcoin` 在众多不同的加密货币之间起到了桥梁作用。这些交易平台独立于比特币系统之外。比特币系统内部不存在美元或任何其他法定货币。

到目前为止，我们对于常规货币和银行的直观理解仍然适用：我们有一种用单位计算的货币，这些单位汇总成钱包里的余额，并且在钱包之间可以进行转账。但在这里，我们开始步入一个相对于传统货币体系而言显得非常奇特的道路。所有曾经发生过的交易都记录在一个公共账本上。这就是所谓的 `blockchain`（区块链）。任何钱包的余额都是通过查看与该钱包相关的所有转入和转出交易（借记和贷记）动态计算得出的。因此，任何地方都没有记录余额；它是基于流入和流出之间的差额计算出来的总额。比特币网络允许用户在任意时刻花费这个总额。它之所以被称为 `blockchain`，是因为大约每十分钟，网络上待处理的交易就会被打包成一个区块，该区块会获得一个编号和一个哈希值。这个哈希值会被用于下一个区块，从而形成一条链。这种方法确保了从第一个区块（称为创世块）开始，整条链上交易记录的完整性。我们稍后会详细讨论细节。

现在，这个系统需要有人来运行它。银行有中央服务器负责记录和清算所有交易，而比特币则采用一个节点网络。这是一个由众多计算机组成的网络，它们都在同一个公共交易记录上工作。该网络会收集用户想要进行的所有交易，并将其记录在一个区块中。这些运行比特币网络的计算机被称为矿工。他们会因验证交易的工作而获得 `bitcoin` 奖励。原则上，任何计算机都可以通过下载开源的比特币软件来开始挖掘 `bitcoin`。但实际上，普通计算机很难产出任何 `bitcoin`，因为挖矿任务已经变得高度专业化，需要定制的芯片才有可能获得奖励。请记住，对于所有其他未能生成区块的计算机，是没有任何奖励的。

矿工们通过尝试解决一个加密谜题来竞争创建下一个交易区块。这是一种工作量证明，需要消耗大量的计算资源。网络上第一个解决谜题并成功创建交易区块的节点，将获得新铸造的 `bitcoin` 作为奖励。每个区块产生固定数量的 `bitcoin` 作为奖励。

以上是对比特币及其大致运作方式的简要概述，旨在提供一个总体概念。现在，让我们深入探讨细节。



## 虚拟货币

比特币汲取了此前虚拟货币探索的成果，直接构建于注重隐私和稳定货币供应的加密虚拟货币体系之上。比特币白皮书中仅引用了两种先前存在的加密货币：亚当·巴克的哈希现金和戴伟的 B-money。比特币从哈希现金继承了采用工作量证明来铸造货币的理念。比特币的许多核心要素已在哈希现金中定义完毕，以至于巴克后来声称自己发明了比特币挖矿函数——这可以说是其核心组成部分。通过比特币挖矿，货币被生产并投入流通，如同哈希现金代币一样，随后即可用于交换。

许多虚拟货币支持者关注的核心问题之一是银行（特别是中央银行）的颠覆性作用，因为在虚拟货币信仰者看来，银行一直在“印钞”。货币供应量增加会产生通胀效应，这本质上是“窃取”货币持有者的财富，因为通胀会降低名义货币价值。而比特币的设计则截然相反：通过逐步减少货币发行量实现结构性的货币供应紧缩，旨在形成通缩机制，使货币持续增值。

比特币的供应遵循既定时间表，随着时间的推移，产生的货币数量会越来越少，而非越来越多。首先，理论上可生成的比特币总量在设计上就被固定了。与人们日常生活中使用的美元、欧元等法定货币不同（其货币供应量不受限制），比特币是有限的：总共只能挖出 2100 万枚比特币。当 50%的比特币被挖出后，矿工奖励首次减半，这发生在 2012 年。此后奖励变为 25 枚比特币。第二次减半发生在 75%的比特币被挖出之后，奖励降至 12.5 枚比特币。这一过程将持续，直至再无比特币可投入流通。根据计算，最后一枚比特币将在 2140 年左右被挖出。

将稀缺性嵌入货币的设计可能受到基于贵金属的虚拟货币启发，因为贵金属的价值正源于其稀缺性。在历史上最常用于货币的三种贵金属（金、银、铜）中，黄金最为稀缺，白银次之，铜的稀缺度最低。相反，像铁这样的金属因过于普遍，作为价值储存手段的价值不大。在这方面，黄金尤为值得关注：一百多年来，全球市场每年的黄金供应量始终维持在黄金总存量的 2%左右。这种状况形成了较高的存量-流量比（目前约为 60），即现存黄金总量与新增开采黄金量的比率。正是这种动态使得黄金能更好地保值。白银的存量-流量比约为 20，相对更为充裕，这形成了类似于通胀性货币供应的不同动态。当总存量中新增量过多时，每个单位的价值就会缩水。

虽然 e-gold、eBullion 等项目交易的是与实物黄金挂钩的电子代币，但尼克·萨博提出了一种纯数字版本，旨在创建数字价值储存手段。比特币最终接近了萨博在其关于“比特黄金”的构想中提出的理念。他设想通过解决密码学难题来铸造“比特黄金”，从而创造类似黄金的稀缺性。解决密码学难题的过程类似于黄金开采。比特币是首个实现此类系统的项目。

## 加密以保护隐私

密码朋克关注的一个核心问题是隐私。他们特别针对银行和政府过度获取客户交易信息这一问题，因为从这些信息中可以收集到大量涉及个人隐私的数据。相比之下，比特币默认提供匿名账户，且没有银行作为交易中介。任何人都无法直接通过用户的交易信息识别其身份。

在讨论比特币时，“钱包”是一个核心概念，但它更多是一种思维概念，而非真正的技术产物。如今，钱包通常等同于钱包软件，其形式多样。钱包可以由在线交易所托管，也可以由用户下载并在本地作为客户端应用程序运行。从某种意义上说，“钱包”这个术语具有误导性，因为钱包本身并不存储任何比特币。要与比特币网络交互进行交易，需要一些中介服务。钱包软件提供这些服务，例如生成比特币地址、存储私钥以及显示每个比特币地址上的余额。之所以需要这些功能，是因为比特币本质上不过是一份巨大的交易清单。

要使用比特币进行任何操作，首先需要一个地址——这是一串用于向他人标识你的身份的字符串，允许你发送和接收比特币。你可以将其视为银行账户；只不过在比特币系统中，你无需请求任何人即可创建地址。

地址基于我们在第 2 章中见过的密码学技术构建。生成过程如下：首先，选择一个大的随机数，这就是你的私钥。由于只需私钥即可获取你的比特币，因此必须格外小心地保护它。基于这个私钥，通过一种称为椭圆曲线乘法的技术生成公钥。这是可用于加密和签名的标准公钥-私钥对。但目前我们还未进行任何加密或签名操作。接下来，将公钥通过单向加密哈希函数处理。首先，从公钥计算`SHA-256`哈希值，再基于该值计算`RIPEMD160`哈希，得到 160 位的数字。然后将此数字编码为`Base58Check`，从而生成一个 58 字符的字符串。最终结果即为地址。地址可以显示为字符串，有时也以二维码形式呈现以便识别。如果你想让别人用比特币付款给你，只需将你的地址发送给他们，并请求其将资金转入该地址即可。

当你以新地址加入比特币网络时，在获得与地址关联的比特币之前，你几乎无法进行任何操作。这可以通过两种方式实现：要么开始挖矿获取比特币，要么请他人向你转账比特币。对于个人而言，通过挖矿获取比特币已不再可行（后续会详细说明）。因此，必须从其他途径获取比特币，例如通过交易所或出售商品。

在线交易所允许你存入美元等法定货币以兑换比特币。交易所随后会将比特币从其某个地址转入你的地址。这笔交易会被比特币网络捕获并记录在一个区块中。完成后，你的余额中就会有相当于刚刚转入的金额可供使用。

如果你想购买某物或向他人转账，只需获取对方的地址，输入转账金额，还可以添加一些文本信息。但请注意，该文本信息在区块链上对所有人可见。交易将包含以下内容：

*   用于提取比特币的地址
*   转账金额
*   你可能希望添加的额外交易费用（以加快交易确认速度）
*   接收比特币的地址
*   附加文本信息



仅仅发送一笔交易是不够的，因为任何人都可以用你的钱包地址来发送交易。为了验证交易，比特币网络需要使用与该地址关联的**私钥**对交易进行**签名**。之后，交易会以“待处理”状态发布到比特币网络。矿工将验证该交易是否已用私钥签名。一旦交易被包含在一个区块中，交易即告完成，且可供所有人查看。

将此与常规银行账户进行比较：银行集中保存所有数据，并持有允许你从账户转账的凭证。银行可以关闭对你账户的访问，或将你的凭证泄露给恶意第三方，而该第三方随后将能够动用你的资金。银行通过其“了解你的客户”（KYC）流程确切地知道你是谁，并且有义务遵守法院裁决，披露你的交易或冻结资金。

使用比特币时，没有人知道你是谁。只有一个个无意义的随机比特币地址。尽管与银行相反，所有交易都是公开的，但没有人能将它们与你这个私人身份联系起来。资金并非集中存储在任何地方，而是无处不在，这意味着即使某个服务器崩溃或在线交易所破产，资金仍然存在并可访问。最后，没有人能控制你资金的访问权限；如果你拥有某个地址的私钥，你就可以访问该地址关联的所有比特币。更重要的是，交易清算在一小时内完成，而传统银行转账则需要几天时间。

另一方面，比特币缺乏传统银行业务所附带的护栏和安全网。如果犯罪分子盗用了你的信用卡，你可以向信用卡公司提出争议，这通常会导致资金成功退回你的账户。而对于比特币，如果有人获取了你的私钥并转移了所有资金，那么任何人都无能为力。你无法向比特币网络申诉。同样，如果你不小心输入了错误的地址，资金将归属于该地址，并且你绝无可能找到那个人并请求他归还你的钱。资金就这样消失了。

## 共享公共账本

传统银行对其所有交易记录保密。事实上，这是银行保密的基本原则，并在许多国家被编入法律。当资金被使用时，银行通过一个复杂的银行间清算系统确保一个账户只能消费一次。确保资金仅被使用一次是所有交易媒介的必要条件，这也是清算过程如此漫长的部分原因。

哈希现金（Hashcash）尚未解决双重支付问题，它依赖一个中央注册表来跟踪代币是否已被使用。公共账本是比特币解决此问题的关键创新。交易发送方和接收方的完全隐私，使得比特币能够彻底颠覆传统的银行保密原则，将所有交易公之于众。当所有交易都在假名钱包 ID 之间进行时，即使公开张贴，隐私仍能得到保障。

公共账本是一份记录所有曾经发生的比特币交易的长期记录。这种共享公共账本的概念由戴伟（Wei Dai）在其 B-money 的愿景中提出。戴伟提出了分布式账本的概念，所有交易都被记录以供所有人查看。基于这个分布式账本，可以在交易发生时计算账户余额，这本质上就是格里格（Grigg）的三式记账法的概念。这些想法在比特币中首次得以实现。

公共账本并非单独交易简单拼接而成的一长串记录。它由交易区块组成，因此得名区块链。一个区块是交易的集合。由于区块是历史的记录，它必须是不可变的。对内容进行哈希处理即可实现这一点。区块通过其包含的所有交易的哈希值进行密封，因此任何人都可以验证后续副本中所包含的交易未被篡改。仅仅在每个区块中放入一个哈希值是不够的，因为正如我们所见，任何地方都没有记录账户余额。因此，所有余额都需要所有涉及任何给定比特币地址的先前交易的完整记录，以验证该地址是否有余额可消费。如果缺少一个或多个区块，双重支付就可能发生，因为缺失区块中的交易将不计入账户余额的计算。

这意味着验证交易的矿工必须能够访问整个公共账本的交易历史。这带来了确保所有区块都存在且已入账的问题。如果你支付了 100 个比特币的那个区块正好从公共账本中消失了，那么这笔钱在你的账户中仍会显示为可用余额，从而允许进行双重支付。解决方案是每个区块必须包含前一个区块的哈希值，正是这一点使其形成了一条链。通过这种方式，从公共账本中移除任何区块都是不可能的。前一个区块的哈希值和交易无法产生与后续区块相同的哈希值。请记住，哈希是基于所有输入的指纹，因此生成下一个区块的指纹需要前一个区块的指纹。尼克·萨博（Nick Szabo）最初在其关于比特黄金（Bitgold）的想法中提出了这一机制，但从未实现。

## 点对点网络

比特币网络是一个运行比特币软件并通过互联网直接连接的计算机系统。交易被发布到该网络以纳入下一个区块。当新区块被创建时，它同样会被发送到其他节点进行验证并纳入区块链。

如果这个公共账本保存在中央服务器上，它将容易受到破坏和故障的影响。这就是系统采用点对点网络进行分布式部署的原因，网络中的每个节点都包含整个公共账本的副本。当有人想要开始挖矿时，必须连接到这个网络并下载区块链。因此，该网络也作为一个具有多个冗余副本的分布式存储系统工作。通过这种方式，即使一个节点发生故障，也会有成千上万个其他节点持有副本。

另一种情况是，创建了一个替代区块，其中插入了替代交易。由于大多数其他节点将拥有该区块的另一个版本，因此该替代区块会被网络拒绝。在比特币网络中，存在一个内置的共识系统，要使区块被接受，需要获得多数同意。网络建立在最长链之上。如果替代区块未被接受，它将不会成为最长链的一部分，因此其中的交易也不会被“结算”。

鉴于这种点对点模型，即使监管机构和权威人士宣称应该关闭比特币，任何中央机构也无法做到。只要有计算机连接并运行比特币软件，比特币就会存在。这使得比特币和其他类似的区块链成为有史以来最具韧性的交易系统。银行可以被关闭，甚至中央银行也可能随着一个国家的消失而消失，但只要有人投入算力和网络连接来运行软件，比特币和区块链就会继续存在。



## 工作量证明  

接下来的一个自然问题是，这些连接到网络的运行比特币软件的计算机究竟在做什么？答案是，它们在进行比特币挖矿。这一过程实际上与 Hashcash 的工作量证明功能几乎相同，但它是如何具体运作的呢？  

工作量证明的原则是，执行某个功能不应该是免费的。在 Hashcash 中，这个功能是发送电子邮件或向网络服务器发送请求。而对于比特币，这个功能则是将交易打包进区块并“印钞”，即生成比特币。这样的功能需要付出一定代价，这是非常合理的；否则，每个人都会随意创建区块来获取比特币。  

问题在于，你如何确切地证明你为创建区块付出了努力。将几笔待处理交易捆绑在一起并进行哈希运算，本身确实需要一些工作量，但这点工作量微不足道，本身并不昂贵。为了制造成本，这项工作被附加了一个加密难题。  

回顾一下，哈希本质上是一个固定长度的数字。比特币使用 SHA-256 算法为区块中的所有交易生成一个哈希值。这个哈希值是一个 256 位的字符串。它通常用十六进制值来表示。十六进制计数系统是一种基数为 16 的编码系统。基数指的是该数字系统中有多少个不同的数字。二进制系统是基数为 2，即只有 0 和 1 两个数字。十进制系统是基数为 10，意味着有 0 到 9 共十个不同的数字。因此，十六进制系统有 16 个不同的数字。由于我们在十进制系统中没有超过 10 的数字符号，因此额外的数字用字符 A 到 F 表示。由于任何十六进制数字都可以用 4 个比特来表示，所以得到的十六进制字符串长度为 64 个字符（256/4），包含十进制数字和字母 A 到 F。  

了解了这些之后，让我们回到这个难题。如果对最终的哈希字符串没有任何要求，生成一个哈希值是轻而易举的，任何哈希值都可以。但如果我们增加一个要求，即生成的哈希值必须以一个 0 开头，我们就无法轻易地直接生成它了。由于我们知道重复运行哈希算法不会有任何不同，我们必须调整输入。当然，我们可以尝试添加或删除一些随机交易，但这很不方便。因此，比特币系统允许矿工向区块添加一个称为“随机数”（nonce）的数字。这个名称指的是“一次性的数字”，它是一个 32 位的字段。由于哈希值对输入的任何变化都非常敏感，随机数中的任何数字都会导致最终的哈希值发生巨大变化。因此，这个难题就在于找出应该在随机数中添加什么值，使得对区块中的交易和其他信息进行哈希后得到的结果，低于设定的目标值。  

显而易见，以 0 开头的哈希值的数量远小于所有可能的哈希值总数。如果我们再增加一个 0，可接受的哈希值总数就会进一步减少。实际上，每增加一个 0，整个解空间就会缩小。这意味着，生成正确哈希值的概率会随着目标值开头每增加一个 0 而降低。其结果是，平均而言，需要尝试更多次才能找到一个满足目标的哈希值。这就是挖矿的本质：一场加密彩票。而难度与矿工必须为最终哈希值找到的前导零的数量成正比。  

现在可能会有人问，这如何体现工作量证明呢？答案是，对哈希值的每一次猜测都需要 CPU 付出工作量。实际上，不一定是 CPU；你也可以手动计算哈希值，有人这样做过，需要花费数小时。CPU 越快，它在单位时间内能尝试的猜测次数就越多。由于只有第一个找到解决方案的矿工才能获得奖励，时间至关重要。CPU 越快，你拥有的加密彩票就越多。难度越高，猜测出正确解所需的总体 CPU 工作量就越大。  

由于摩尔定律确保了计算能力持续增长，中本聪撰写比特币白皮书时认为困难的计算，在今天可能微不足道。换句话说，在 2009 年解决特定难度所需的 CPU 工作量，今天可能要便宜得多。这可能会削弱比特币的工作量证明机制，因此比特币协议会持续设定新的难度目标。每产生 2016 个区块，难度就会重置一次。  

为了避免区块生成过快，从而一次性释放所有比特币，中本聪将创建一个区块的目标时间设定为平均十分钟。如果平均出块时间少于十分钟，难度就会增加；如果平均时间多于十分钟，难度就会降低。生成正确解所需的时间取决于比特币网络可用的总 CPU 周期数，或者简单来说：比特币网络的总计算能力。这种计算能力体现在“算力”（hash rate）这个概念上，因为其最终目的是生成哈希值。目前使用的单位是每秒太赫兹（TH/s），这意味着整个网络每秒能产生多少万亿次哈希。用我们的彩票术语来说，这相当于所有彩票参与者每秒能购买多少张彩票。在撰写本文时，这个值约为每秒 2.6 亿太赫兹。  

当矿工最终成功构建了一个区块时，他们需要获得报酬。这笔报酬是在新区块中创建的新铸造的比特币。这个数字被记录在“币基”（coinbase，不要与 Coinbase 公司混淆）中。区块中的第一笔交易被称为币基交易，由创建该区块的矿工生成。它包含了矿工的地址，以及一个不超过比特币协议定义的当前区块奖励的数字。回顾一下，比特币是根据已铸造数量的既定时间表来分发的。目前，每个区块的奖励是 6.25 个比特币。  

在这个奖励之上，还要加上用户支付的交易手续费。平均交易手续费大约相当于 1 美元，但曾在 1 美元到 5 美元之间波动。目前，交易手续费仍然是矿工总奖励中的一小部分，大约在个位数百分比附近徘徊，但有时也曾占到奖励的 50%以上。随着比特币发行量的减少，预计来自交易手续费的奖励比例将会增加。在最后一个比特币被挖出后，这将是激励矿工继续验证交易和创建区块的唯一来源。



## 什么是比特币？

简而言之，比特币是一个提供虚拟货币（bitcoin）使用途径的系统。任何人都可以通过下载软件或使用在线服务创建钱包来开始使用它。这个钱包本质上是一组公共身份标识的集合，只有使用创建钱包时所用的私钥才能进行交易。进行交易时，需使用钱包软件指定想要发送到的钱包 ID 以及比特币数量。该系统由一个自愿提供计算资源的计算机网络支撑，使得交易能够被记录在区块链上，该区块链包含了所有过往的交易记录。这些计算机被称为矿工，他们通过交易手续费和新生成的比特币获得奖励。

从这个概述可以看出，比特币是一个复杂的系统，融合了来自众多不同来源的技术创新。事实上，很难指出比特币中有哪个特定特征是完全全新的发明。与此同时，比特币的整体解决方案又是如此彻底地新颖和富有创新性，以至于在 21 世纪几乎找不到能与之匹敌的事物。某些方面在孤立看来似乎过于复杂，甚至有些刻意，但这自有其道理；当所有要素整合在一起时，便产生了一个独特且功能完备的解决方案，它已经证明自己经得起十多年的考验。

比特币和区块链技术是技术发展轨迹的产物，这些轨迹有时可追溯到数千年前，但直到现在才达到足以构建一个全球性、具有韧性的交易系统的水平。这些发展轨迹常常以令人惊讶且矛盾的方式交汇。回想一下，会计、银行和密码学主要关注信息的隐私和保密，而比特币则完全不同，它是完全公开的，任何人都可以访问。另一个例子是，虚拟货币，就像以往所有的货币一样，都依赖于一个中央铸币机构。历史上，货币发行者面临的最大挑战是防止任何其他方铸造货币，这种行为被称为伪造，通常伴随着严厉的惩罚。而比特币再次颠覆了这一传统，它允许任何人铸造货币。

比特币在诞生之初几乎只引起了人们耸肩和不多的几句批评评论。自问世以来，它被宣告死亡的次数已不止几次。尽管如此，它作为世界上最有价值的虚拟货币，至今仍在蓬勃发展。但这一切是如何融合在一起的呢？这现在正是我们关注的焦点。
