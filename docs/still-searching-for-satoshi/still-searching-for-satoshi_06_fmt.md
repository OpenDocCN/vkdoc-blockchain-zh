# 工作量证明

不幸的是，隐私并非电子邮件引入数字通信的唯一新问题。网络世界的另一个问题恰恰源于其初衷：让交流变得即时、便捷，并且可以免费地触达任何距离、任何数量的人。绕过传统的沟通壁垒——例如印刷和分发书籍报纸，以及写信和寄信——对大多数人来说是一种解放，但其负面影响是，它催生了这些壁垒原本能遏制的新现象：诈骗。

我小时候记得，有时你会收到一张通知，说你中了某个德国彩票，只需支付彩票费或类似费用即可。当然，这并非真实，而是一种骗取人们钱财的诈骗。不过，这类事相对少见，因为邮寄信件并非免费。诈骗的经济账必须确保有足够比例的人会上当。如果一封信的邮费是`$0.50`，而诈骗能获利`$100`，那么只能寄出`200`封信。这意味着转化率必须达到`1/200`。

随着电子邮件的出现，这种经济账发生了变化。如果一封信的成本是`$0.0001`（预留一些资金用于支付网络连接和域名注册），那么转化率现在只需达到`1/1,000,000`。不仅诈骗可以经济地传播；任何形式的广告或政治宣传都能以这个价格点进行传播，相比报纸或电视广告的费率，这极具竞争力。这催生了垃圾邮件现象，至今仍在困扰着我们。

这其中的根本原因并非人们突然变得更容易受广告和欺诈的影响。而是简单的事实：通信成本被降低得如此之多，以至于即使极低的成功率也能盈利。由于对通信征税有悖于互联网的基本理念，因此必须开始以不同的方式思考这个问题。如果能在发送方身上施加成本，而接收方不用——就像旧的通信模式那样——那么就有可能改变这种动态，通过提高成本来限制垃圾邮件。

## 打击垃圾邮件

加密领域再次出手相助。1993 年，辛西娅·德沃克和莫尼·纳奥尔提出了一种想法：施加他们所谓的“定价函数”来打击垃圾邮件。他们认识到，电子通信的成本不会构成威慑，而实施任何法律限制也不可行。相反，他们寻找了另一种传输成本：“主要思路是让邮件系统要求发送方计算消息及一些附加信息的某种适度昂贵、但并非棘手的函数。”这里的成本将由计算的价格（即硬件和电力成本）施加。这一想法后来被称为工作量证明。

这个问题看起来与加密中的单向函数类似。我们看到，有了密钥，解密很容易，但没有密钥则几乎不可能。我们仍然希望存在这种不对称性：一方（接收方）容易，另一方（发送方）困难，但我们不希望让它变得过于困难。因此，目标就是找到一种通过参数来调节问题难度的方法。

为了使函数可预测且不依赖于电子邮件的长度，他们建议该函数应基于哈希函数——正如我们所知，它总是输出相同大小的字符串。基于此输入，定价函数应该能够通过某个参数来调节计算结果所需的难度。结果应该易于验证。他们提出了三族函数，但指出需要进一步研究，因为直到那时，密码学对“适度困难”的函数关注甚少。所有的注意力都集中在开发几乎牢不可破的函数上。

## 应用工作量证明

直到 1999 年，这一想法才由马库斯·雅各布森和阿里·朱尔斯在一篇文章中正式提出，并命名为“工作量证明”。他们指出了工作量证明可以使用的几种不同方式。其中之一是作为控制延迟的方法。这种用法旨在控制解决密码学问题所需的时间。这种用途早在 1996 年就由里维斯特、沙米尔（`RSA` 论文的作者）和瓦格纳开发出来，他们称之为“时间锁谜题”。

里维斯特和沙米尔还开发了一种工作量证明的应用，他们称之为`MicroMint`。实际上，这是一个为互联网小额支付设计的密码系统。他们这样描述该方案：“`MicroMint`‘硬币’由经纪人生产，经纪人将其出售给用户。用户将这些硬币作为支付手段交给供应商。供应商再将硬币返还给经纪人，以换取其他方式的付款。一枚硬币是一个比特串，任何人都可以轻松验证其有效性，但生产起来却很困难。”

作为实现此目的的一种方法，他们建议使用哈希碰撞。回忆一下，哈希碰撞是指哈希函数的两个不同输入产生相同输出（即相同哈希值）的情况。在这篇文章中，他们展示了可以控制这种碰撞的难度。

在这个方案中，经纪人生产`MicroMint`“硬币”，客户购买这些硬币，然后从供应商那里购买物品，供应商再向经纪人兑现价值。这个方案中的问题正是密码学界在讨论数字货币时经常提到的一个问题，即双重支付问题，因为供应商如何知道这枚“硬币”没有被其他供应商使用并兑现过呢？

## 数字邮资

亚当·巴克接过了这个挑战。他早在 1997 年就开始研究`Hashcash`系统。`Hashcash`解决方案有一个计算令牌的函数。他的方案允许我们动态调整生产`Hashcash`令牌的难度。这些令牌类似于里维斯特和沙米尔提出的`MicroMint`硬币。不幸的是，与他们一样，除了在一个中心化数据库中记录之外，他也没有解决双重支付问题。通过`Hashcash`，巴克试图提供一种机制，使得能够对电子邮件等未计量互联网资源进行节流。尽管他暗示这可以用于数字货币，但并未明确地朝那个方向推广。双重支付问题的解决方案以及这一应用，要等到十年后比特币的提出才得以实现。

## 为免费信息增加成本

工作量证明是大规模通信成本下降的产物。它引入了一种使用密码学函数来提供谜题的方法，这些谜题需要可预测的 `CPU` 开销。这进而转化为了成本。现在可以为之前免费的功能附加一个可变成本。在电子邮件中应用此方法的问题在于，没有任何系统得到广泛采用。相反，业界专注于垃圾邮件检测和黑名单机制来消除垃圾邮件。如果不是后来在`比特币`及随后的`区块链`中应用了工作量证明，它可能早已被限制在密码学已经十分广泛的“稀有品”之列了。
