# 书籍总结

总体而言，本书讲述的是利用`IoT`（物联网）进行**业务驱动的数字化转型**。业务驱动的数字化转型意味着业务目标决定技术路线图，而非反之。物联网（`IoT`）正是这样一种能够助力企业实现数字化转型的技术。

物联网（`IoT`）的核心在于通过网络连接设备和工厂设备，从中获取洞察，帮助企业提升市场竞争力，或以高度优化且经济高效的方式开展运营。

在本书的第一部分，我们向读者介绍了关于利用`IoT`进行数字化转型的通用行业视角。我们还讨论了企业为适应并保持市场相关性的三种业务战略：**客户参与战略**、**业务转型战略**和**业务生产力提升战略**。企业有必要选择一项业务战略，以确保基于该战略确立清晰的使命与愿景，从而定义`IoT`路线图。随后，我们将围绕所选业务战略研究所有相关业务流程，以确定可以应用`IoT`实现该战略的用例。

在第二部分，我们讨论了`IoT 标准参考模型`。这是一个抽象框架，由一组相互关联、定义清晰的组件构成，企业可借此成功实施`IoT`解决方案。`IoT 标准参考模型`可应用于任何行业的`IoT`用例，并始终保持抽象，以便基于该模型实现多种可能、各不相同的`IoT`架构。

在这一部分，我们还讨论了企业在实施`IoT`用例时面临的挑战，以及如何利用`IoT 标准参考模型`克服这些挑战。这些挑战涵盖了从设备多样性及其通信协议，到将`IoT`设备连接在一起的`IoT 网关`，再到企业用于存储大数据和执行分析的`IoT 云平台`等各个方面。

本书的第三部分讨论了人工智能和`区块链`对于`IoT`用例的重要性。

`区块链`是技术圈内引人关注的话题之一，它正在从根本上改变企业未来的商业模式。我们讨论了将私有`区块链`用于`IoT`用例的重要性，以及五种`IoT-区块链`实施模式。企业可借助这些模式实现`IoT`设备、`智能 IoT 网关`、`IoT`平台与`区块链`之间的无缝通信。这些模式有助于实现信任、互操作性和可扩展性，而这正是当前企业在`IoT`实施中面临的主要挑战之一。

人工智能（`AI`）是接下来的话题，它是一门通过教导机器模仿人类行为使其变得智能的艺术。人工智能是`IoT 标准参考模型`中非常重要的特性之一。该参考模型建议应用`AI`模式从数据中生成洞察并自动采取相应行动。书中还提供了在`IoT`背景下如何以及何时应用`AI`的视角。

在本书的最后一部分，我们讨论了围绕大数据与分析、产品组织以及`IoT`产品团队的`IoT`执行层面问题。

随着`IoT`引领数字化转型，管理数据的复杂性已呈指数级增长——因为过去我们讨论的是 TB（太字节）和 PB（拍字节）级的数据，而面对`IoT`，我们将处理 ZB（泽字节）级的数据。在这一部分，我们讨论了如何利用`数据优先参考模型`搭建高效的数据平台，使企业能够减少在管理数据平台上的资金和精力投入，并将更多注意力放在实际的洞察与行动上。我们还讨论了企业需要从数据角度验证的关键能力，以便为其`IoT`用例选择合适的产品。

随后，我们还讨论了实施`IoT`用例所需的产品组织。企业必须从基于项目的思维转向产品型组织，才能从`IoT`用例中获益。然而，这是一项非常复杂的转型，涉及企业人力资源职能的重大转变。在这一部分，我们讨论了企业如何转型为产品型组织，以及企业内部人力资源职能需要如何改变。最后，我们还讨论了负责确保`IoT`用例成功实施的`IoT`产品团队。我们介绍了`IoT`实施所需的不同角色，以及`IT`和`OT`团队如何协同合作，共同推动`IoT`用例的成功。

总而言之，本书为企业定义了一个名为`IoT 标准`的数字化转型框架，该框架将帮助企业更好地开展业务，并利用`IoT`实现运营效益。
