# 第 1 章 区块链简介

我们有两个函数，它们应具有与对称密钥算法相同的属性。

- `E(x, p)`，使用公钥 `p` 加密消息 `x`
- `D(x', s)`，使用私钥（秘密密钥）`s` 解密加密消息 `x'`

在我们的示例中，我们将依赖于取模运算。请回忆：`a mod b` 表示 `a` 除以 `b` 后的余数。例如，`4 mod 2 = 0`，因为 4 除以 2 没有余数；而 `5 mod 2 = 1`。

下面是一个基于加法和取模运算的基本加密算法示例：

1. 选择一个随机数，例如 100。这将代表一个公共的、公开可用的密钥。
2. 在范围 (1, 100) 内选择另一个随机数，例如 97。这将代表私钥 `s`。
3. 公钥 `p` 通过从私有密钥中减去公共密钥得到：`100 - 97 = 3`。
4. 要加密数据，将其与公钥相加并对 100 取模：`E(x, p) = (x + p) mod 100`。
5. 要解密数据，我们使用相同的逻辑，但使用我们的私钥，因此 `D(x', s) = (x' + s) mod 100`。

例如，假设我们要加密 5。那么 `E(5, 3) = (5 + 3) mod 100 = 8`。要解密 8，我们有 `D(8, 97) = (8 + 97) mod 100 = 105 mod 100 = 5`。

这个示例使用了一个非常简单的生成对 `(x + y) mod c`。但在实践中，密钥对生成算法要复杂得多，且更难以被攻击者破解。毕竟，算法计算的复杂性才是使其难以被破解的原因。

我们可以使用类似的算法进行数字签名：

- `S(x, s)`，使用私钥 `s` 签署消息 `x`（加密）
- `V(x', sig, p)`，使用签名 `sig` 和公钥 `p` 验证已签名的消息 `x'`（解密）

正如我们之前所说，每条记录还将包含一个特殊数字（或哈希值）。这个哈希值将由 `S(x, s)`（加密）产生。通过使用验证函数来确认记录的归属（解密），可以对哈希值进行验证。

钱包将包含一对公私钥。这些密钥将用于接收或发送货币。使用私钥，可以在区块链上写入新区块（或交易），从而实际花费货币。使用公钥，其他人可以向其发送货币并验证签名。

## 1.3 哈希

**定义 1-8** 哈希是一个单向函数，它对文本进行编码，并且无法找回原始值。

哈希比之前描述的加密方案更简单。哈希函数的一个示例是返回字符长度，如 `H("abc") = 3`，但 `H("bcd") = 3`。这意味着我们无法仅通过返回值 3 来恢复原始值。

正如我们之前提到的，使用这种技术的原因是因为它有一些有趣的特性，例如为我们提供工作量证明。

**定义 1-9** *挖矿*是验证交易的过程。为此付出的努力，成功的矿工将获得金钱作为奖励。

**Hashcash** 是一种工作量证明系统[³]。我们将使用它来实现挖矿。我们将在后续实现该算法的章节中详细看到这个算法是如何工作的。

哈希函数还有一个有用的特性，即通过在每个区块中包含当前区块哈希（当前哈希）和前一区块哈希（先前哈希）的信息，可以将两个或多个不同的区块连接起来。例如，区块 1 可能有一个哈希值 `123456`，而区块 2 可能有一个哈希值 `345678`。现在，区块 2 的先前哈希将是区块 1 的当前哈希，即 `123456`。这里，我们将这两个区块链接起来，从而有效地创建了一个包含交易账本的区块链表。图 1-4 描绘了这种连接。

区块的哈希基于区块数据本身，因此要验证一个哈希，我们只需对区块数据进行哈希，并将其与当前哈希进行比较。

两个或多个相连的区块（或交易）构成一个区块链。区块链的有效性将取决于每笔交易的有效性。

## 1.4 智能合约

**定义 1-10** *智能合约*是一种自动执行的合约，其中买方和卖方之间的协议条款被直接写入代码行中。

如果交易本身可以由用户编程，那么区块链就是可编程的。例如，用户（不一定是程序员）可以编写脚本，添加在发送资金之前必须满足的额外要求。它可能看起来像这样：

```
1 if (用户拥有超过 10 的货币)
2 then 批准交易
3 else 拒绝交易
```

智能合约被实现为在区块链上发生的计算。

## 1.5 比特币

比特币是世界上第一个区块链实现。2008 年 11 月，一篇由中本聪撰写的题为“比特币：一种点对点的电子现金系统”的论文链接发布在一个密码学邮件列表中。比特币的白皮书共有九页；然而，它主要是一个理论上的设计解释，因此对新手来说可能有点难以理解。

比特币软件是开源代码，于 2009 年 1 月在 SourceForge 上发布。比特币的设计包括去中心化网络（点对点网络）、区块（挖矿）、区块链、交易和钱包，我们将在本书中详细探讨每一个部分。

尽管有许多区块链模型，且每个模型在实现细节上有所不同，但我们在本书中将要构建的区块链将与比特币非常相似，只是对某些部分进行了简化。
