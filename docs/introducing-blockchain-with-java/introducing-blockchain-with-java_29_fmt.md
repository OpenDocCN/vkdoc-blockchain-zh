# 第 6 章 服务层

在第 99–102 行，你会发现我们的 `addTransactionState` 方法。这是一个简单的方法，用于将给定交易添加到当前交易账本中并对其进行排序。如果你还记得，我们在 `init()` 应用程序方法和 `AddNewTransactionController` 类中已经提到过这个方法。

现在让我们看看 `addTransaction` 方法。顾名思义，这个方法的总体思路是将一笔新交易添加到我们的账本中。该方法会接收一个 `Transaction` 对象和一个 `Boolean` 标志，用于标记该交易是矿工的奖励交易还是普通交易。当我们稍后在本章讨论 `mineBlock()` 方法时，会进一步深入探讨奖励交易的用途和实现方式。不过需要注意的一点是，在我们的区块链中，奖励交易会创建新的代币并将其授予挖出该区块的矿工，因此我们需标记此类交易，以便跳过 `getBalance` 方法执行的余额检查。

**重要！** 请注意，不同的区块链在代币总量是否设限以及如何奖励矿工方面有不同的实现方式。我们的做法只是其中一种。

回到我们的方法，观察第 106–109 行。可以看到我们调用 `getBalance` 方法进行检查，判断发送方的代币数量是否大于交易金额。这自然可以确保我们拒绝任何试图作弊、发送超出发送方实际持有代币数量的交易。但请同时注意末尾的说明：我们利用奖励交易标志来跳过此检查（当为奖励交易时）。如果不这样做，奖励交易将无法通过检查，因为 `getBalance` 方法会发现这些代币凭空出现。鉴于该功能旨在奖励矿工，
