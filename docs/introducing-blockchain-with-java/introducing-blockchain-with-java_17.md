你会注意到，和我们的字段一样，所有方法都是我们在关联的 FXML 文件中引用的，因此我们还需要使用 `@FXML` 标签对其进行注解。

**重要提示！** 最佳实践是：控制器类中只包含视图所使用的字段和方法。任何其他可能需要的计算或方法，都应通过服务层进行调用和处理。

让我们从第 57 行的第一个方法 `toNewTransactionController()` 开始。回顾一下，我们在 `On Action` 字段中将此方法与名为 `Make Transaction` 的 `MenuItem` 关联了起来。这意味着一旦我们点击该菜单项，此方法就会执行。我们希望这个方法能够打开一个新对话框，并在其中显示 `AddNewTransactionController.fxml` 文件的内容。第 58 到 68 行实现了这一功能。

第 69 行会创建一个 `FINISH` 按钮，第 70 行则创建了一个监听器，等待该按钮被点击。第 71 到 74 行在 `FINISH` 按钮被点击后刷新我们的交易列表和货币余额。

在第 78 到 83 行，我们定义了 `refresh()` 方法，该方法在刷新按钮中被引用。换句话说，一旦点击刷新按钮，就会触发 refresh 方法，该方法会利用我们的服务层来刷新交易和余额信息。

最后，第 87 行的 `handleExit()` 方法设置了一个 `布尔值`，该值将触发所有并行线程退出其循环并结束运行，而第 88 行则告知我们的 UI 线程结束运行。
