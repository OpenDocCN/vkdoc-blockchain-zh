

# 分布式算法

分布式算法在多台计算机上并发运行，以在分布式系统中完成某项任务。在分布式系统中，同一算法在所有计算机上并发执行，以实现共同目标。

与顺序算法（其中执行中的每个操作依次发生）相比，分布式算法是操作并发执行的算法。并发算法和并发性在计算中相当常见，例如，处理器中同时运行的多个线程、计算机中同时运行的多个应用程序、计算机中的多核处理器，或分布式系统中同时运行的多个进程。

我们可以将分布式算法定义为在多台机器上并发运行的算法。

分布式算法有若干优势，包括但不限于：与顺序算法相比，某些计算可以并行化以实现更高性能。此外，分布式算法具有容错能力；例如，如果一定阈值（我们将在第 3 章中解释）的节点发生故障，分布式算法仍能继续运行。

存在消息传递算法，其中进程通过网络发送和接收消息进行通信。另一种类型是共享内存分布式算法，其中算法通过从共享内存中读写进行通信。

集中式算法顺序执行，而分布式算法并发执行。集中式算法通常不以防故障为特性，而分布式算法则被设计为能容忍各种类型的故障。顺序算法通常更直观，因为它们是为顺序执行而设计的，而分布式算法可能难以理解。这也是集中式算法的正确性证明相对容易做到，并且在大多数情况下甚至通过观察就能显而易见的原因之一。但分布式算法并非如此，其正确性可能具有欺骗性。一个看似正确的算法在实践中可能无法正确运行，并且可能遭受各种故障和属性违反。为此，需要使用多种形式化规范和验证技术来确保分布式算法的正确性。我们将在第 9 章中介绍其中一些方法。与顺序算法相比，分布式算法在设计、调试和实现上往往更具挑战性。此外，从复杂度度量的角度来看，顺序集中式算法通常以指令数量来衡量；然而，分布式算法的复杂度则以消息数量来衡量。
