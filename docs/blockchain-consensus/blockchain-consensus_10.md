# 域名系统与分布式共识

域名系统（DNS）是应用最广泛、最终保证一致性的系统。当域名更新时，它会按照配置的模式进行分发，最终所有客户端都能看到更新内容。

从 CAP 定理的角度来看，分布式共识属于 CP 系统，它牺牲了可用性以换取一致性。因此，分布式共识被用于提供强一致性保证。

例如，假设你有一个五节点系统，其中三个节点发生故障，那么整个系统将停滞，直到另外三个节点恢复。这样做的目的是维护一致性保证，即使系统在一段时间内不可用。

以比特币为例，它看起来像是 AP 系统，由于分叉，会在短时间内牺牲一致性，但最终仍能达到一致。因此，比特币也可以被视为一个最终实现强一致性的 CP 系统。

通常，强一致性（也称为线性一致性）是分布式共识的目标；然而，像比特币这类系统中的最终一致性也是可接受的。

## 分布式系统中的密码学

分布式系统在充满挑战的环境中运行，面临着对手攻击、故障和不可信用户的威胁。在这种条件下，保护分布式系统免受所有威胁变得至关重要。因此，必须采取适当的安全措施来挫败任何破坏系统的企图。

分布式系统中所需的常见安全服务包括机密性、完整性、访问控制和身份认证。为此，密码学在分布式系统中发挥着至关重要的作用。

密码协议通过利用对称密码、非对称密码和哈希函数等密码学原语，在分布式系统中提供安全服务。

我们将在第 2 章中讨论这个主题，届时我们将研究密码学的基础知识及其在分布式系统和区块链中的应用。第 2 章将介绍几种在共识协议和区块链中有助于实现数据完整性和相关安全目的的原语。

## 本章小结

我们在本章中讨论了以下几个主题：

- 分布式系统是一组相互连接的计算机，通过消息传递协同工作以实现共同目标。
- 分布式系统面临若干挑战，其中最突出的是：(1) 缺乏全局知识和一致性认知，(2) 缺乏全局时间，(3) 实现一致性，以及 (4) 故障处理。
- 分布式系统模型使我们能够对整个系统进行推理，并抽象掉所有不必要的细节。
- 闰秒错误曾导致互联网服务严重中断，人们已经开发了多种技术来解决这一问题。
- 分布式系统必须确保安全性和活性属性。
- 崩溃容错分布式算法仅处理崩溃故障，而拜占庭容错算法则用于处理任意类型的故障。
- 逻辑时钟不依赖于物理时钟，可用于定义分布式系统中事件的顺序。
- 在分布式系统中，所有计算机上同时运行相同的算法以实现共同目标。
- CAP 定理指出，分布式系统只能同时满足三个理想特性中的两个，即一致性、可用性和分区容错性。
