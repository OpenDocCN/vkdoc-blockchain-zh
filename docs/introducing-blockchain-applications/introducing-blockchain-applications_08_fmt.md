# 成员服务提供者（MSP）

MSP 能够验证身份。换言之，它们能判定某个身份是否属于特定组织，以及该身份在组织内所扮演的角色。因此，尽管 CA 可以颁发身份和数字证书，但 MSP 能在网络上识别这些身份。

MSP 还向成员提供网络内部的多种角色与权限。

MSP 是将组织身份与其成员资格关联起来的方法。成员的公钥（也称为证书、证书签名）被添加到组织的 MSP 中，以获取成员资格。

MSP 也用于为网络参与者分配各种角色和权限。MSP 是连接身份与组织成员资格的机制。成员的公钥（也称为证书、证书签名）被添加到组织的 MSP 中，以获取成员资格。

另一方面，MSP 的功能不仅限于列举谁是网络或通道的成员。通过识别参与者在节点或通道上的特定权利，MSP 负责将身份转化为角色。用户在向 CA Fabric 注册时必须被分配一个角色，例如管理员、对等节点、客户端、排序节点或成员。

### 本地 MSP

针对客户端和节点的 MSP 是按每个客户端和每个节点（对等节点和排序节点）分别指定的。本地 MSP 控制节点的权限（例如能够运行节点的对等管理员）。在 MSP 的客户端环境中，用户可以将自己认证为通道的成员（例如在链码交易中），或系统中特定角色的持有者（例如作为管理员组织）。

客户本地 MSP 设置在节点的系统文件中，仅适用于该节点。由于排序节点与对等节点一样属于单一机构，因此它们拥有单个 MSP 来标识其信任的参与者或节点。

### 通道 MSP

另一方面，通道 MSP 在通道级别确立管理和参与权限。应用程序通道上的对等节点和排序节点与通道 MSP 共享相同的视图，从而能够正确认证通道访客。这意味着，如果某公司想要加入通道，通道配置中应包含一个与该公司内部人员协作的 MSP。因此，涉及该组织身份的交易将被拒绝。

本地 MSP 记录在通道配置中，而通道 MSP 以文件夹结构的形式存储在文件系统上。通道 MSP 决定谁在通道级别拥有权力。通道 MSP 定义了通道成员身份与通道级规则执行之间的关联。

通道 MSP 将所有参与组织的 MSP 连接到一个统一的排序系统中。排序服务很可能拥有来自多家企业的排序节点，整个排序服务由这些企业共同管理。

### 存储 MSP 数据

本地 MSP 记录在通道配置中，而通道 MSP 以文件夹结构的形式存储在文件系统上。通道 MSP 决定谁在通道级别拥有权力。通道 MSP 定义了通道成员身份与通道级规则执行之间的关联。

通道 MSP 将所有参与组织的 MSP 连接到一个统一的排序系统中。排序服务很可能拥有来自多家企业的排序节点，整个排序服务由这些企业共同管理。

因此，通道 MSP 在逻辑上驻留在通道或网络中并由其管理，但每个节点的本地文件系统都包含所有通道 MSP 的副本。
