# 3.7 交易

比特币的主要功能是促进交易。比特币的其他特性都是围绕实现交易的生成、广播、验证，并最终将其添加到全球交易数据库（区块链）而构建的。在比特币网络中，货币转账被编码为称为“交易”的数据结构。比特币的区块链是一个去中心化的全球数据库，充当全球复式记账系统，公开记录所有交易。

## 3.7.1 交易生命周期

比特币交易的创建、验证和添加到区块链的过程涉及多个步骤，这些步骤构成了其生命周期。

### 3.7.1.1 创建与广播

资金转移的过程始于由一方或多方创建并签署授权文件。交易被验证后，将在比特币网络上广播，网络中的每个节点负责验证交易并将其传播给其他节点。

### 3.7.1.2 验证与收录

矿工节点检查交易并验证其合法性。交易验证通过后，会通过将其包含在由先前交易组成的区块中，进而添加到区块链。为了被永久记录并被所有参与者视为合法，后续的区块（即确认数）必须随之产生。当新所有者花费这些资金时，一个新的交易生命周期就可以开始了。

## 3.7.2 创建交易

创建比特币交易的过程类似于填写纸质支票。在实际执行之前，该交易仅作为传达转账意图的工具，对金融系统不可见。与支票的情况类似，发起交易的人不一定是签署交易的人。

任何人，无论是在线还是离线，都可以创建交易，即使是未经授权的签署人。例如，应付账款文员可以草拟比特币交易，然后由首席执行官批准。与支票不同，比特币交易不会指定某个特定账户作为资金来源。

交易必须由原始资金的拥有者签署才能开始创建过程。一旦准备妥当并签署，交易即为有效，并包含执行资金转账所需的所有信息。最后一步是将有效交易广播到比特币网络并添加到公共账本中。

## 3.7.3 向比特币网络广播交易

将比特币交易提交给网络可以确保它能被众多节点看到并添加到区块链中。数千个比特币节点需要接收一个大约 300–400 字节的小型数据结构，该结构代表一笔交易。

只要使用多个节点来确保传播，发送者就无需信任用于广播的节点。由于交易已签名且不包含任何私人信息，节点也不需要信任发送者的身份。与需要加密网络的信用卡交易不同，比特币交易可以通过任何合适的网络传输技术公开广播。

WiFi、蓝牙、近场通信、条码、网页表单，甚至更非常规的方式如分组无线电和卫星中继，都可以用来广播比特币交易。由于比特币的去中心化结构，交易可以通过多种不同的渠道完成，这使得阻止其创建和完成变得极其困难。

## 3.7.4 在比特币网络上传播交易

当一笔比特币交易被提交给比特币网络上的一个节点时，该节点负责验证交易的合法性。如果交易是真实的，该节点会将其转发给网络中的其他节点，并向发送者发回交易成功的信号。如果交易无效，该节点会向发送者发回一条拒绝消息。

比特币网络中的每个节点在网络首次启动时，都会与多个其他节点建立连接。由于网络呈去中心化、无拓扑结构的网状形态，网络上的所有节点地位平等。通过一种称为“泛洪”的技术，交易从一个中心节点广播到与其通信的所有对等节点。一旦交易被网络处理并接受，它将以指数级方式扩散到每个节点。

比特币网络旨在快速可靠地传播交易和区块，同时防御垃圾邮件、拒绝服务攻击和其他潜在问题。每个节点都独立检查交易，这能阻止恶意交易的扩散范围超过单个节点。

## 3.7.5 交易的数据结构

交易是一种数据结构，它编码了从称为输入的 funds 来源到称为输出的目的地的价值转移。交易输入和输出与账户或身份无关。相反，你应该将它们视为被特定秘密锁定的比特币数量——比特币碎片，只有所有者或知道秘密的人才能解锁。

交易的结构由几个字段组成，如表 3-2 所示。

`Version` 字段指定了此交易遵循的规则。`Input Counter` 字段表示交易中包含的输入数量，而 `Input` 字段包含一个或多个交易输入。

类似地，`Output Counter` 字段表示包含的输出数量，`Output` 字段包含一个或多个交易输出。

最后，`Lock Time` 字段表示一个 Unix 时间戳或区块号，它作为交易被添加到区块链之前必须满足的条件。

`表 3-2` 比特币交易结构

| 字段 | 描述 |
| --- | --- |
| Version | 交易版本号 |
| Input Counter | 交易中输入的数量 |
| Input 1 | 第一个输入的详细信息 |
| Input 2 | 第二个输入的详细信息 |
| … | … |
| Output Counter | 交易中输出的数量 |
| Output 1 | 第一个输出的详细信息 |
| Output 2 | 第二个输出的详细信息 |
| … | … |
| Lock Time | 交易锁定时间 |

## 3.7.6 交易类型

比特币系统中不同类型的交易具有不同的功能和特性。一些典型的业务交易如下：

### 3.7.6.1 标准交易

标准交易涉及双方之间简单的价值交换。它由引用先前交易输出作为资金来源的输入，以及将转移的价值分配给新接收者的输出组成。

### 3.7.6.2 多重签名交易

多重签名交易需要在花费资金之前获得多方批准。多个公钥，通常属于不同的个人或实体，被用作交易的输入。通常，交易的输出随后会用脚本锁定，该脚本指定花费这些资金所需的签名数量。

### 3.7.6.3 隔离见证交易

隔离见证是比特币协议的一次升级，它将交易签名数据（见证数据）与实际交易数据分离开来。隔离见证交易具有独特的结构，并使用独特的交易格式，从而能够提高交易容量和增强可扩展性。

### 3.7.6.4 Coinbase 交易

隔离见证协议增强将交易签名数据（见证数据）从交易数据本身中分离出来。隔离见证交易具有独特的结构，并使用不同的交易格式，从而能够提高交易容量和增强可扩展性。

### 3.7.6.5 支持锁定时间的交易

锁定时间允许交易的发送者指定交易生效的未来时间或区块高度。支持锁定时间的交易适用于多种用例，包括创建时间锁定交易、实现条件性支付，以及创建类似闪电网络的支付通道。

## 3.7.7 交易输入与输出

未花费的交易输出（UTXO）是比特币交易的基础构建块。UTXO 是比特币的不可变单位，它们被锁定给特定的所有者，记录在区块链上，并被整个网络识别为货币单位。比特币网络跟踪数百万个可用的（未花费的）UTXO。每当用户收到比特币时，其数量都会作为 UTXO 记录在区块链中。因此，用户的比特币可能作为 UTXO 分散在数百笔交易和数百个区块中。实际上，并不存在所谓的比特币地址或账户余额；只有分散的 UTXO 被锁定给特定的所有者。钱包应用程序生成了用户比特币余额的概念。钱包通过在区块链中搜索并汇总用户的所有 UTXO 来计算用户的余额。

比特币可以分割为聪（satoshis）的倍数，以便于人们进行比特币交易，同时使其可读。一旦创建，UTXO 必须被整体消耗并生成找零。例如，一笔包含一个 `20` 比特币 UTXO 的交易将消耗整个 `20` 比特币 UTXO，并产生两个输出：支付 `1` 比特币给接收方，并将 `19` 比特币找零返回给钱包。大多数比特币交易都会产生找零。用户的钱包应用程序会自动从其可用的 UTXO 中选择各种单位，以组合出等于或大于期望交易金额的数量。

钱包应用程序可以使用组合较小单位、寻找精确找零，或使用大于交易价值的单一单位并产生找零等策略。这种可花费 UTXO 的复杂组合由用户的钱包完成，对用户是不可见的。交易输入是被交易消耗的 UTXO，而输出是由交易创建的 UTXO。这种消耗和创建 UTXO 的交易链将比特币价值块从所有者转移到所有者，产生找零并确保数字货币的稳定性。

# 3.8 数字签名

比特币使用椭圆曲线数字签名算法（ECDSA）来实现数字签名。ECDSA 是一种基于椭圆曲线的密码算法，可提供安全的数字签名。发送者在创建比特币交易时，使用其私钥为交易数据生成数字签名。此签名作为证据，证明私钥的所有者授权了该交易。ECDSA 通过利用椭圆曲线的数学特性来保证交易的完整性和真实性。除 ECDSA 外，比特币还使用了 SIGHASH（签名哈希）功能。SIGHASH 用于指定数字签名涵盖交易的哪些方面。它允许签名者选择他们授权交易的哪些部分，以及哪些部分可以在不使签名失效的情况下被更改。通过指定特定的 SIGHASH 类型，签名者可以决定签名是否包含整个交易、特定输入或特定输出。这种灵活性使得更复杂的交易方案成为可能，例如需要多个签名的交易或带有附加条件的交易。比特币将 ECDSA 和 SIGHASH 结合使用，确保了整个网络中数字签名的安全性、真实性和适应性。它使参与者能够使用他们的私钥安全地授权交易，同时保持对交易各个组成部分的控制。
