# 共识

共识是指区块链网络中的参与者就账本状态达成一致的过程。在去中心化网络中，没有中央机构来进行决策。相反，网络参与者必须就哪些交易有效以及哪些区块应被添加到链中达成共识。共识机制，例如工作量证明或权益证明，用于确保网络中所有节点对账本状态达成一致。

#### 共识的类型

在区块链网络中，共识是指网络参与者就区块链状态达成一致的过程。共识对于网络的安全性和完整性至关重要，因为它确保所有节点对区块链拥有一致的视图，并防止双重支付和其他类型的欺诈活动。

区块链网络使用多种不同的机制来实现共识，包括：

-   `PoW`：这是最广为人知的共识机制，被比特币和以太坊等网络使用。在`PoW`系统中，矿工们竞相解决复杂的数学问题，以便将新区块添加到区块链中。最先解决问题并添加区块的矿工将获得新创建的加密货币和交易费用作为奖励。问题的难度会随着时间的推移进行调整，以保持稳定的区块创建速率。

-   `权益证明`（`PoS`）：在`PoS`系统中，网络参与者“质押”他们的加密货币作为抵押品，以获得验证新区块的资格。验证者根据其质押的加密货币数量被选出，并通过将新区块添加到区块链来获得交易费用奖励。`PoS`背后的理念是，它比`PoW`消耗的资源更少，因为它不需要矿工解决复杂的数学问题。

-   `委托权益证明`（`DPoS`）：`DPoS`是`PoS`的一种变体，被 EOS 和 BitShares 等网络使用。在`DPoS`系统中，代币持有者投票选举出一小组验证者（称为“受托人”），由他们来验证交易并将新区块添加到区块链中。受托人获得交易费用作为奖励，代币持有者可以投票替换未能令人满意地履行职责的受托人。

-   `拜占庭容错`（`BFT`）：`BFT`是一种共识机制，用于 Hyperledger Fabric 等部分区块链网络。在`BFT`系统中，网络参与者相互通信以就区块链状态达成一致。该系统的设计能够容忍一定程度的恶意行为（称为“拜占庭故障”），例如节点故意试图破坏网络。
