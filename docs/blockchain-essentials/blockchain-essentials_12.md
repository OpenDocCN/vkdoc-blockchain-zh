# 3.6 比特币地址

比特币地址是用于发送和接收比特币的虚拟标识符。它们类似于用于发送和接收电子邮件的电子邮件地址。比特币地址通常由字母和数字组成，长度在 26 到 35 个字符之间。它们拥有私钥，能够实现地址间的交易。比特币地址有多种类型，包括隔离见证（SegWit）或 Bech32 地址、传统地址或 P2PKH 地址、兼容性地址或 P2SH 地址，以及 Taproot 或 BC1P 地址。每种类型都有其自身的特性和优势。由于比特币地址是跨兼容的，因此可以在不同的地址类型之间发送比特币。但是，务必仔细核对收款地址，以免将比特币发送到错误地址。在进行大额交易前，先用少量比特币测试地址，有助于建立信心。找回发送到错误地址的比特币十分困难，但在某些情况下，联系地址所有者或使用 `OP_RETURN` 功能可能会有所帮助。在发送比特币之前，务必验证地址以确保准确无误。

#### P2PKH 或传统地址格式

如果你的比特币地址以 `1` 开头，那么你使用的是 P2PKH 或传统地址，例如 `1BvBMSEYstWetqTFn5Au4m4GFg7xJaNVN2`。P2PKH 代表 Pay-to-Pubkey Hash（支付到公钥哈希），即支付到收款人公钥的哈希值。传统地址不兼容 SegWit，但你仍然可以毫无问题地将比特币从 P2PKH 地址发送到 SegWit 地址。然而，由于 P2PKH 地址的交易体积较大，其交易手续费通常也更高。

#### P2SH 地址格式

P2SH 地址的结构与 P2PKH 地址类似，但以 `3` 开头，例如 `3J98t1WpEZ73CNmQviecrnyiWrnqRhWNLy`。P2SH 代表 Pay-to-Script Hash（支付到脚本哈希），其功能比传统地址更复杂。它通常用于多重签名地址，即需要多个数字签名才能授权一笔交易。P2SH 地址也通过 P2WPKH-in-P2SH 用于非原生 SegWit 交易。对大多数用户而言，重要的是这种地址类型得到广泛支持，并且可以用于向 P2PKH 和 bech32 地址发送资金。

#### Bech32 地址格式

Bech32 地址外观独特，以“bc1”开头，并且由于前缀的存在，比传统地址或 P2SH 地址更长。Bech32 是原生 SegWit 地址格式，受到大多数软件和硬件钱包的支持，但并未被所有交易所完全采用。尽管目前只有不到百分之一的比特币存储在 bech32 地址中，但这个数字正在逐渐增加。

#### 比特币现金地址格式

比特币现金（BCH）地址可以采用传统格式（以 `1` 开头）或现金地址（Cash Addr）格式。现金地址格式基于 bech32，并以“q”或“bitcoincash:q”开头。BCH 钱包可以同时支持这两种格式，允许用户在其中切换。使用现金地址格式的主要目的是为了区分 BCH 与 BTC（比特币），防止资金被发送到错误的地址。
