# 确认

区块确认是指验证交易并将其添加到区块链的过程。这种验证涉及解决数学难题、确保交易完整性以及防止双重支付。节点竞争解决这些难题，最先成功的节点将因其努力获得奖励。验证后的交易被添加到区块中，然后通过挖矿过程将该区块追加到区块链上。

区块确认对于维护区块链的完整性和安全性至关重要。一旦交易被确认并添加到区块链中，它就变成永久记录且无法更改。这个过程消除了对中间商的需求，提高了网络效率，并降低了交易成本。确认有助于防止欺诈活动，并确保区块链网络内部的信任。

### 1.12.7 工作量证明

`工作量证明`（`PoW`）是一种共识算法，用于大多数加密货币以防止双重支付。它要求用户解决复杂的数学谜题来验证交易并将其添加到区块链中。`PoW`确保了交易的完整性并防止欺诈，使其成为维护安全可信网络的必要条件。

在`PoW`中，交易被分组为区块，矿工使用计算能力对区块数据进行哈希处理并寻找谜题的解。哈希过程为每个区块创建一个唯一的指纹，使得无法逆向工程原始数据。矿工进行一场猜谜游戏，修改一个`nonce`值，直到找到满足协议条件的哈希值。成功的矿工将获得加密货币奖励，并可以将新区块广播到网络，其他参与者随后更新其区块链。

寻找有效哈希的难度随网络哈希率增加而增加，从而确保区块发现速率受控。虽然挖矿可能计算成本高昂，但潜在的奖励激励矿工贡献资源以确保区块链安全。

### 1.12.8 区块奖励

区块奖励（也称为挖矿奖励）是一种提供给区块链网络中矿工的激励，作为其验证交易并向区块链添加新区块工作的补偿。

区块奖励由新生成的代币和交易费组成，作为对矿工保障网络安全的奖励。例如，当前奖励是每区块`6.25`个代币，每四年发生一次减半事件以限制总供应量。虽然区块奖励保持稳定，但交易费可能波动。交易通过钱包发起并传输到去中心化的节点网络，这些节点验证和认证交易细节。节点在保障区块链网络安全中起着关键作用，挖矿节点将已验证的交易组装成候选区块。矿工因解决`PoW`算法并提供已确认的金融结算而获得区块奖励。

### 1.12.9 交易与 UTXO

交易是表示加密货币或其他数字资产从一方用户转移到另一方用户的基本构建块。交易通常包含发送方地址、接收方地址、转移的加密货币数量以及交易费等信息。

节点在将交易添加到区块链之前会对其进行验证。未花费交易输出（`UTXO`）在每笔交易中创建，表示与特定地址关联的未花费输出。`UTXO`通过使追踪加密货币流动变得困难并防止从未经授权的地址盗窃，增强了隐私和安全性。

在基于`UTXO`的区块链中，每笔交易创建一个或多个`UTXO`，表示交易中未花费的输出。每个`UTXO`与一个特定地址关联，只能由该地址的所有者花费。

例如，假设`Alice`想向`Bob`发送`1`个`Bitcoin`，`Alice`将创建一笔包含转账细节（包括`Bob`的地址和转账金额）的交易。这笔交易将创建一个与`Bob`地址关联的新`UTXO`，代表转账的`Bitcoin`数额。与`Alice`地址关联的`UTXO`也会被创建，但会被标记为已花费。

当`Bob`想花费他从`Alice`处收到的`Bitcoin`时，他会创建一笔包含转账细节（包括接收方地址和转账金额）的新交易。这笔交易会引用`Bob`在从`Alice`处接收`Bitcoin`时创建的、与其地址关联的`UTXO`。与`Alice`地址关联的`UTXO`仍会被标记为已花费。这使得恶意行为者更难从另一个用户的地址窃取加密货币。

**示例**

*   假设`Alice`想从一家接受加密货币的在线商店进行购买。她通过向商店的钱包地址发送所需加密货币金额来发起一笔交易。交易细节包括接收方的钱包地址、支付金额和一笔交易费。一旦确认并添加到区块链，这笔交易会建立一个与商店地址关联的`UTXO`，代表收到的付款。该`UTXO`现在可以在商店被花费。

*   想象一个去中心化应用（`DApp`）通过本地代币奖励生态参与。当用户与去中心化应用交互并赚取代币时，会生成一笔交易将代币传输到用户的钱包地址。这笔交易会建立一个代表收到代币的`UTXO`，该`UTXO`链接到用户地址。当用户希望在`DApp`内使用这些代币或将其传输给其他用户时，他们可以在后续交易中引用与其地址关联的`UTXO`。这种机制确保了`DApp`生态系统内的代币转账是安全且可追溯的。

### 节点

在区块链网络中，节点是连接到网络并参与交易和区块验证及传播的计算机或设备。节点在区块链网络的安全性和去中心化中起着关键作用。

如下所述，存在几种不同类型的节点。

区块链网络包含具有不同角色的多种类型节点：节点通过`P2P`网络促进通信和验证，在区块链网络中起着关键作用。它们在没有中心化权威的情况下为网络的安全性和去中心化做出贡献。此外，节点可以参与治理过程，例如对网络变更进行投票。通过作为节点运行，用户积极支持区块链网络的整体完整性和功能。

*   `全节点`：这些节点维护区块链的完整副本，验证交易和区块，并将信息传播给其他节点。
*   `挖矿节点`：这些节点解决复杂的数学问题以向区块链添加新区块，并因其工作获得奖励。
*   `轻节点`：也称为轻客户端，它们依赖全节点来验证交易和区块，无需维护区块链的完整副本。
*   `主节点`：在某些网络中存在，这些节点执行附加功能，如网络治理、增强安全性以及启用高级功能。

节点通过`P2P`网络促进通信和验证，在区块链网络中起着关键作用。它们在没有中心化权威的情况下为网络的安全性和去中心化做出贡献。此外，节点可以参与治理过程，例如对网络变更进行投票。通过作为节点运行，用户积极支持区块链网络的整体完整性和功能。
