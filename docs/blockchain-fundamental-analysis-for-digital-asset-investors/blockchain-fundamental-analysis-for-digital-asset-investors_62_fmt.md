# 快速词汇表

本快速词汇表收录了与加密领域内区块链或基于 dApp 的项目基本面分析相关的术语。这些术语在全书各处均有使用；因此，请将以下页面加入书签，以便快速查阅。

### 51% 攻击

*   在基于工作量证明（PoW）的区块链语境中，`51% 攻击`指的是某个实体或团体控制了网络超过 50% 的算力（也称为哈希率）的情况。

### 架构层

*   区块链架构是分层构建的——即代码或协议设计的不同层级——每一层都旨在处理特定的角色。第 0 层、第 1 层、第 2 层和第 3 层协同工作，以提高网络的安全性、速度、效率、互操作性以及跨区块链和去中心化应用（`dApps`）的整体用户体验。

### 增值机制

*   `代币增值机制`是内置于代币设计中的特定策略，旨在随时间推移增加代币的价值。这些方法和技术包括质押分配、回购销毁、费用返还和治理奖励。

### bitcoin

*   `bitcoin` 是第一个去中心化的数字货币和支付系统，运行在安全的点对点区块链上，无需中介即可实现无需信任的交易。

### 区块浏览器

*   `区块链浏览器`（或 `block explorer`）是一种基于网络的工具，用于实时查看和追踪区块链上的交易数据，包括地址、代币、智能合约、源代码、钱包地址、价格及其他活动，以及通过历史（存档）记录查询。

### 区块链

*   区块链是一种去中心化的、不可篡改的数字账本，负责记录跨计算机网络发生的交易，在没有中介的情况下确保透明度、安全性和信任。

### 区块链三难困境

*   每个区块链都具备三个至关重要的特性：`去中心化`、`可扩展性`和`安全性`。`区块链三难困境`指的是在不牺牲其中一项的前提下，同时以最优水平实现这三者所面临的挑战。

### 流通供应量

*   `Circulating Supply` 指的是目前在市场上可获取并活跃交易的总代币或硬币数量，不包括那些被锁定、预留或留待未来发行或开发的代币。

### 闭源

*   “闭源”指的是其源代码为专有且不对外公开的软件，仅允许授权方进行修改或再分发。

### 代码库

*   项目 `codebase` 是软件项目所有源代码文件的完整集合，包括构建、执行和维护应用程序所需的一切。

### 币

*   加密货币（也称为“币”或“coin”）是一种原生的 `协议代币`，由其运行的区块链协议直接发行，这就是它常被称为区块链本地货币的原因。

### 共识

*   共识是区块链网络参与者就区块内交易的有效性以及账本状态达成一致的过程，确保在没有中央权威的情况下实现准确性和信任。

### 共识机制

*   共识机制是一种协议，确保区块链网络中的所有参与者就交易的有效性和账本状态达成一致。

### 密码学

*   区块链中的`密码学`通过使用哈希和数字签名等数学技术，确保数据安全、保护用户身份，并保障交易不可篡改。

### 加密货币

*   `加密货币`（也称为 crypto 或 coin）是一种由密码学保障安全的数字资产，运行在去中心化的区块链技术上，其主要目的是通过激励验证者或矿工维护共识并抵御攻击来保障网络安全。它也用作交易媒介和点对点支付。

### 跨链互操作性

*   跨链互操作性是指不同区块链网络之间能够相互通信、交换数据和资产的能力。这通常通过跨链桥、中继链或原子交换等技术实现，旨在打破区块链孤岛，构建更互联的生态系统。
