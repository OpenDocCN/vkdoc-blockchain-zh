# 区块链架构

本章将分析区块链项目的架构设计。区块链架构可以描述为管理点对点网络上核心操作及参与者交互的结构性基础设施设计。同样的原则也适用于传统企业，其中嵌入式系统和流程决定了日常运营、制造过程、营销策略、业务发展等。这些系统的设置和设计直接影响最终用户体验、效率、盈利能力以及公司的声誉。

就区块链技术而言，架构设计会影响可扩展性、互操作性、网络拥塞程度、交易时间、交易速度、交易成本、功能性和安全性。因此，采用区块链技术的公司，其架构设计对项目的成功至关重要。目标是拥有一个快速、高效、高性能、多功能且具备出色互操作特性的区块链。

**本章讨论的基本概念：**

*   区块链架构层
*   区块链三难困境
*   公有链、私有链、联盟链和混合链
*   无许可链和许可链
*   匿名与隐私赋能区块链
*   治理
*   跨链互操作性
