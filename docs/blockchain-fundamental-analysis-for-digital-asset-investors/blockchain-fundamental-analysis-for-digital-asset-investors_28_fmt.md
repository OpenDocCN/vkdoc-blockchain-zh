# 代币设计与应用场景

## 评估目标

通过评估代币的功能性、是否真正解决现实世界问题，以及其在生态系统中的整体必要性，来判断代币的应用场景是否具有真实且合理的用途。

加密货币代币的设计是其产品或服务不可或缺的一部分。有效的代币架构侧重于设计机制和结构，是确保协议安全性和投资者盈利能力的关键。总体目标是最大化代币的功能和价值，从而有助于提升代币单价。功能不足且核心目的不明确的劣质代币设计，会严重影响任何项目的成功。创新的、多用途的代币设计机制对于满足利益相关者和协议的需求至关重要。当利益相关者认可产品和代币本身的价值时，代币需求就会上升。在本节结束时，投资者将了解如何评估代币是否具有合法、可行的应用场景，以及其是否具备真实的用途和强大的价值获取机制。

如果一个代币的用途和功能无法得到合理解释，那么该项目注定会失败——换句话说，用户是否可以在没有代币的情况下操作项目的核心产品？如果没有有效的代币用例，并且无法确认项目的核心产品无法通过功能性代币来运作，投资者应避免投资。至关重要的是，代币必须具备必要的功能，使用户和客户能够享受产品，而这些功能是传统方法无法实现的。

那些转向区块链技术来解决现实世界问题或提高系统效率的传统公司，与那些创造没有任何真实需求的代币的公司相比，更有可能拥有合法的代币应用场景和需求。欺诈性公司通常会捏造极其拙劣的代币应用场景，并通过营销将其描绘成独特且具有吸引力的投资项目，其目的仅仅是通过首次代币发行（ICO）及其他类似的公开销售来筹集大量资金。因此，投资者必须掌握识别和区分那些基本面差、具有欺诈性的代币应用场景与那些具备良好且必要功能的合法应用场景的技能。

例如，考虑一家汽车经销商，它使用以太坊的智能合约创建了`CAR`代币。购车时，客户可以选择使用`CAR`代币（而非法定货币）支付，以此获得未来购车的奖励。乍一看，这似乎很有吸引力；然而，人们已经在使用现金购买汽车，且没有任何问题，并利用传统的忠诚度计划来获取奖励。因此，`CAR`代币的应用场景并不具备任何传统方法无法实现的合法用途或功能。不幸的是，这种情况在区块链世界中屡见不鲜，有时很难识别。事实是，大多数加密货币项目在第一年就失败了，无论它们从第一天起就是骗局，还是因为团队经验不足或缺乏资金；最终都归结为基本面极差的应用场景，毫无希望提供真正的价值或获得发展动力。

## 评估代币设计

专门用一整本书来讨论代币设计也不为过，因为它本身就是一个专业领域。然而，对于本次基本面评估的目的而言，一个简化的分析就足够了。表 7-5 展示了两个代币：`Chainlink`（LINK）和一个虚构代币`HotelStay`（HTS），在最少但有效的代币标准下进行了对比，这些标准有助于验证代币设计用例和功能的合法性。

**表 7-5** 代币设计合法性

| 代币设计评估 |
| --- |
| 代币“合法性”标准 | `Chainlink`（LINK） | `HotelStay`（HTS） |
| --- | --- | --- |
| **代币功能/实用性** | − 用于支付预言机服务费用。例如，使用`Chainlink`预言机获取链下数据需要支付`LINK`。− 作为维持数据准确性的激励和抵押品。当数据提供商向智能合约提交信息时，如果提供不准确的数据，他们将面临质押的`LINK`丢失的风险。这激励数据提供商提供可靠的数据。− 奖励通过质押`LINK`来帮助维护网络安全的持有者。− 预计`LINK`持有者将在计划中的治理框架上线后，参与去中心化的链上治理。 | − 专门用于酒店预订。− 用作忠诚度奖励。 |
| **解决现实世界问题** | 连接区块链和外部数据源。 | − 没有解决独特的问题。− 服务可与现有的传统支付和忠诚度系统配合使用。 |
| **代币的必要性** | − 对预言机操作和网络安全至关重要。 | − 对于平台功能而言并非必需。− 可以被传统货币取代。 |

如表 7-5 的结果所示，`LINK`代币拥有许多对`Chainlink`预言机服务至关重要的基本功能。该代币是`Chainlink`提供的整体运营和服务（通过向智能合约提供现实世界数据，将外部数据源连接到区块链）中不可或缺的关键要素。另一方面，`HotelStay`并没有解决实际问题，也没有提高传统系统的效率，因此，利用区块链技术进行酒店预订没有任何益处。更具体地说，代币`HTS`不具备足以证明其作为“代币”合理性的重要功能，因为法定货币和传统忠诚度积分系统也能达到同样的效果。`HotelStay`的代币`HTS`不符合合法代币应用场景的条件，应被视为一个重大危险信号。

> **事实：** 如果一家公司的代币应用场景很薄弱，他们可能会试图隐瞒关键细节——这应被视为一个警告信号。拥有合法用例的知名公司是透明的，并提供关于其运营以及他们旨在解决的问题的充分信息。

### 行动步骤

按照以下步骤评估代币的应用场景是否具有真实且合理的用途：评估其功能性、是否真正解决现实世界问题，以及其在生态系统中的整体必要性。

1. **代币设计与功能**
   审查官方项目文档，全面了解代币的功能、应用场景、用途等。

2. **评估代币的合法性**
   根据表 7-5 中指定的设计标准（解决现实世界问题、代币功能/实用性以及代币的必要性）对代币进行评估。

3. **附加思考与问题**
   1. 提议的应用场景能否通过传统（链下）解决方案实现？
   2. 团队是否强调了实施其产品或服务的任何显著优势？
   3. 对于该应用场景，选择链上而非链下是否有其他原因？
   4. 在链上还是链下使用产品或服务成本更低？
   5. 在链上还是链下使用相同的产品或服务更方便、更用户友好？
   6. 在链上还是链下使用相同的产品或服务速度更快？

4. **记录笔记并以你自己的风格记录调查结果**

5. **将调查结果与基本面评估过程的其他部分相结合**

### 结果评估

在评估了代币的合法性后，如果确定该代币不符合合法应用场景的条件，建议避免进行任何形式的投资，无论规模大小。另一方面，一个基本面强大且功能合理的代币对投资者来说被视为一个绿灯信号。

## 代币价值累积流程与机制

### 评估目标

判断项目是否具备有效的代币价值累积流程与机制，以帮助确保持续的价值增长和长期的投资者需求。

理解代币价值累积流程，首先要掌握价值累积的概念——即代币生态系统如何创造需求、随时间维持需求，并将该需求转化为有形价值。在区块链领域，价值累积在于创建一种机制，将外部的使用或兴趣转化为协议内持久的经济效益。

理解代币价值累积流程，首先要掌握价值累积的概念——即代币生态系统如何随时间创造并维持需求，并将该需求转化为已捕获价值。在区块链领域，已捕获价值是指协议或代币生态系统保留在其系统内部，而非让其流出的经济价值。当协议或代币创建了一种机制，将外部需求或使用转化为可持续的内部价值形式时，就实现了价值捕获。例如，在去中心化金融协议中，用户通常需要支付协议费用才能访问服务——这笔费用（通常是交易的一部分）可以被协议“捕获”，用于再分配或增长。这个被捕获的价值随后可以再投资到系统中，用于奖励代币持有者，或储存在财库中以资助未来的增长。

分解来看，价值累积始于为代币创造需求，随着时间的推移维持这种需求，并——关键一步——捕获由此产生的部分经济活动（例如，通过费用、销毁或收入分享），使其回流到代币持有者和项目手中。

**事实：** 单纯的价格变动并不代表价值累积，价值也可能在没有立即价格变化的情况下累积。投资者关注协议如何独立于价格、通过需求来捕获、保留并反映价值，这一点至关重要。

### 代币价值累积流程与机制

代币价值累积流程与机制是推动价值累积的工具，它们将实际效用与需求捕获和稀缺性机制（例如费用、销毁和质押锁定）相结合，使代币能够随时间吸收网络经济活动的一部分份额。在讨论为代币创造并维持需求时，本节概述的流程和机制就是人们长期持有或使用代币背后的“原因”。投资者有责任了解项目团队是否以及在如何实现价值累积。更具体地说，要验证是否采用了有效的价值累积流程和机制（如果有的话）。

#### 代币价值累积流程

代币价值累积流程主要有三种类型，如下所示：

- **协议再投资**

  此流程涉及收集一部分已获得的利润，并将其再投资回协议。这些资金用于支付开发者以及其他能够整体提升产品或服务的杂项费用。此策略能增加价值输出和代币累积。

  大部分利润通常会存放在财库中。任何产生收益的部署（例如质押和提供流动性）都应分散化、设置上限并经由治理批准，以避免过度集中的风险。财库资金得到维护，只在需要时才使用——这种类型的价值累积源于*未来机会*。然而，关键在于财库应由代币持有者治理，而非仅由项目团队掌控。请注意，本质上，这种价值累积是通过扩大协议财库并增加其未来潜力，间接提供给代币持有者，而非以经济利益的形式直接提供给代币持有者。

- **治理参与**

  一个公平、去中心化的治理模型是价值累积的关键部分，因为它允许代币持有者影响协议随时间捕获和保留价值的方式——没有它，所有决策都由项目团队做出，导致项目高度中心化，并且在多数情况下缺乏可信度。这种引导协议发展的能力意味着投资者直接参与价值创造，而不仅仅是寄希望于价格上涨。关键治理领域包括与核心产品和服务相关的项目，以及利润分配与再投资策略等关键领域。通过治理做出的决策，其总体目标应是将价值保留在生态系统内部。

- **代币供应管理**

  代币供应管理指的是如何管理代币供应，即减少和分配供应量。请注意，这并非关注启动时全部供应量的初始代币分配，而是关注由特定代币累积机制所产生的供应量减少和分配。例如，根据代币累积机制的不同，一部分代币供应可能会被销毁，因此理论上会创造对剩余供应的更高需求。或者，一部分交易费用可能会按比例分配给代币持有者。项目团队和利益相关者（通过治理）必须利用专门设计、符合产品和整体代币经济结构的代币累积设计机制，来有效管理和控制代币供应。一个现实中的例子是 BNB 智能链上的 `BNB`，其季度自动销毁机制会根据链上活动和收入永久移除一部分供应量。

#### 代币累积机制

代币累积机制是区块链协议中的功能，旨在通过激励持有和可持续需求为代币创造长期价值。它们对投资者至关重要，因为它们能帮助代币随时间增值，而非仅追求快速买卖获利。这些机制内置于协议中，确保代币的长期需求——无论是通过质押奖励、回购，还是通过销毁代币来减少供应量。换言之，这些功能旨在持续为代币增加价值，使其更具吸引力而令人愿意持有而非立即抛售——这是随时间推移增强价值累积并吸引更多投资者的关键要素。代币累积机制有以下几种类型：

- **费用返还机制** —— 将部分交易手续费返还给代币持有者。
- **通缩机制** —— 随时间推移减少代币供应量，从而提升需求和剩余代币的价值。通缩机制可以涵盖多种形式，例如：
    - **交易即销毁** —— 按比例销毁部分交易手续费。
    - **回购并销毁** —— 项目方（使用财库资金）在市场上回购部分代币并将其销毁。
- **服务机制** —— 用户可能需要使用代币支付费用才能享受核心产品。部分服务费有时会被销毁，这有助于产生需求。
- **质押机制** —— 质押涉及锁定代币，减少流通供应量，从而产生需求。质押奖励也会根据质押的代币数量按比例分配给持有者。
- **治理机制** —— 随着链上治理的兴起，代币持有者可以对协议决策进行投票，有时会因其参与而获得奖励。通常，投资者的投票权与其持有的代币数量成正比，尽管有些协议通过替代模式来抑制过大的影响力——例如二次方投票，每增加一票所需代币成本呈指数级增长；信念投票，对锁定时间更长的代币所支持的投票赋予更大权重；或委托投票。
- **锁定机制** —— 项目方通常会向持有者提供激励，要求其将代币锁定一定期限。锁定机制将代币移出流通，从而产生需求。
- **折扣效用机制** —— 当用户使用代币支付时，为其提供费用或价格减免，主要目的是吸引和留住用户；任何价值累积都是间接的，来源于这些折扣所产生的额外活动。
- **其他分发机制** —— 这可能包括空投、流动性引导和流动性挖矿。

#### 行动步骤

请按照以下步骤判断项目是否具备有效的代币累积流程和机制，以帮助确保持续的价值增长和长期投资者需求。

1. **识别代币累积流程和机制**
    进行研究，以识别项目所采用的代币累积流程和机制。这些信息通常可通过官方项目文件获取，例如白皮书、博客、项目网站等。

2. **评估代币累积流程和机制**
    使用以下标准评估代币累积流程和机制。
    1. **机制的真实性** —— 投资者应确保代币累积流程和机制与项目白皮书一致，而非基于模糊的承诺。
    2. 代币设计中内置了哪些类型的代币累积流程和机制？
    3. 哪些因素导致代币或币种价值上涨，它们为何会产生这种效果？
    4. 代币价值的根本驱动因素是什么？
    5. 除了潜在的价格上涨，代币持有者还能通过哪些方式从代币中获利？
    6. 是否存在可能对代币价值产生负面影响的明显限制？
    7. 项目方采取了哪些保护措施来确保代币的内在价值？
    8. 协议如何产生收入（例如费用和价差），这些资金是如何在财库、核心团队和代币持有者之间保留或分配的？

3. **竞品研究**
    将项目的代币累积流程和机制与其他成功的竞争者进行比较，并依据以下标准进行评估。
    1. 竞争对手是否拥有相同或不同的代币累积流程和机制？
    2. 如果相同，他们的表现如何？代币是否提供了很大价值？
    3. 如果使用了不同的代币累积流程和机制，它们是更有效、更具创新性，还是表现不佳？
    4. 竞争对手在创造和保留收入以支撑代币价值方面是否更胜一筹？
    5. 竞争对手的机制在设计上风险更小或限制更少吗？

4. **记笔记并以你自己的风格记录你的发现**
5. **将这些发现与基本面评估过程的其他部分相结合**

### 结果评估

如果项目团队已实施有效的代币和协议价值累积策略与机制——最好有经过验证的业绩记录和可验证的表现，那么基本面评估的这一部分结论是正面的。另一方面，如果团队缺乏有效的代币和协议价值累积机制，则应极度谨慎。
