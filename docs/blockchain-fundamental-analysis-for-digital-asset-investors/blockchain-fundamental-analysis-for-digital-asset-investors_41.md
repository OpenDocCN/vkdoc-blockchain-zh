# 背景与经验

**评估目标：评估加密团队的法律合规性、背景与经验，确保其具备资质与能力，能够为项目的成功做出贡献。**

项目团队，更具体地说，他们的背景、经验、声誉和技能，是决定项目成败的另一个关键因素。正如之前所讨论的，团队负责将项目变为现实，开发愿景、产品或服务，并执行整体战略。然而，即使确保了关键职位的人员配备，但如果每个成员缺乏高效完成手头任务的足够经验，也会损害项目的表现。虽然并非百分之百必要，但如果团队成员拥有区块链领域的经验，包括参与过以往成功的项目，这一点将非常值得赞赏。这样的专业知识使他们能够优化技能组合，从过去的错误中吸取教训，并从自身工作以及与其他区块链专业人士的合作中获得宝贵的见解。

深入调查团队历史的另一个关键原因是有助于避免欺诈项目和骗局。例如，2018 年，一个名为 [Pincoin](https://techcrunch.com/2018/04/13/exit-scammers-run-off-with-660-million-in-ico-earnings/%253Fguccounter%253D1%2526guce_referrer%253DaHR0cHM6Ly93d3cuZ29vZ2xlLmNvbS8%2526guce_referrer_sig%253DAQAAACtX5mBmXxM-LnsPnkC1W0ps3ERcpzVUZFnL7k_R-E1vsWzYTjUHaK7ugK0UiiyddZ-w6OkUbi2BNl7R7kfZrvKVm2qBKtZWUovJh68roCzR6naR6oqNvfqMGQ4gDyLUjkV3BgscQGUWuohArO6y84jdGZ1wt5T7wFymQ4obgVGI) 的项目进行了首次代币发行（ICO），从大约 32,000 名投资者那里筹集了 6.6 亿美元。不久之后，该团队便卷款消失。另一家名为 [LoopX](https://www.zdnet.com/article/loopx-pulls-exit-scam-walks-away-with-4-5m-in-investor-funds/) 的加密公司通过 ICO 筹集了 450 万美元后便销声匿迹，然后删除了其所有在线内容，让投资者血本无归。

**事实**

一个全面的项目团队应包含健康平衡的专业人士组合，他们在区块链、软件工程、金融、代币与代币经济学设计、市场营销和法律等领域拥有扎实的背景。

## 评估团队背景与经验

分析项目团队的主要途径是通过领英（LinkedIn）。如果某个加密项目没有包含团队成员的领英页面，这是极不寻常的——如果发生这种情况，应将其视为一个危险信号。以下程序概述了评估不同项目团队成员的标准。请注意，团队成员的数量将根据项目的知名度而有所不同。例如，虽然一个新兴初创公司可能只有十到二十名成员，但像以太坊这样知名的项目则有超过数百名成员。话虽如此，并不需要评估所有成员，而应侧重于表 10-1 中列出的关键角色。分析八到十个个人资料，就能对团队成员的质量、经验和可信度有一个全面的了解。

1. **领英公司页面**

   在项目网站上寻找团队领英公司页面的链接，或者在领英上搜索“公司”来找到它。

2. **按角色筛选**
   1. 在领英上，在公司页面中按“人员”进行筛选。然后，搜索特定角色（例如，CEO 和开发者）以查看相应团队成员的资料。
   2. 建议从创始人、CEO、联合创始人、软件开发者、代币经济学家、业务发展、市场营销等人员开始查看。

3. **审查团队成员资料**

   根据以下领英特定标准评估每个团队成员的个人资料。
   1. **个人资料照片** – 确保该人员有头像。在谷歌图片上对该图片进行“反向图片搜索”，以帮助排除诈骗的可能性。
   2. **关注者和人脉** – 该人员有粉丝和人脉吗？拥有 1000 以上的粉丝和 500 以上的人脉是值得赞赏的。
   3. **关于部分** – 这部分提供了个人工作经验和成就的简要总结。
      1. 是否有任何在区块链技术方面或参与过往加密项目方面的突出点？
      2. 是否有任何重大成就？
   4. **动态** – 此部分仅显示个人在发布、分享和与其他帖子互动方面的动态。
      1. 最近几天、几周和几个月内是否有足够的活跃度（这有助于证明个人资料的真实性和可信度）？
      2. 是否有对帖子的互动？如果有，发表评论的人是否是合法资料？
   5. **经历**
      1. 他们之前曾在哪些公司工作过？
      2. 他们在那里工作了多长时间？
      3. 他们以前工作过的公司知名吗？
      4. 这些公司在领英上有公司页面吗？如果有，它们看起来合法吗？
      5. 该人员是否曾拥有或共同创立过以前的加密或金融公司？如果是，这些公司成功吗？
      6. 该人员在以前的职位上有哪些任务和职责？
      7. 在以往工作中获得的技能和专业知识是否与当前的项目和职位相辅相成？
      8. 自从职业生涯开始以来，他们是否一直在进步？
   6. **教育** – 此部分列出了个人的全部教育经历。虽然拥有完整的区块链相关背景值得赞赏，但这并非必需。这适用于各种专业人士，包括网络开发者、营销专家以及法律和财务顾问。
      1. 该人员是否拥有与其当前或过去职位相匹配的足够教育背景？
      2. 该专业人士近期是否参加过课程以跟上技术进步的步伐？
      3. 专业学位、硕士和博士学位是否来自经认可的机构和大学？
   7. **执照与认证** – 任何专业执照和认证都列在此部分。
      1. 该人员是否拥有与项目需求互补的任何专业认证？
      2. 这些执照和认证是近期获取且仍然有效的吗？
      3. 这些执照和认证是否经过认证？
   8. **技能** – 在此资料部分，个人列出了他们的技能。
      1. 评估列出的技能，重点关注那些得到其他专业人士认可的技能。
      2. 这些获得认可的技能是否与区块链项目所需的技能和专业知识相符？
   9. **推荐信** – 在这里，你会找到来自其他专业人士（通常是曾与该人员共事的人）的推荐。由于任何人都可以给出推荐，因此最好进行一些快速核查：
      1. 通过查看给出推荐的人的个人资料，来检查任何“推荐信”的有效性。
      2. 检查双方的工作经历，以确定他们是否真的共事过。
   10. **发表物** – 此部分列出了任何发表物。
       1. 点击发表物的链接，检查其有效性。
       2. 这些发表物是否与该专业人士在项目中的职位直接相关？

**专业提示**

访问该人员以前工作过的领英公司页面。查看他们当前员工的资料以确保真实性。确保这些员工已填写了可验证的领英资料，并显示他们曾在合法的公司工作过。



### 行动步骤

按照以下步骤评估加密货币团队的合法性、背景和经验，确保他们具备资质且有能力为项目成功做出贡献。

1.  **通过领英评估项目团队的经验和背景**

#### 评估结果

加密货币项目必须在领英上拥有公司页面，且团队成员必须是合法且完全可验证的。如果你无法确认这一点，请将其视为一个危险信号。至少，团队成员——尤其是核心团队成员——应具备区块链或金融领域的经验，这有助于建立对项目成功潜力的信心。

## 开发者团队规模

**评估目标：确定项目团队的开发者人数，以确保项目拥有足够的人才来构建、改进和扩展其产品，同时保持竞争力。**

开发者被认为是构成任何基于区块链的项目基础的、核心的基石要素之一。没有活跃的开发者基础，大多数项目会陷入停滞并面临僵化风险——尽管少数模因币或分叉链在无需大量持续工作的情况下仍能存续。开发者的职责涵盖从区块链基础设施开发、去中心化应用（dApps）、智能合约、通过 API 集成软件应用、发现和消除漏洞、产品测试、实施持续更新等诸多方面。

规模适中的开发者团队对任何项目来说都是一项重要资产。拥有足够的开发者，工作可以高效分配，使每个开发者能专注于各自的专长领域，例如构建区块链基础设施、智能合约、去中心化应用、压力测试以及应对各种技术挑战。大型团队还能随项目扩展而成长，行动更迅速，并通过均衡分配工作负荷来避免过度疲劳。

在大多数情况下，项目会由几位核心团队开发者组成，并辅以更多来自社区的、为项目贡献力量的开源开发者——通常使用 [GitHub](https://github.com/) 等软件进行协作。例如，在撰写本文时，[Polkadot Network](https://www.linkedin.com/company/polkadot-network/people/%253Fkeywords%253Ddevelopers) 列出了大约三十位核心开发者——你可以在领英上看到一个大致的数字，但更可靠的衡量标准是其 GitHub，上面显示有 [409 位开源贡献者](https://github.com/paritytech/polkadot-sdk)。对于 Polkadot 这类热门项目，这样的开发者团队规模是标准的；然而，并非每个项目都如此。

加密货币项目的团队规模，包括开发者人数，因诸多因素而异，例如项目类型（如 dApp 或基础设施）、项目阶段、项目预算、复杂程度，以及市场条件和劳动力需求等外部因素。新启动的项目的核心团队和开源贡献者人数会明显较少，而受欢迎且成功的项目则能吸引整个行业内开发者社区更多的关注，从而获得更多的核心团队和开源开发者。

鉴于所有这些变量，投资者很难精确预测一个项目应该拥有多少开发者。根据一般经验法则，核心团队中至少有十名开发者是值得赞赏的。开源开发者的人数会因项目的热门程度而有巨大差异；因此，建议通过评估直接竞争对手来获得准确的见解。

### 成立年份

在分析项目核心团队的开发者人数时，考虑项目的成立年份是有益的。例如，假设一个项目仅仅在几个月前启动。在这种情况下，期望一个较小的团队是合理的，因为他们可能仍在积聚势头并吸引新成员加入社区。然而，对于一个已经存在一段时间（例如超过一年）但仍然只有少数开发者的项目来说，这被认为是一个危险信号。这可能表明他们的产品或服务不佳、资金困难、社区兴趣不高，或者这些因素兼而有之。考虑到成立年份，投资者可以更好地判断团队规模是否符合其成长阶段，以及这对项目的长期潜力可能意味着什么。

### 行动步骤

按照以下步骤确定项目团队的开发者人数，以确保项目拥有足够的人才来构建、改进和扩展其产品，同时保持竞争力。

1.  **确定项目团队的开发者人数**
    1.  访问项目网站，找到其领英公司页面的链接，该链接通常位于“团队”或“关于”部分。如果找不到，请在领英上使用搜索栏中的“公司”过滤器搜索该项目。
    2.  按“人员”进行筛选。
    3.  **按专业职位筛选** – 在领英搜索栏中按开发者职位进行筛选（例如，搜索“开发者”、“软件工程师”、“区块链工程师”等）。这将显示在该项目团队中工作的开发者人数。
    4.  **查看 GitHub 贡献者** – 打开项目的主要 GitHub 仓库，查看右侧面板上的“贡献者”模块；显示的数字是已提交代码的个人总数，可以快速了解项目周边的更广泛的开发者社区规模。

2.  **竞争对手分析**  
    将核心团队的开发者人数与竞争对手进行比较。

3.  **成立年份**  
    访问项目的领英页面，进入“关于”部分，并查看底部的“成立”副标题，以此找到该加密货币公司的成立年份。有关成立年份的更多详情，请参阅第 4 章“核心产品”中的“成立年份”一节。

4.  **记录笔记，并以你自己的风格记录发现**

5.  **将发现与基本面评估流程的其他部分相结合**

#### 评估结果

如果项目已经存在超过一年，但核心开发者仍然少于十人，这表明缺乏成长和成功，是一个潜在的危险信号。对于仅成立几个月的新项目来说，拥有 5-10 名核心开发者是正常现象。然而，超过二十人是值得赞赏的，因为这显示了强劲的活跃度和受欢迎程度。
