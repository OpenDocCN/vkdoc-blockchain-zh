# 比特币区块链的动荡与 Corda 平台的崛起

2011–2013 年对比特币区块链来说是动荡的时期，这段时间其声誉受损的影响在该技术上留下了印记。由于投资者在`Mt. Gox`比特币交易所（主要由于管理不善和安全系统不完善）遭遇投资损失，`Silk Road`作为非法物品热点，登上了大多数新闻网站的头版。

如果说这些事件有一个积极影响，那就是其获得的关注度。随着`Mt. Gox`的管理缺陷被审视，越来越多的企业家意识到已经犯下的错误以及`Blockchain`的潜力。随着早期采用者和天使投资者开始参与并建立业务，风险投资很快跟进。在很短的时间内（到 2014 年），甚至连`Andreessen Horowitz`这样的风投公司也开始投资这项技术。

随着风投开始被这些新业务吸引，银行和华尔街也随之跟进也就不足为奇了。2014 年，`R3CEV LLC.`成立，旨在利用`Blockchain`创造金融创新。`R3 Consortium`是一个由全球 50 多家领先金融机构（包括所有“大到不能倒”的银行）组成的合作机构，他们共同且独立地工作，为现代金融市场创建“分布式账本技术”。银行已经意识到，如果要获得这项技术的好处，就必须建立共同标准和共享平台。`Corda`是底层的分布式账本软件，作为一个通用平台运行。

## 分布式账本技术与区块链的区别

有必要说明“分布式账本技术（`DLT`）”和`Blockchain`之间的区别。分布式账本和加密货币系统在交易验证方式上有所不同：虽然`Bitcoin`使用假名和匿名节点来验证交易，但分布式账本要求具有法律身份（许可节点）来验证交易（`Swanson, 2015`）。因此`DLT`的行话与其上下文相关，它专门针对许可型`Blockchains`而设计。

## Corda 的核心原则：通用性

`Corda`的基本原则是通用性。`Corda`旨在作为一个全球数据平台，记录其上发生的所有交易，而不是让一组分散的账本独立工作并服务于一部分市场参与者。这使得市场能够协作并维护准确、共享的记录，让重复、错误和对账成为过去。如果我们都在同一个平台上，能够访问相同的数据，那么我们为什么还需要各自独立保存多份副本呢？通过`Corda`，授权方可以连接到账本，创建协议（以智能合约的形式），并将该协议记录在账本上。此外，只有交易对手方（以及有其他合法需求者，如监管机构）才能访问这些交易细节。

## 开放生态与平台特性

为了提高平台的广泛采用，系统部分内容是开放的：开源、开放开发和开放标准。因此，`Corda`允许现有的和新的提供商在不同层面竞争开发新服务，促进选择和竞争。

根据`Richard Brown`（`R3`的`CTO`）的说法，`Corda`的关键特性将包括（`Brown, 2016`）：

*   不必要的全球数据共享：只有那些有合法知情权的当事方才能看到协议内的数据；
*   无中央控制器的跨公司工作流编排；
*   在单个交易层面而非系统层面达成公司间的共识；
*   设计直接支持监管和监督观察节点；
*   交易由交易参与方而不是众多不相关的验证者进行验证；
*   支持多种共识机制的系统；
*   在人类语言的法律散文文档和智能合约代码之间建立明确的联系；
*   行业标准工具；
*   无原生加密货币。

`Corda`的最终目标是创建一个未来的金融市场，在这个市场中，“金融协议被记录并自动管理而不会出错，任何人都可以为任何合同目的无缝交易而无需摩擦。” `Corda`即将推出，在撰写本简短总结笔记时，只有一份介绍性白皮书解释了平台的范围和描述。2017 年对于`DLT`、`R3`和整个市场来说都将是充满趣味的一段时间。

## 结语：市场与去中心化

资本主义市场是价值创造的源泉、繁荣传播的手段，因此也是人类社会的命脉。但当前技术的影响及其所采取的形式，要求我们更新对市场的理解。直到今天，市场被定义为一种媒介，商品和服务的交换在其中发生，这是买卖双方直接或通过中介机构或代理人相互接触的结果。但随着去中心化成为当今市场的一个显著特征，我们需要确定这对我们市场的概念化意味着什么，以及需要什么样的法规才能使这个新的市场框架更具包容性。
