# 复杂性经济学模型与基于 Agent 的计算经济学

> "我们塑造工具，而后工具塑造我们" —— 约翰·卡尔金

模型是对过程、事件或实体的抽象描述。当一个模型能够接收特定输入，并以算法方式处理这些输入值以生成输出时，它就被称为`计算模型`。在计算模型中，`智能体`(`Agent`)是计算机模拟中具有特定属性、状态和行为的自主独立元素。

`基于 Agent 的建模`（`ABM`）是一种计算建模范式，使我们能够描述智能体在预定义模型中的行为方式。在`ABM`中，智能体的行为以简单（或复杂）规则的形式编码，这些规则支配着智能体与模型及其他智能体的交互方式（Wilensky and Rand, 2015）。这些模型可用于研究广泛的过程和系统，如今正逐渐应用于经济学研究。

## 传统模型：基于方程的建模（EBM）

以往用于经济学研究的模型通常采用方程形式。这种建模形式被称为`基于方程的建模`（`EBM`）。`EBM`通常假设同质性。`ABM`与`EBM`的不同之处在于，它可以用于对异质群体进行建模。通过这种方式，`ABM`能够可视化智能体之间的交互，并给出离散且非连续的结果。正如我们在本章前面部分讨论过的，连续模型无法代表经济现象，因为它们无法让我们进行精确计数。例如，在人口研究中，`EBM`将人口视为连续量，而实际上人口是由离散个体组成的（不可能有 1.5 个男人或 0.5 个女人）。因此，`EBM`要发挥作用，就需要对大量（如人口规模）进行假设，并需要对这些大群体进行结果汇总。

这种汇总特性是`ABM`与`EBM`的一个关键区别——`EBM`需要完全了解智能体之间的关系。如果有两个智能体群体，'A'和'B'，那么要构建一个`EBM`，你需要理解群体'A'和群体'B'之间的关系。由于群体被视为连续量，`EBM`要求我们知道群体 A 的总体行为和群体 B 的总体行为。编码这种总体知识需要使用微分方程，这使过程变得复杂。即使完成了这些，仍然存在另一个劣势。`EBM`建模者可能认识到这两个群体之间存在相互关系，但他们无法在`EBM`中明确表示这些行为。

## 基于 Agent 的建模（ABM）的优势与特点

另一方面，`ABM`并不试图模拟全局的总体模式。`ABM`不关注群体的总体行为，而是专注于为每个群体中的离散智能体编写规则。这减轻了建模智能体之间交互的压力。模型专注于观察这些智能体如何交互以及会产生什么模式。

基于此，在不同时间周期内，将生成并观察到总体行为趋势。由于`ABM`模型描述的是智能体而非总体，因此`ABM`与现实世界的对应关系更加紧密。因此，它更容易理解，因为它为我们提供了一个我们在自然界中习以为常的参照系，而不是依赖一套与现实毫无相似之处的代数方程。

由于`ABM`模型对每个智能体、它们的决策以及交互进行建模，它们可以同时提供个体层面和总体层面的结果。还可以研究一个智能体或一组智能体的历史，以及随时间演变出的总体趋势。这种`ABM`的自下而上方法为我们提供了比`EBM`更详细的信息——后者仅提供总体结果，而不提供关于智能体及其交互的任何信息。

`ABM`还允许我们轻松地加入随机性和概率元素。`EBM`要求模型中的每个决策都是确定性的。然而，这同样不符合现实世界的运作方式。智能体所处的经济环境容易受到冲击（随机性）和其他智能体态度变化（反馈）的影响。因此，随着时间的推移，智能体根据反馈和不断变化的环境轮廓做出系列决策。他们的决策并非确定性的，也非沿着一条直线前进。

构建一个基于确定性决策结果的模型，意味着我们需要了解环境的每一个细节以及智能体的每一个思维概念。这可以在一个受控环境中针对一小群智能体完成。但现实世界是混乱的、极其复杂的，远非这种无菌环境。智能体受到随机事件的影响，并在一个连续的适应过程中做出基于概率的决策。

由于`ABM`专注于概念化个体智能体，它不必考虑每个智能体将如何对众多环境因素做出反应。环境因素是模型的输入，并创建了生态系统的氛围，这些是单个智能体无法控制的（与现实生活非常相似）。但真正的焦点是智能体。因此，这些模型的开发速度更快，因为不需要确定由环境变化产生的精确结果。

随着智能体对环境变化做出反应以及与其他智能体交互，可能会出现模式，这些模式可以通过经验测试来确定其是否是一种稳定的行为属性。如果是这种情况，我们将看到确定性行为，这些确定性决策随后可以添加到智能体的规则制定概念中。因此，只要推理在科学和经验上得到证实，`ABM`就允许将确定性规则与随机性和概率相结合。

这并不是说应该将`EBM`连同洗澡水一起倒掉。在许多领域，`EBM`是至关重要的。这里提出的论点是，它不适合经济学。原因是智能体的异质性。当智能体是同质的，且总体足以代表一个群体时，`EBM`很有用。但经济学不属于这一类。例如，当我们观察股票市场时，会立刻被智能体、它们的决策以及市场变化速度的复杂性和动态性所震撼。将这些复杂的运动简化为基于代数的方程，是对经济的一种不具代表性的严重过度简化。

`ABM`通过聚焦于智能体来克服这一挑战。例如，如果我们使用`ABM`对股票市场进行建模，建模者会专注于理解个体层面的智能体行为。不同的智能体有不同的风险阈值，并会做出不同的决策。因此，`ABM`专注于指定智能体的属性，并通过创建关于它们如何相互交互的规则来利用它们的异质性。此外，在`ABM`中，智能体可以访问它们与其他智能体交互的历史记录。
