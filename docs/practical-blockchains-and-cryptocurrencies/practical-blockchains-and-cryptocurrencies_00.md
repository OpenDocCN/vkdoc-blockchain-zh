## 引言

让我们开门见山。在本书中，我们将用 Python 语言从头构建一个名为 Helium 的加密货币。你将学习区块链和加密货币应用中使用的关键算法，这将帮助你有信心地用任何语言开发自己的应用。

Helium 的通用语言是 Python。据我观察，许多对加密货币和区块链应用感兴趣的开发者都精通 Python 以外的语言。为了让这些开发者也能轻松读懂本书，我尽可能地降低了 Python 语言的入门门槛。例如，Helium 的源代码没有使用 Python 类。此外，本书还有八个附录用于补充各种预备知识。我没有刻意遵循 Python 的最佳实践，这是有意为之的设计。重点在于简洁。本书旨在让你掌握加密货币和区块链应用背后的关键算法。

如果你对 Python 完全陌生，可以下载我的著作《Python 火箭科学》（[`https://leanpub.com/python_rocket_science`](https://leanpub.com/python_rocket_science)）；这本书曾在训练营中使用，能在最短时间内让学生掌握 Python 语言技能。你只需阅读其中大约 70 页的内容，就足以学习《实用区块链与加密货币》这本书。

本书第一章概述了货币的历史，并将比特币置于这一历史背景中。市场上流通的加密货币远不止比特币一种。第二章讨论了加密货币生态系统中一些较为知名的币种。

在第三章至第五章中，我们将讨论区块链和加密货币应用所需的密码学知识。我们将深入探讨对称加密、加密哈希以及公钥密码学。通过这三个章节，你不需要任何微积分或数论知识。

第六章详细讨论了区块链的理论。我们将研究区块链的基本特征及其构建方式。

在第七章中，我们开始创建 Helium 加密货币。Helium 是模仿比特币设计的。本章指定了 Helium 的配置常量。第八章为 Helium 实现了区块链。按照本书结尾之前将反复出现的模式，我们将使用 `Pytest` 单元测试框架通过单元测试来验证我们的 Helium 源代码。本书附录三提供了有效利用 `Pytest` 所需的所有知识。

第九章是理论章节，讨论加密货币网络中的交易处理。本章详细讨论了类似比特币的加密货币网络中的交易数据结构及交易验证。下一章（第十章）讨论了默克尔树，这是一种特别有用的二叉树数据结构，可用于对区块内的交易进行排序和验证。在第十一章中，我们实现了 Helium 交易处理的所有源代码。

Helium 使用两个键值存储（数据库）：`chainstate` 键值存储和 `blk_index` 存储。这两个存储都是 `LevelDB` 数据库。本书附录二为你提供了必要的 `LevelDB` 专业知识。第十二章讨论了这些数据库在加密货币应用中的用途，并提供了它们在 Helium 中的 Python 实现。

挖矿是模仿比特币设计的加密货币网络的核心特征。挖矿是创建新货币单位并将交易添加到区块链的过程。第十三章对挖矿理论进行了广泛讨论。挖矿涉及将区块添加到区块链。比特币发明者中本聪的关键洞见是：可以在无需某个中央仲裁机构的情况下，就区块应添加到区块链的顺序达成共识。这就是分布式共识算法。该算法的一个重要特征是，交易的有效性不依赖于按时间或其他方式对交易进行排序和优先级划分。在本章中，我们将详细讨论像 Helium 这样的加密货币如何实现分布式共识。在第十四章中，我们实现了 Helium 挖矿的 Python 代码，其中包括分布式共识算法的实现。

加密货币应用通常运行在对等网络上。交易和区块从网络接收，节点也会向网络广播交易和区块。本书第十五章展示了如何为加密货币应用实现网络接口。在本章中，我们将实现让 Helium 在本地主机网络（`127.0.0.x`）上运行的 Python 代码。这对开发者非常有用，因为我们可以在单台机器上拥有一个功能完整的加密货币生态系统。将其推广到可路由的互联网地址是一个简单的练习。本章使用了并发挖矿操作。附录四和附录六包含了你理解 Helium 并发性所需的所有并发 Python 背景知识。
