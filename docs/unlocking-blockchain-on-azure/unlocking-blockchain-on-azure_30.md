# 去中心化 KYC 与信用数据验证

诸如政府等国家及国际治理机构，一直在尝试通过护照、生物特征收录、社会安全号码等方式实现身份标准化。然而，对于金融机构而言，由于个人情况的多样性，身份证明或开户流程通常需要提交非标准化文件。另一个现实情况是，在姓名变更或搬迁时，文件缺失会给金融机构和客户双方带来极大不便。

在此背景下，KYC 的相关方主要包括主要客户以及可能与之交易的金融机构等企业。对于任何金融交易——无论是银行账户交易、信用卡交易、存款还是共同基金——银行都需要了解客户。这种信任关系通过提交经授权的政府文件或在某些情况下通过推荐来建立。然而，监管金融经济安全的监管机构要求银行花费时间进行 KYC 合规检查，以防止洗钱。因此，背景调查、证书认证等工作日益增加。现在设想一下，这一检查过程发生在区块链网络上，而非线下流程。在线下流程中，客户数据必须以对最终客户不透明的方式在各方之间传递。这增加了机密数据泄露的风险，同时，此类文件可能以已知方式被伪造，使得追踪此类欺诈行为变得困难。在接下来的章节中，我们将探讨身份证明如何在区块链上进行转换，这有望增强贸易金融领域的 KYC，从而改善内部流程以便获取信用证和银行保函。
