# 降低欺诈风险与合规成本

设想一个特定区域内完全互联的贸易金融网络。所有贸易活动都被数字化记录在区块链上，每个已知实体都在链上。未经该链更新，任何链下活动都不得跨境。在这种理想情况下，交易流程可实现匿名追踪。然而，若此类网络中出现洗钱欺诈行为，重复的非法活动将被识别。同样，当任何贸易实体试图寻找漏洞绕过传统贸易方式时，链上记录也能识别此类异常案例。这使得监管机构更易于执行反洗钱法规，并避免经济失衡。

为了构建全球贸易互联网络，香港和新加坡的监管机构——香港金融管理局与新加坡金融管理局——联合二十家银行共同启动了跨境贸易金融活动的贸易流数字化进程。其他一些去中心化倡议，由不同银行组成联盟，包括`We.Trade`、`Voltron`、贸易信息网络、`Marco Polo`、`BankChain`等。所有努力都旨在实现数据无纸化、流程透明且安全。从技术角度看，其中多项倡议的试点阶段仍在为技术选型而挣扎——是选择`企业级闭源区块链平台`，还是`开源区块链平台`。若尝试采用`企业级平台`，托管该区块链的源代码能否被其他利益相关方盲目信任？该平台版本的`开源分叉`是否更能被公开接受？它是否支持互操作性，并鼓励更多利益相关方加入此类生态系统？当前的工作组是否准备好将现有系统与这种尖端技术集成？这些都是构建完全去中心化系统时尚待尝试、测试和探索的问题。

作为技术人员，我们可以为`智能合约`贡献逻辑应用，创建连接跨链网络的有效节点系统，并培训我们的工作组适应此类开放、去中心的平台。随着主要金融机构致力于拓展跨境关系与政策，针对这些用例开发的任何概念证明，若能真正解决痛点，必将获得关注。例如，可以开发一个欺诈检测工具，审查区块链网络上的匿名数据，并在发现无效交易上存在现金流搭便车行为时向链上发出警报。此类工具可被纳入`验证节点`中。同样，`智能合约`的自动检查功能可以核对进行贸易所需的所有文件的合规性。
