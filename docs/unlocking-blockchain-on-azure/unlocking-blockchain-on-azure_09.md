# 代币

纵观数百年历史，各国、各部落乃至整个人类，都一直在试图量化交易中的付出，无论是古老的物物交换体系，还是现代的金融交易时代。对时间、空间和经济体的正确量化，一直以来都起伏不定。

比特币采用了工作量证明机制，这有助于量化代币的价值。通过求解方程，在所有节点间完成一笔交易所需的算力，就转化为了比特币的价值。因此，通过这种数字货币进行交易，使得此类加密货币的交易成为可能。

> *加密货币是一种使用密码学保障安全的数字或虚拟货币。*
>
> ——Investopedia

若干区块链已成功通过其扩容、运行和以安全代币或功能代币形式产生影响的*能力*，量化了代币的价值。

安全代币通常指那些著名的首次代币发行（ICO）或加密货币，它们通过认可去中心化区块链平台理念和运营的利益相关者获得投资。

功能代币则纯粹因其用途而被估值。其用途通过区块链的运行机制（如前文所述通过工作量证明产生的价值）来量化。

通过 BBChain，代币的功能产生了安全价值。该链的用户使用 BBChain 上的功能代币进行交易，以便查看、订阅和购买其他用户提供的船舶数据。

> *现在的问题是，在现存世界各国不同的经济体系中，人们如何相信所谓链的真正价值？*

链的价值取决于其预先设计的本质，而这又基于共识机制及其协议。

让我们看看 BBChain 是如何做到的。

BBChain 协议融合了三种共识机制：即联邦拜占庭协议（FBA）、权益证明（POS）和重要性证明（POI）。公共账本由 FBA 治理。私有账本由 POS 监管。POI 则普遍适用于公有链和私有链。

假设公有链上有 `N` 个节点，分别为 `n1`、`n2`、`n3`、`n4`……`nN`。区块链上的活动是通过对公共数据和决策的仲裁交集进行验证来绘制的，从而运用 FBA 原则在公有链活动上形成共识。

在 BBChain 协议下，数据生产者通过 BBChain 创世节点在私有链中获得权益。此外，这些数据生产者（船东、港口所有者、制造商）可以将他们的权益分包给这个私有侧链上的其他用户。根据智能合约，权益会被重新分配。在交换权益时，每个节点和状态的重要性值会根据以下因素在私有链和公有链中重新计算：

1.  私有账本中节点在给定时间的权益价值
2.  所有私有账本中权益的比较/相对价值
3.  全年内的平均权益值
4.  成为权益所有者所需的归属金额阈值计算
5.  权益所有者发起侧链的阈值计算
6.  任何人加入侧链的阈值计算
7.  区块的价值和区块数量
8.  区块的龄期和区块组的大小
9.  区块的状态（数据吸引力和新鲜度）
10. 链中代币的总价值以及节点上的最低归属价值

## 数据流排序

由于在 BBChain 上流动的主要元素是海事数据，让我们按照以下顺序，基于共识算法和链上交易来理解数据流及影响因素：

1.  在对链上的交易、状态和数据进行优先级排序时，会根据前述因素计算重要性值，并为每个节点设定能够执行某些活动的计算阈值。
2.  这种计算主要在有限的节点以及硬件/网络带宽受限场景下出现高并发和数据拥塞时，优先用于数据呈现。
3.  其他操作，例如创建侧链、分发订阅数据以及生成内部（高度隐私数据）数据请求，要求节点必须跨越一定的重要性值才能加入共识（例如加入一个侧链）。
4.  数据过时（由区块的龄期因素决定，例如，超过 5 年，或大量数据块高度冗余且几无吸引力）的节点可以根据协议规则从链中移除，并可以选择进行集中化和存储。
5.  节点可以向公有链发起请求以访问这些数据，高重要性节点可以使用 FBA 机制执行共识来允许/验证该请求。
