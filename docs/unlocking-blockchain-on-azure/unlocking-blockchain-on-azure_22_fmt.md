## 用例：跨医院共享患者医疗健康记录

在医疗过程中，我们观察到患者在一段时间内并不总是由同一位医生或同一家医院治疗。这种情况可能由于患者迁移到不同地点、当前医院转诊至另一家医院、在其他城市遭遇不幸事件，或仅仅因为患者不喜欢医生的诊疗方式而发生。在这种情况下，将患者的完整病史与新医院/医生共享，对于保持医疗的连续性以及恰当理解患者的病情和过往治疗至关重要。

由于医疗记录是高度个人化和机密的文件，必须以安全、不可篡改且仅对正确用户可见的格式提供数据。这为探索使用区块链解决该问题的可行性打开了窗口。

进一步探索发现，基于 `CAUSE` 矩阵，使用区块链确实合适，因为它具有以下特点：

*   多方贡献者和协作者
*   健康记录属于资产
*   两种类型的贡献者：
    *   已知（认证专业人士、医生和医院）
    *   未知（最终用户、患者、客户）
*   健康记录的每一个状态都至关重要
*   为了实现完全匿名和隐私，此不可篡改的数据必须在链上实现端到端加密。

### GHOSSTTT 协议

一旦业务团队通过 `CAUSE` 矩阵定义了区块链平台的愿景并指定了区块链的类型，解决方案架构师就必须识别更深层次的设计组件，以桥接业务与技术。

因此，我们提出了 `GHOSSTTT` 协议，使解决方案架构师能够将区块链的设计接触点归纳如下：

*   **治理**是否必要？是单独、选择性授权、所有节点参与治理，还是基于智能合约规则的共识？
*   **高速**交易与并发？在较小的网络网格中使用私有账本的混合链。
*   **开放**/封闭、许可型还是非许可型？
*   大规模数据的**存储**状态——在分布式服务器上建立共识的定制解决方案，数据根据平台机制在私有链上复制存储。
*   **链上**存储是分布式并复制的吗？交易状态是否根据链上事件进行交互？
*   是否需要**代币**？现有的评分机制？奖励计划？不足之处？是否需要价值自动校准？
*   **交易**与已知参与者交互——分布式账本、`Ripple`、`Corda`
*   **可信**网络还是无需信任网络？

解决方案架构师在此的责任是识别一套正确的工具，使其与业务需求以及可用的开发者技能集保持一致。`GHOSSTTT` 协议将业务需求转化为技术决策。在 `CAUSE` 矩阵明确后，必须根据 `GHOSSTTT` 协议推导出具体细节。

让我们针对跨医院共享患者医疗健康记录的用例来考虑 `GHOSSTTT` 协议。

#### 治理

在这个日益数字化的时代，治理带来了此前在生态系统、企业和平台中可能未曾考虑过的新挑战。为了在数字平台上存储、引用和交易时覆盖公平实践，治理需要不断演进。随着区块链的发展，引入了去中心化自治组织（`DAO`）的监管系统。该系统使参与的利益相关方能够受到一套所有各方都同意的、公共预定义规则的约束。在基于 `DAO` 的系统中，不允许任何超出这些规则的行为。因此，在设计区块链时，必须考虑是否需要在智能合约中定义此类监管措施，以实现不受链上任何一方偏袒影响的自治功能。即使是发起或托管区块链的一方也不应享有偏袒。

在上述医疗用例中，健康器官的条件构成了治理的一部分，以维护流程的完整性，并且不偏袒提供服务的医疗机构。这是与仅由医疗机构托管的、可能被篡改的普通网站的主要区别所在。然而，基于 `DAO` 的区块链上的记录是无法被篡改的，从而保持了所需的治理效果。

#### 高速交易与并发处理

`Azure CosmosDB` 提供低延迟、高可用且全球分布的多模型数据库，以支持此类交易以及在联盟网络中运行并发交易。针对这一需求，也可选择类似的 `NoSQL` 数据库。

医疗记录在全球各地时刻都在生成，并会针对不同利益相关方从账本中被多次索引。因此，低延迟对于去中心化账本而言是一个关键因素。

#### 开放或封闭，许可或无许可

在医疗领域，大量信息（如医疗服务、政策等）会公开分发给利益相关方。然而，对医疗记录等机密信息的访问则需要进行许可控制。因此，该区块链可采用开放式许可账本。

例如，`Azure Marketplace` 提供的 `Quorum` 通过将公共数据和私有数据分离到不同的网络，来维护交易和合约的隐私性，从而在保持私有数据隐私的同时，确保公共数据的透明性。

#### 存储

在区块链平台的分布式账本中，包含两个方面：一是链节点状态的去中心化存储，二是数据本身的存储。在此阶段，需要决定应用程序是运行在内网、互联网，还是带有虚拟节点集群的共享磁盘上。

综合考虑医疗记录的共享需求以及前述对 `Quorum` 的选择——`Quorum` 保障的是交易及其状态的隐私，而非数据存储本身。因此，状态的存储是加密的，可存储在 Azure 的 `Blob`、`CosmosDB` 或任何其他存储中。但对于那些数据存储比交易本身更关键的应用程序，可以考虑使用 `BigChainDB` 这类扩展至分布式数据存储的工具。若需对文件进行完整加密，则可考虑 `IPFS`。当需要安全存储、交易/共享医疗记录时，可以设计将 `Quorum` 与 `IPFS` 集成到 Azure 云组件中，从而使医疗记录的共享真正安全，并使存储在全球范围内同样得到保障。

#### 代币、交易与信任

医疗记录作为关乎患者健康的高度敏感文件，其来源的可信度至关重要。因此，在治理第一阶段选定的 `DAO` 建立了无偏见网络的信任基础。通过将认证委员会的投票机制上链，可以使记录更具可信度。此处的交易可能包括共享记录、购买医学检验和报告，或根据链上规定购买通用的匿名公共数据。在此，解决方案架构师需要设计联盟链，它可以是国家、地区或区域内的公有链，也可以是医生与患者的私有链，或企业与员工医疗记录的私有链，亦或是保险公司与客户的私有网络。交易网络的连接方式需在此阶段决定。一旦设计完成，便可基于业务需求量化链上操作。因此，首先要判断您的业务用例是否需要代币化；例如，医院/外科医生/病理学家的信用评级。客户对健康保险公司的信用评级等，可以通过链上约定的代币经济学在链上给予激励。然而，如果仅仅是维护账本上的数字保险库，则无需代币化。
