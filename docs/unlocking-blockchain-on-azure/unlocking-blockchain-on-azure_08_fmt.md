# 加密

当我们登录银行门户网站或社保系统时，能感受到安全感和信任感；而在照片分享和社交媒体等开放平台上，数据的开放程度远高于个人的银行账户详情。

![小知识](img/474923_1_En_2_Figf_HTML.jpg)

请找出你的社交媒体账户与数字银行账户在感知安全性方面的三大主要区别。

那么，这与区块链中我们称之为透明、共享且安全的特性有何关联？如同存在多种用于社交媒体的在线应用和金融交易应用一样，区块链也有多种服务于不同目的的平台。在区块链中植入信任感的基本概念，是数据的端到端加密，使得不可篡改的数据只有在参与节点达成共识时才能被解密。在此，我们使用沙米尔秘密共享算法进行分布式拆分密钥管理。不同的区块链平台可以根据数据的关键性采用不同的密钥管理机制。在分布式账本上使用拆分密钥管理，使得黑客难以从外部攻破，除非链上利益相关者之间发生 51%挖矿攻击（即多数节点/用户为欺诈节点）。

最著名且最古老的区块链加密货币——比特币，是一种被称为电子现金的数字货币，其网络节点间的交易通过 `SHA256` 进行加密。每个区块都包含前一个区块的 `SHA256` 加密哈希值，从而将其链接到前一个区块，区块链也因此得名。为了被网络中的其他节点接受，新区块必须包含工作量证明。矿工们依靠对大量输入计算“SHA256 哈希函数”，直到找到给定区块的临时数（nonce），然后才能将其添加到区块链中。

在图 2-4 中，我们看到，对账本中的数据进行加密和分片处理，使得黑客或外部用户难以检索到任何相关信息。

让我们回到社交媒体的话题，在那里，我们通过分享帖子、照片和个人观点，从朋友和家人的社交圈中获得关注。当这些数据被直接上传到服务器时，网络和服务器容易受到攻击，或者被其他已知或未知的未授权用户访问。这有时可能导致数据泄露，落入不法之徒手中。

现在，想象一下将区块链用于社交媒体的场景。其功能是相同的，但数据的存储和管理方式却不同。在区块链上，用户的节点对数据进行加密并上传到账本。根据该链的共识机制，数据被代币化并分配给用户希望分享的利益相关者。任何节点或用户若要读取这些数据，必须解密存储于多个节点中的整个分布式数据序列，这使得数据高度安全，且破解需要更多的计算量。

分布式账本中使用了多种加密技术。可以根据应用和区块链类型来决定采用哪种技术。

`BBChain` 使用 `ED25519` 进行端到端加密，特别是在共享更机密数据的私有账本侧链中。在像 `BBChain` 这样的混合网络中，用户同时存在于私有账本和公共账本上，密钥共享机制被用于加密和解密在区块链上共享的数据。

多种密码学机制，例如 SHA 2 系列、沙米尔秘密共享算法以及 `EdDSA` 技术，被广泛应用于区块链中。想象一下，一个熟悉设置密码的用户——我们如何让他们能够在区块链上安全地加密自己的数据？一种方法是在区块链上使用预定义的加密规则集，或者赋予他们选择自己加密技术集的能力。用技术术语来说，区块链上的用户可能是一个发送数据（以添加区块）的节点/服务器。因此，加密是在这个数据源服务器上通过其私钥设置的。链上用户可以使用公钥检索数据，并进一步使用私钥解密数据。此处提到的机制提供了这样的加密选项组合。
