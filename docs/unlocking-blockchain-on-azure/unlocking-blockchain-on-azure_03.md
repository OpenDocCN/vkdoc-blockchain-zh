# 区块链术语

区块链的技术定义：

- 区块链可定义为一种仅可追加的区块列表，这些区块以去中心化存储形式分布于点对点网络中，并通过密码学进行加密，且遵循共同协议原则——由网络中的多数节点验证链上的交易。

以下是对专业术语的通俗解释：

| 专业术语 | 通俗说法 |
| --- | --- |
| 仅可追加 | 记录每一次状态变更。一旦添加，不可删除。 |
| 区块 | 交易信息的记录，这些信息被加密为哈希值并编码成默克尔树。 |
| 哈希 | 一种函数，将字母和数字组成的输入转换为固定长度的加密输出。 |
| 加密 | 将（信息或数据）转换为代码，尤其用于防止未经授权的访问。 |
| 密码学 | 研究在第三方存在的情况下实现安全通信的技术学科。 |
| 共识 | 全体或多数达成的一致意见形式。 |
| 分布式 | 不存储在单一或中心化来源中，从而降低被黑客攻击的风险。 |
| 信任 | 一种通过计算方式建立的可靠性方法。 |
| 默克尔树 | 一种数据结构形式，确保区块能够高效且安全地被验证。 |

以下是实际运作流程：

1.  一条数据被分解为经过哈希处理的区块。
2.  多个设备（节点）以链式连接形成网络。
3.  该网络形成一套通用的共识规则。
4.  共识有多种类型，例如基于已完成的工作、涉及的归属权，或共同商定的各种投票机制。
5.  数据区块被传输到该网络，当网络根据规则达成多数共识批准后，所有设备的信息都会同步更新。
6.  一旦传输完成，除非所有区块都随新变更（即一个新增的数据区块）更新，否则任何一方都无法单独篡改该区块。
7.  现有区块在任何情况下都不会改变。
8.  在此传输、验证和存储过程中消耗的能量，就是交易的真实价值成本。

上述说明会根据区块链的愿景及其服务用户的目的而有所不同。例如，如果区块链用于验证身份，那么共识方法以及对数据的处理和操作流程都会随之变化。

然而，去中心化的基础保持不变。其他变量的比例则根据节点配置而变化。在接下来的章节中，您将看到因共识、存储、密码学类型等参数不同而产生的各种区块链变体。
