## 哈希函数

理论上，**哈希** 是识别数据的行为。它是一种特殊的方法，可以为任何数据（无论是句子、照片、电子表格还是下载的程序）分配一个唯一的、随机的值。你可以将哈希函数视为“识别机器”——一种为特定输入分配值的机器。输入是任意的——可以是图像、文档、文件、原始字节、数字，任何你喜欢的东西——但对于**相同的输入，其输出总是可预测的**。

### 命名约定

人们倾向于将术语 **哈希** 同时用作动词和名词：“看看这张照片的哈希值”（名词），或者“让我们对以下文件进行哈希”（动词）。哈希函数输出的正确术语是“摘要”，但“哈希”已经成为普遍用法。

### 唯一性

前面第二句话声称哈希值是唯一且随机的——这稍微有些不准确。在本书中，我们处理的是特殊类型的哈希函数，称为密码学哈希函数，其中计算出的哈希值是从一个足够大的值池中选择的，以至于在已知宇宙中，两个输入共享一个值的可能性极低。所以，修改一下上面的说法：哈希值实际上是*在可实际达到的范围内最接近唯一的*。

### 示例 1：Python 中的哈希运算

Python 内置了常用的标准哈希函数集，这些函数位于 `hashlib` 库中。我们启动 Python 并开始使用它们。

```python
import hashlib
# 哈希函数期望输入为字节：encode() 方法可将字符串转换为字节
input_bytes = b"backpack"
output = hashlib.sha256(input_bytes)
# 我们使用 hexdigest() 将字节转换为十六进制，因为更易读
print(output.hexdigest())
```

你得到的输出是什么？应该是如下所示：

```
5f00368a6ad231c3c439c4f6bc33c27014b4d35a904ff1656d74f9528636f496
```

现在，尝试将 `"backpack"` 中的某个字母改为大写。尝试在末尾添加一个空格。每次调整输入后都进行实验。你是否注意到生成的哈希值*完全*不同？很好。那么你刚刚发现了*雪崩*特性：输入的微小变化会导致输出的巨大差异。

通常，如果一个哈希函数满足以下特性，则被认为是**加密**哈希函数：

-   **确定性：** 相同的输入始终产生相同的哈希值。
-   **难解性：** 除了通过穷举（尝试海量可能的输入）之外，无法根据给定的哈希值找到对应的输入。
-   **碰撞安全性：** 无法找到两个产生相同哈希值的不同输入。
-   **雪崩效应：** 输入的微小变化应产生一个与旧哈希值看似毫无关联的全新哈希值。
-   **速度：** 生成哈希值的计算速度*很快*。

## 哈希函数的选择

点对点区块链会在其协议中声明所选的哈希函数：比特币使用*双重* `sha256`，而以太坊使用 `keccak256`。需要了解的重要一点是，所有这些哈希函数都做同一件事：对给定输入提供可预测的输出。

让我们回顾一下目前学到的知识。

| 术语 | 解释 | 示例 |
| --- | --- | --- |
| 哈希函数 | 一种通过输出一个巨大的、随机的十六进制数来标识你输入的任何内容的函数。对于相同的输入，该数字*始终*相同。 | 一种加密哈希函数，例如 `sha256`、`md5`、`blake2b` 等。 |
| 哈希值/摘要 | 哈希函数的输出：一个巨大的、随机的十六进制数。 | 单词 `"dan"` 的哈希值：`ec4f...f1cb` |

你能发现修改之处吗？可能能，也可能不能。仔细看看。（我把单词 `"STATES"` 改成了单数。）但有趣的是：如果我对这张图片运行哈希函数，结果会是 `f4c56f530133b8de6b3b0b39a610be32`。这看起来可能并不惊人，但如果比较这两个哈希值，你会发现它们截然不同且看似随机。我将它们并列展示：

```
e25641dc52387baba19751783ae4e060
f4c56f530133b8de6b3b0b39a610be32
```

> **注：** 比特币背后的突破性理念是，哈希值可以用来证明计算机完成过工作量——我们将在后续章节中探讨这一点，但就目前而言，请记住比特币利用了加密哈希值的明显“随机性”。

让我们尝试用 Python 为我们自己的图片（或文件）计算哈希值。

```python
from hashlib import sha256
file = open("my_image.jpg", "rb")
hash = sha256(file.read()).hexdigest()
file.close()
print(f"The hash of my file is: {hash}")
```

让我们逐行审视并解释上述代码：

```
from hashlib import sha256
```

这一行从 Python 自带的 `hashlib` 库中导入了 `sha256` 哈希函数。`sha256` 是最流行的哈希函数之一（在比特币中被广泛使用）。

这里，我们打开一个名为 `my_image.jpg` 的文件，该文件位于代码所在的同一目录：

```
file = open("my_image.jpg", "rb")
```

参数 `"rb"` 表示该文件应以只读模式打开，并以字节形式读取。

```
hash = sha256(file.read()).hexdigest()
```

然后我们将打开的文件读入哈希函数，如：`sha256(file.read())`，并将 `hexdigest()`（即以十六进制字符串形式输出的结果）赋值给一个名为 `hash` 的变量。

最后，我们关闭打开的文件并将哈希值打印到终端。

### 类比

如果你仍然感到困惑，这里有一个现实生活中的类比，可以帮助你更深入地理解这个想法。

想象一下你在机场通过安检，将背包放在传送带上。但传送带配备了一种特殊的扫描仪——一种能够对您的物品进行 X 光扫描并输出哈希值的机器。

背包被扫描后，扫描仪会吐出一个标识您背包的 8 位数字哈希值：`13371817`。如果你的背包及其所有物品完全相同，扫描仪将始终输出相同的数字。但如果你决定从背包中取出一个物品，扫描仪会返回一个完全不同的数字。

## 不可逆性

正如我们讨论过的，区块链使用了**加密**哈希函数，这些哈希函数极难逆转。这意味着，如果我们拿到一个哈希值，想要猜出原始输入是什么是极其困难的（达到超级计算机级别）。

在前面的例子中，如果我给你哈希值 `e25641dc52387baba19751783ae4e060`，你几乎不可能猜出它是一张 100 美元钞票的照片。

理解这个概念至关重要，因为它构成了比特币乃至整个区块链安全性的基础。正如我们稍后将看到的，还有许多具有不同特性的不同哈希函数。

### 示例 3：发送防篡改的电子邮件

## 问题

爱丽丝想通过开放互联网等不安全渠道向鲍勃发送一封电子邮件。鲍勃并不在意其他人能否阅读邮件内容，但他希望确保邮件未被篡改。鲍勃如何验证邮件信息未被篡改？

> **注：** 在我们给出解决方案之前，运用你刚刚掌握的哈希知识，你能想到如何设计这样的方案吗？

### 解决方案

整体思路是让爱丽丝和鲍勃事先约定一个共享密钥。然后爱丽丝从消息中删除密钥短语，将消息和哈希值一起发送给鲍勃。鲍勃随后执行与爱丽丝相同的操作：他附加预共享的密钥短语并输出一个哈希值。如果该哈希值与爱丽丝发送的不一致，他就知道消息已被篡改。

总结如下：

1.  爱丽丝和鲍勃共享一个密钥短语 `S`。
2.  爱丽丝将密钥附加到消息 `M` 的末尾，然后创建消息 `M` 的哈希值 `H`：`H = hash(M + S)`。
3.  爱丽丝将 `H` 和 `M`（即消息和计算出的哈希值）发送给鲍勃。
4.  鲍勃自己计算 `H` 来检查消息的完整性，看其是否与爱丽丝公布的哈希值相同。

**注意**
这个通用方案描述了一系列称为 HMAC（哈希消息认证码）的协议，该协议由 IETF RFC 2104 定义。

让我们看一个例子。在爱丽丝和鲍勃开始通信之前，他们都同意使用一个秘密密码 `bolognese`。爱丽丝使用给定的方案编写她的电子邮件。以下是鲍勃收到它时的样子：

```
发件人: alice@example.com
主题: 你听说过比特币吗？
嘿鲍勃，我认为你应该了解一下区块链！我一直在投资比特币，目前我的账户里正好有 12.03 BTC。
哈希值: 71890dc61c21370874d2a7b74064396cb613a1924f09aa06925abc7842e6802c
```

鲍勃注意到爱丽丝在邮件正文中包含了哈希值。他将哈希值剥离，然后将密钥短语 (`bolognese`) 附加到邮件正文的末尾，并计算其哈希值。如果该哈希值与邮件正文中包含的哈希值匹配，那么他就可以断定消息没有被篡改（并且爱丽丝确实拥有 12.03 BTC）。

爱丽丝和鲍勃都必须知道他们将用于验证的协议，这非常重要。这包括要使用的哈希函数 (`sha256`)、密钥短语在消息中的放置位置（他们同意将其附加到末尾），以及任何可能进一步增强方案安全性的其他数据（例如时间戳）。

让我们看看鲍勃如何用 Python 验证爱丽丝的消息。

```python
from hashlib import sha256
secret_phrase = "bolognese"
def get_hash_with_secret_phrase(input_data, secret_phrase):
    combined = input_data + secret_phrase
    return sha256(combined.encode()).hexdigest()
email_body = "嘿鲍勃，我认为你应该了解一下区块链！ " \
"我一直在投资比特币，目前我的账户里正好有 12.03 BTC。"
print(get_hash_with_secret_phrase(email_body, secret_phrase))
```

当鲍勃运行这段代码时，他就能验证爱丽丝的哈希值。

## 防止垃圾邮件如何促成了工作量证明

你知道哈希函数可以用来防止电子邮件垃圾邮件吗？这个想法背后的算法被称为“工作量证明”算法，它是比特币的核心。它由英国密码学家亚当·巴克于 1997 年发明，他称之为 Hashcash。

其总体思路是只接受哈希值满足特定约束的电子邮件。这会导致电子邮件发送者在发送邮件之前执行某种计算工作。换句话说，发送者想要向人们发送垃圾邮件的成本变得很高。

但是约束是什么呢？哈希值只是些数字，但它们看起来是随机的，所以我们可以应用任何我们想要的约束。我可以规定，任何给我发送电子邮件的人都必须提供该电子邮件的哈希值，并且该哈希值是一个奇数。但这并不是一个好的约束，因为发送者生成一个奇数哈希值的概率有 50%。

Hashcash 算法应用的约束是：哈希值必须小于某个特定数字。这听起来简单得令人难以置信，但请思考一下：既然加密哈希函数对于给定的输入会输出一个随机数，我们如何确保邮件正文的哈希值是一个特定的数字呢？我们做不到。我们唯一能做的就是向邮件正文中插入任意数据，并不断尝试，直到我们的哈希值满足约束条件为止。

让我们看一个例子，同时设定约束条件为：邮件正文的哈希值必须小于 10,000。

**注意**
为简单起见，我这里使用一个只输出十进制数的八位哈希函数。

```
发件人: dan@example.com
主题: 购买比特币！
嘿伊森，我认为你应该买一些比特币！
哈希值: 95119035
```

**不行。** 这封邮件正文的哈希值是 `95119035`，它不小于 `10,000`。因此，发送者没有完成所需的工作：他必须确保邮件正文的哈希值满足约束条件。

发送者可以这样做的一种方法是在邮件正文中插入任意数据：

```
发件人: dan@example.com
主题: 购买比特币！
嘿伊森，我认为你应该买一些比特币！
s8763ASdh727212213098
哈希值: 00000891
```

现在我们看到正文的哈希值是 `891`，满足我们的约束条件！（我用 0 填充了哈希值，因为十六进制数字通常这样显示）。我们的邮件服务器会对邮件正文进行哈希运算，看到它确实是 `891`，并接受这封邮件。

这就是工作量证明：通过展示一个特定的哈希值来证明工作已经完成，而服务器可以轻松验证。另一种理解这个方案的方式是，它生成困难但验证容易。

**注意**
正是这一关键思想，使得任何工作量证明区块链能够通过向网络中的每个人明确证明完成了计算工作来生成（或挖出）货币。通过利用加密哈希函数看似随机的特性，消耗了能量！
