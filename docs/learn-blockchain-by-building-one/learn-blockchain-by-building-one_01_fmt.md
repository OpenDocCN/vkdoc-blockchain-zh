# 引言

## 又一本关于区块链的书？为什么？

理解区块链并不容易。至少对我来说是这样：当比特币首次登上新闻头条时，我试图了解它的工作原理，却发现面向程序员（像我这样的人）的资源少之又少。虽然有比特币参考维基百科（[`https://en.bitcoin.it`](https://en.bitcoin.it)），但在当时，它的组织方式并不像今天这样清晰；尽管我读了中本聪的白皮书，但起初并没有真正理解它——至少不理解加密部分是如何运作的。

我在 YouTube 上漫无目的地浏览，完成了许多不完整的教程，并因那些未能清晰传达概念的示例而感到沮丧。于是，我决定尝试自己构建一个区块链，并记录下在这个过程中学到的所有东西。通过这样做，我发现了为什么加密货币如此难以解释和理解；因为首先你需要定义数字货币的构成要素：

-   货币是如何创造的？（挖矿）
-   爱丽丝如何向鲍勃发送货币？（数字签名交易）
-   谁负责追踪所有这些交易和生成的货币？（每个人，通过分布式账本）

这些高层次的概念依赖于不同的知识单元，必须首先理解它们，然后才能将它们组合成一套大家共同遵守的通用规则。而理解这些不同概念的最佳方式是逐一击破——通过实际动手构建自己的加密货币来实践。因此，我写这本书是为了那些和我有同样挫败感的人，并通过在代码层面处理主题来克服它——这才是真正让知识扎根的方法。如果你坚持读下去，并动手实践，我相信在本书结束时，你将扎实掌握它们的运作原理。

## 为成功做好准备

### GitHub 仓库

最终代码位于 [`https://github.com/dvf/blockchain-book`](https://github.com/dvf/blockchain-book)。但请尝试自己动手编码——代码结构设计为高层方法已预留接口，细节逐步填充。此代码会保持更新，因此它可作为参考的北极星。

### 花时间配置你的开发环境

使用一个好的集成开发环境（`IDE`），例如 Microsoft VSCode 或 JetBrains PyCharm。它们都是免费的，并且能在你之前发现代码中的错误。在开始之前花时间设置好 `IDE` 是非常值得的。把你的时间花在操心区块链上，而不是代码的语法错误上。

### 知道去哪里获取答案

浏览并在 `GitHub` 仓库的 *Issues*（问题）页面提问。该仓库拥有庞大的社区关注者，因此你很可能会遇到其他有类似问题的人。如果你遇到错误或漏洞，恳请你提交一个 *Issue*。

### 不会 `Python`？

没关系。`Python` 以其可读性著称；它是一种非常易于转译的语言。我曾见过其他程序员（`C#`、`JavaScript` 和 `Rust`）实时完成书中的示例。
