# 第二章：区块链

有些区块链主要关注的并非维护和保障涉及各自加密货币的交易记录安全。然而，每条区块链都会记录任何涉及其相关加密货币或代币的交易。可以将交易视为对数据的简单记录。对数据含义的解读，是通过首先赋予其价值来完成的，正如在金融交易过程中所做的那样。

## **2.3.2 链**

链可以被看作是区块的链表。位于第一个区块（也称为创世区块）之后的区块会维护一个对前一个区块的引用，以此类推。这种引用是通过生成前一个区块内容的哈希值来维护的。这对链的不可篡改性有重大意义。任何想要篡改前一个区块的人，都需要修改该特定区块之后的所有区块。通常这在实践中是不可能的，因为对于像工作量证明这样的算法，它可能需要巨大的计算资源；而对于像以太坊这样基于权益证明的算法，则需要在网络中投入巨大的权益。

## **2.3.3 网络**

“全节点”是网络的构建模块。可以将它们想象成一台正在积极执行算法以确保网络安全的计算机。网络中的所有节点都拥有相同的记录副本（称为账本）。账本包含了所有曾在区块链上记录过的交易。

网络节点是去中心化的，可以由世界上任何地方的任何人运行。由于运行这些节点会产生成本，人们需要一些激励才会去运行。他们运行节点的动机是为了赚取加密货币，这作为他们的激励。区块链的底层算法会为他们的服务提供补偿。通常，会以比特币或以太币等代币或加密货币作为奖励。

## **2.4 区块链应用于何处？**

基于区块链的应用是围绕网络应作为仲裁者这一原则设计的。这种系统为其用户创造了一种冷酷无情的氛围。计算机代码实际上成为了法律，规则按照其编写方式执行，并由网络进行解释。计算机中不存在人类特有的社会行为和偏见。

网络无法解释用户的意图（至少目前还不能）。作为这一概念的一个潜在用例，在区块链上进行仲裁的保险合同已经受到了广泛关注和研究。保持记录不可篡改的能力是区块链技术的另一个迷人应用。

你可以利用它来创建一个清晰的时间线，记录谁在什么时候做了什么。无数行业和监管机构花费大量时间试图解决这个问题的规模。一个不可篡改的账本将使我们能够从区块链创建之初就保持交易的一致时间线。

尽管人们经常互换使用“比特币”和“区块链”这两个术语，但这两个概念并不相同。比特币使用了区块链。允许比特币安全转移的底层协议被称为区块链。比特币是驱动比特币网络的数字货币的名称。

因此，比特币是构建在区块链基础设施之上的应用之一。

## **2.5 演化**

区块链技术最初是与加密货币比特币一同开发的。比特币的整个理念是开发一个去中心化的货币和银行系统。比特币网络构建的初始目的是确保其加密货币的安全。它拥有大约 5000 个全节点，分布在世界各地。其主要目的是买卖比特币和其他加密货币，但社区很快意识到它可以用于更多方面。由于其规模以及其安全性已随时间得到验证，它也被用于保护其他规模较小的区块链和区块链应用。

随着以太坊网络的创建，区块链技术经历了第二次迭代。它通过在其结构中嵌入一种编程语言，改变了区块链的传统结构。与比特币类似，它也拥有超过 5000 个全节点，并分布在世界各地。以太坊的主要用途包括以太币交易、智能合约开发以及去中心化自治组织（DAO）的形成。此外，它也被用于保护区块链应用以及更小型的区块链。

以太坊生态系统将区块链生态系统转变为了一个可编程的区块链。

## **2.6 共识**

共识是分布式系统设置中最难解决的问题之一。

区块链是有用的工具，因为它们能够自我纠正，并且无需第三方干预即可实现。

通过使用其共识算法，它们能够成功地执行规则。

在区块链技术领域，“达成共识”指的是在一组通常互不信任的节点之间达成一致的过程。这些是网络上功能完备的节点。全节点正在检查提交到网络以便记录在账本上的交易的有效性。每条区块链都有自己的一套算法，用于让其网络对正在添加的新条目达成共识。由于每条区块链都在生成不同类型的条目，因此存在多种多样的模型来实现共识。

不同区块链用于达成共识的一些机制如下：

- **工作量证明** – 由比特币区块链使用
- **权益证明** – 由以太坊使用
- **历史证明** – 由 Solana 使用

### **2.6.1 工作量证明**

第一种采用共识机制的去中心化加密货币是比特币，该机制就是工作量证明。挖矿和工作量证明是密切相关的概念。“工作量证明”这个术语源于网络需要极高水平的处理能力才能正常运行。工作量证明区块链是指来自世界各地的虚拟矿工相互竞争，看谁能先解出数学谜题。这些区块链随后得到保护和验证。胜利者会从网络中获得预定数量的加密货币，并有机会用最新已验证的交易来更新区块链。
