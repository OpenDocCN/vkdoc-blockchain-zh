# Ethereum 地址与 Solidity 中的地址使用

## Ethereum 地址概述

以太坊和 Solidity 中的标准地址长度为二十字节（160 位或 40 个十六进制字符）。它等价于对公钥执行`Keccak-256`哈希后得到的最后 20 个字节。由于地址总是以十六进制格式（基数为 16 的表示法）表示，因此其开头总是添加前缀`0x`（已显式定义）。

创建以太坊地址的步骤如下：

1.  首先，我们使用`ecparam`命令（由 OpenSSL 提供）来生成椭圆曲线私钥。以太坊的标准实现采用`secp256k1`曲线。此命令将以 PEM 格式（利用出色的 ASN.1 密钥结构）输出私钥，并将其发送到标准输出。
2.  使用`openssl`命令从私钥中获取公钥。
3.  对公钥应用`Keccak-256`哈希算法。最终应得到一个总长度为 64 个字符（即 32 字节）的字符串。
4.  取最低有效字节，即所生成哈希的最后 40 个字符（即 20 字节）。（或者，换句话说，去掉前 24 个字符和前 12 个字节）。地址由这 40 个字符和 20 个字节组成。

当添加`0x`前缀后，地址的实际长度增加到 42 个字符。此外，务必记住字母的大小写无关紧要。所有钱包都应能接受以大写或小写字母书写的以太坊地址，且其解释方式不应有任何差异。

合约地址的构建方式有所不同。它们是基于以下两项内容以确定性的方式生成的：
1.  创建者（发送者）的地址
2.  创建者已经发起的交易数量（`nonce`）

合约地址的形成过程如下：
1.  考虑`sender`和`nonce`的值。
2.  使用 RLP 对它们进行编码。
3.  使用`Keccak-256`对它们进行哈希。

## Solidity 中地址的使用

要创建地址数据类型的变量，必须在变量名前加上关键字`address`。

```
address sender = msg.sender;
```

这里，我们在`sender`变量中保存发送者的地址。`msg.sender`是 Solidity 的内置函数，它表示调用智能合约上函数的用户的地址。

Solidity 中还有另一种类型，称为`address payable`。

在 Solidity 0.5.0 版本中，首次引入了`address`和`address payable`之间的区别。其目的是为了区分能够接收资金的地址和不能接收资金的地址（用于其他目的）。换句话说，普通的`address`无法接收以太币，而`address payable`则允许这样做。简单来说，只能将以太币发送给定义为`address payable`的变量。

我们将与地址相关联的 Solidity 方法分为两个不同的组（即交易类型）：
1.  有三种与以太币相关的方法：`.balance()`、`.transfer()`和`.send()`。
2.  以下是与合约交互相关的技术：`.call()`、`.delegatecall()`和`.staticcall()`。

## `balance` 方法

此方法以`wei`为单位返回账户余额（1 ether = 10¹⁸ wei = 1,000,000,000,000,000,000 wei）。该技术可用于任何声明为`address`的变量。`address.balance()`使得访问外部拥有账户（EOA/用户）以及合约账户持有的以太币数量成为可能。返回的数字是以`wei`为单位的以太币数量。因为它是读取存储在区块链上的数据，类似`address.balance`这样的语句被认为是一种“从状态中读取”的操作。因此，任何返回`address.balance`的 Solidity 函数都可以声明为`view`。
