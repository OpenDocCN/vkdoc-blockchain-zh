# 第 6 章 以太坊架构与概览

公开主网是以太坊的生产区块链，以太币具有价值，并可在交易所进行交易。在本节中，我们提及了与以太坊区块链交互所需的最简可行工具集，包括钱包、浏览器、节点客户端、智能合约 IDE，以及 Web3。

#### MetaMask：与以太坊区块链交互的最简方式

要获得以太坊区块链的实操经验，最简单的步骤之一就是使用钱包创建一个账户，并进行一些简单的加密货币发送和接收交易。在以太坊中，这可以通过使用 `MetaMask` 实现。`MetaMask` 是一个 Web 扩展程序，可以安装到网页浏览器中，用于创建账户并与以太坊交互，具体步骤如下：

*步骤 1：安装 `MetaMask` 扩展程序并创建加密账户*

前往 [`metamask.io`](https://metamask.io) 并将 `MetaMask` 安装到您的网络浏览器中，例如 Chrome 或 Firefox。

安装完成后，打开 `MetaMask` 并记下钱包的助记词。该助记词由 12 个英文单词组成，用于代表钱包的私钥。此助记词必须安全存储，因为如果 `MetaMask` 被意外卸载，或者计算机系统停止工作需要重新安装 `MetaMask` 时，需要使用它来恢复钱包。一旦您使用 `MetaMask` 创建了一个加密账户，就会获得一个该账户的公共地址。此地址也称为账户地址，可用于接收发送给该账户的加密货币。

*步骤 2：为 `MetaMask` 账户充值*

根据您连接的以太坊区块链类型，为钱包账户充值的方式可能有所不同。

-   对于由开发者创建或通过 `ganache_cli` 创建的开发用区块链，会默认创建并充值一些账户。启动该开发区块链的人，其拥有的账户中会自动铸造一些以太币。开发者可以将这些已充值的账户导入到他们的 `MetaMask` 钱包中。

-   对于诸如 `Rinkeby`、`Kovan`、`Ropsten` 和 `Goerli` 等测试网，可以通过水龙头获取测试代币。开发者可以前往水龙头网址，粘贴一条包含钱包账户公共地址的社交媒体消息，并将其发送到水龙头网络服务器。之后，水龙头会向消息中指定的地址发送少量测试网以太币。

-   对于以太坊主网，以太币需要通过购买获得。这可以通过在 `Coinbase` 等服务商处开设交易所账户，并使用法定货币支付来换取以太币。

*步骤 3：发送和接收代币*

一旦加密账户创建并充值完成，用户就可以开始发送和接收代币了。

要发送代币，只需点击“发送”，填写 `Nonce`（交易序号）、接收金额和接收方地址，并在核实信息后点击“发送”。

要接收代币，只需将账户地址提供给发送方，让发送方向该地址发送加密货币即可。

#### Etherscan：最全面的区块链浏览器

当交易被传输到以太坊并执行时，它们会引起以太坊在账户余额、区块、交易、智能合约和事件日志等方面的状态变化。以太坊区块链仅在其存储中记录原始数据。为了浏览区块链并理解其内容，部分数据需要经过处理并存储在数据库中，以便于查询和可视化展示。`Etherscan` 是一个提供卓越以太坊区块链浏览体验的网络服务。要使用 `Etherscan`，只需打开任意网络浏览器，并将网址指向 `http://etherscan.io`。

您可以使用 `Etherscan` 浏览器浏览以太坊区块链上的每一个账户、智能合约、区块和交易。对于需要通过编程方式使用区块链数据的开发者，`Etherscan` 还提供了 API 函数和 API 密钥，以便将动态的以太坊区块链数据集成到您的去中心化应用中。



与其他以太坊区块链浏览器一样，仅适用于以太坊公共区块链，例如主网（mainnet）、Rinkeby、Kovan、Ropsten 和 Goerli；它不适用于开发用区块链。

**Geth：以太坊区块链的瑞士军刀**

`Geth` 代表 Go Ethereum，是一个独立的以太坊客户端。它可以在操作系统上作为挖矿节点运行以赚取以太币，或作为 RPC 节点为客户端应用程序提供 RPC 服务。我们建议开发者下载一个版本的 `geth`，并按照以下步骤将其用作沙盒来与以太坊区块链交互。

`Geth` 客户端受主流操作系统支持，如 Linux、Windows 和 MacOS。由于它是开源的，建议开发者在 Linux 系统（如 Ubuntu）中运行它。

要从源代码编译 `geth`，需要安装 Go 编程语言编译器。开发者可以使用 Golang 编译器进行 `geth` 编译。

*任务 2：运行 Geth 应用程序*

`geth` 应用程序可以使用各种选项运行。完整说明位于 [`geth.ethereum.org/docs/`](https://geth.ethereum.org/docs/)。你可以使用 `--miner` 选项运行它来进行挖矿，但此方式不推荐，因为以太坊的挖矿难度级别很高，在工作量证明（POW）区块链中几乎不可能与大型矿池竞争。

`Geth` 可以作为沙盒运行，用于以下功能：
- 使用 `geth` 构建一个开发用区块链。
- 使用 `geth` 作为完整的区块链客户端，下载整个以太坊测试网或主网区块链。
- 使用 `geth` 作为 RPC 服务，查询区块链信息并将交易广播到所有区块链节点。
- 使用 `geth` 作为 JavaScript 控制台与区块链交互。

*任务 3：与 Geth RPC 服务交互*

一旦 `geth` 节点设置完成并运行了 RPC 服务，用户就可以使用钱包、应用程序或 JavaScript 控制台连接到 RPC 端点。

**A. 通过 MetaMask 自定义 RPC 连接**

要从 MetaMask 连接到 RPC 服务，只需打开 MetaMask，点击打开网络列表，然后选择“自定义 RPC”以打开一个新的网络小部件。指定一个网络名称，输入 RPC 端点和网络的 `chainId`。`chainId` 是一个对所有区块链都唯一的标识符，由以太坊标准 `EIP-155` 指定。如果你正在构建一个开发网络，你可以在创世文件中指定一个 `chainId`。你可以选择指定区块链的货币符号以及区块浏览器的 URL。点击“保存”以挂载新的 RPC 节点。

一旦 MetaMask 挂载到之前指定的 RPC 节点，它将与节点所连接的区块链进行交互。你可以使用 MetaMask 通过 RPC 节点为开发用区块链发送和接收交易。

**B. 通过 JavaScript 控制台**

`Geth` 不仅可以作为区块链节点运行，还可以作为客户端附接到 RPC 端点并启动一个命令行控制台，为区块链提供交互式脚本功能。

要附接到一个 RPC 节点并打开 JavaScript 控制台，只需输入：
```
geth attach [rpc_url]
```

C. 通过第三方应用程序，例如 `Truffle` 或 `Remix`

第三方应用程序会附加到一个 RPC 端点，以便与以太坊区块链交互。诸如 `Truffle` 和 `Remix` 等智能合约编译工具都具有连接到 RPC 服务器的功能。一旦 RPC 端点连接成功，编译后的智能合约字节码文件就会被发送到所附加的 RPC 服务器，然后广播到区块链网络的其余部分。

`Geth` 是开发者社区采用的最流行的以太坊客户端。除了 `geth`，还可以使用其他客户端，例如基于 Java 的 `Besu` 和 `quorum` 客户端。`Geth` 节点可以将整个区块链下载到本地存储，因此，以全节点模式运行 `geth` 需要系统拥有数百 GB 的存储空间。

## 第 6 章 以太坊架构与概述

#### Truffle：最全面的智能合约开发工具

`Truffle` 是一个智能合约开发工具，允许用户编译、测试、调试和部署 `Solidity` 代码。在沙箱中试用 `Truffle` 可以很好地了解如何编译和部署智能合约。

##### 任务 1：安装 Truffle 软件包

`Truffle` 软件包可以通过 Node 包管理器（`npm`）下载和安装：

```
npm install -g truffle
```

此处，标志 `g` 表示全局安装，这样 `Truffle` 应用程序就可以从任何目录启动。

##### 任务 2：准备一个项目

`Truffle` 有一个 `init` 命令，允许开发者创建一个用于智能合约的沙箱项目。只需进入目标目录并运行以下命令：

```
truffle init
```

系统将创建一个包含示例智能合约和配置文件的示例项目。你也可以将示例智能合约替换为自己的智能合约，并使用新的配置文件。

##### 任务 3：编译和部署智能合约

在编译或部署智能合约之前，需要配置 `Truffle` 配置文件。`Truffle` 工具使用默认文件名 `truffle-config.js` 来查找配置。此配置文件有两个主要组件需要配置。第一个是网络组件，你需要指定编译后的智能合约将部署到哪个区块链网络。网络的配置将包括网络名称、RPC 端点的主机地址、RPC 端口的端口号以及 `network_id`。第二个配置组件是编译器。对于编译器配置，需要指定 `Solidity` 编译器的版本。

```
module.exports = {
  networks: {
    development: {
      host: "127.0.0.1",
      port: 8545,
      network_id: "*" // 匹配任何网络 ID
    }
  },
  compilers: {
    solc: {
      version: "⁰.8.0"
    }
  }
};
```

一旦项目准备好进行编译，只需运行：

```
truffle compile
```

编译结果将存储在 `build` 目录中。此目录将包含字节码和 ABI（应用程序字节码接口）文件。智能合约字节码是要部署到区块链的可执行代码。ABI 代码用作外部应用程序调用智能合约的函数查找映射表。

要从 `Truffle` 部署编译好的智能合约，只需输入：

```
truffle deploy
```

此命令将使用 `truffle-config.js` 文件调用部署操作。它将通过使用配置文件中网络部分指定的 RPC 客户端服务，将智能合约部署到网络。

还有一点需要强调。在部署智能合约时，开发者需要有一个钱包来向区块链发送部署交易。对于开发区块链，由于这是内部使用且没有安全问题，用户只需在配置文件中添加一个钱包地址即可。对于生产环境部署，则需要遵循严格的生产安全流程。

##### 任务 4：使用 Truffle 控制台与区块链交互

Truffle 提供 `console` 命令来与连接的区块链节点交互。共有两种类型的控制台可用。一种是与测试网或主网区块链连接的控制台，可通过以下命令启动：

`truffle console`

另一种是与开发用区块链连接的控制台，可通过以下命令启动：

`truffle develop`

普通控制台与开发控制台的主要区别在于：普通控制台连接到现有区块链，而开发控制台会创建一个嵌入式区块链、默认账户并发行代币供开发者使用。
