# 区块链核心技术支持概览

可扩展性通过将价值交换系统所需的数据和计算分布到多个独立计算机上来处理，这些计算机执行相同版本的软件。容错性通过将区块链数据复制到区块链中的所有节点，并在每个区块链节点上冗余执行所有计算来处理。一致性通过实施提供拜占庭容错的共识协议（或算法）来处理。这些算法成为区块链系统中所谓的挖矿组件。计算成本高昂的共识协议通常会带来更可靠的区块链系统，因为该协议使得网络中的节点难以串通，从而破坏区块链。正如我们在“分布式系统”一节中所看到的，参与区块链系统的计算机越多，计算机串通并破坏区块链的可能性就越小。一般来说，当我们比较两个区块链实现时，拥有计算成本更高共识协议的区块链实现以及网络中拥有更多节点的区块链实现将更可靠、更安全、更受信任。

与`CAP`定理相关的设计权衡体现在对区块大小和新区块创建频率的精心选择上。区块大小决定了区块可以容纳的交易数量。区块越大，需要在节点之间通信的信息就越多，从而增加了节点中数据不同步的可能性。同样，如果区块创建更频繁，则需要在节点之间通信的数据也会更多，这同样会增加节点中数据不同步的可能性。同时，较小的区块和较低的区块创建频率会限制区块链在每秒交易数方面的吞吐量。

最后，密码学确保了区块链内交易的安全性和匿名性。在所有交易中，参与方通过其公钥进行识别。由于参与方可以拥有多个公钥，因此仅凭公钥很难推断出参与方在所有交易中的身份。区块链交易中的价值接收方需要拥有其私钥才能访问他们接收到的价值。属于某一方的私钥存储在他们控制且只有他们可以访问的安全钱包中。这使得一方收到的价值难以被盗取或被欺诈性夺走。

## 哈希函数

哈希函数在区块链交易中被广泛使用。它们用于数据压缩、安全性以及实施共识协议。区块链交易中的信息可能经过多个加密和哈希循环，这使得区块链中的交易信息难以被破解（如“破解代码”）。有人断言，区块链中的多重加密哈希甚至可以提供针对量子计算机的保护。

## 工作量证明

哈希概念也被用于一种广泛使用的共识协议——工作量证明，该协议将在第 5 章中描述。该共识协议要求生成符合特定模式的哈希值。正如我们在密码学与数字签名一节中所讨论的，生成哈希值的算法非常高效，不需要大量的计算资源。然而，创建具有特定模式的哈希值在计算上是昂贵的，因为寻找具有特定模式的哈希值是一个暴力搜索过程。

本节对区块链核心技术的支持机制进行了非常高级的描述。
