# 比特币经济学与共识协议

使用比特币购买咖啡和甜甜圈？一旦我们处于均衡经济学状态，美元兑比特币汇率会发生什么？这些都是我们仍需弄明白的问题，但理解比特币的创造数量及其价值背后的经济学原理，使得这个话题非常有趣。

## 共识协议

在上一节中，我们学习了比特币使用的工作量证明共识机制。我们注意到工作量证明是一种计算成本极高的共识机制。工作量证明以可扩展性乃至可持续性为代价保证了不可篡改性。

在本节中，我们将回顾其他共识机制，如权益证明、委托权益证明和权威证明。

### 权益证明

在权益证明中，节点质押一定数量的货币或财富来充当验证者（具有创建新区块功能的节点）。这些节点拿出一定数量的财富（财富单位因区块链而异）并将其搁置或保留，以获取充当验证者的特权。这种质押激励这些节点确保区块链不被破坏并保持不可篡改。理论依据是，如果区块链被破坏，这些节点将失去其质押的财富。

在所有质押一定数量货币的节点中，使用确定性算法选择一个验证者。这种确定性算法可以根据区块链实现方式而改变。确定性算法的一个示例是采用轮询方式选择验证者，从质押量最高的节点开始。其他确定性算法可能通过对质押量进行加权来选择验证者，因此质押量高的节点被选中的频率会高于质押量低的节点。被选中的节点随后创建区块。这里没有像工作量证明中那样为了求解难题或寻找具有特定数量前导零的哈希值而进行的竞赛。在权益证明中，验证者不会获得挖矿奖励或区块奖励；他们只收取所有交易中的交易费用。

权益证明的优势在于：能源效率更高，耗电量更少，不需要昂贵的专用硬件，并且我们已经处于均衡经济学状态，所有运作仅基于交易费用，而非区块奖励加交易费用。

权益证明的风险在于：选择验证者可能存在问题。一些共识协议试图在选择验证者时引入某种随机性，但这可能被操纵，从而产生区块链被破坏的风险。质押也无法唯一保证。这意味着我们是基于公钥进行质押。同一个个体可能拥有多个密钥进行质押，这会在区块链上造成大量中心化控制，这可能成为问题。最后，质押并不总能保证更高的交易吞吐量性能。我们节省了电力，节省了计算能力，但在某些情况下，这并不一定转化为每秒交易处理性能的提升。

### 委托权益证明

作为对权益证明的改进，另一种称为委托权益证明的共识机制被发明出来。

在委托权益证明中，所有拥有质押或为验证而进行过质押的节点向社区提出提案，说明他们希望从验证或挖矿中获得什么回报，以及他们愿意在验证性能方面提供什么保证。

网络随后会对这些提案进行投票，以决定哪些节点可以被选中执行验证；在选中的节点中，算法会挑选出一组验证人；然后，在这一组验证人中，会采用某种确定性方式来选定最终的验证者。

现在，如果委托人（或任何执行验证的人）在提案中声明了希望获得奖励，并且被社区投票选入了验证人小组，那么他们就能获得奖励。委托权益证明在投票过程中营造了一定程度的民主。虽然我们在权益证明的基础上有所改进，但某些缺点依然存在。总的来说，权益证明和委托权益证明都让我们更接近中心化，但它们在能耗和可持续性方面带来了一些优势。

另一种共识机制是权威证明。在权威证明中，想要被选中的验证人需要先确立自己的身份。他们决定不再匿名；一旦身份确立，并且我们现在能唯一地确定这个地址对应这个身份，我们就可以沿用权益证明或委托权益证明的方法。但关键在于，通过确认身份，他们为自己合谋或破坏区块链的行为制造了阻碍。

这是区块链和分布式计算领域一个令人兴奋的研究方向。除了本节简要描述的这三种共识机制外，还有更多机制被提出，例如耗时证明和历史证明。`Oyinloye, Teh, Jamil, & Alawida, M. (2021)` 对其他共识协议进行了综述。

## 以太坊

本书的第二部分将深入探讨以太坊和企业以太坊，解释如何使用以太坊开发区块链商业应用。在本节中，我们将简要概述以太坊及其与比特币在实现上的区别。以太坊是区块链的另一种实现方式，在一些非常重要的方面与比特币有所不同。

在上一节中，我们看到，在比特币里，构成交易基础的条款和条件必须非常简单。比特币希望保持智能合约的极其简单性，因为复杂的智能合约存在计算成本高昂以及软件缺陷风险更高的问题。

以太坊的创始人维塔利克·布特林[¹³]很早就参与了比特币社区。他认为，虽然比特币是一种优秀的数字货币，但用于指定智能合约的脚本语言`Bitcoin Script`功能太弱。除了比特币促成的价值交换支付之外，布特林还设想了一类去中心化应用，这些应用将借助分布式账本、隐私保护、共识机制和智能合约的特性来实现。布特林勾勒了一个更加强大的智能合约语言的愿景，这种语言能够在区块链之上创建复杂的去中心化应用。他希望创建一种图灵完备的智能合约语言，支持循环和复杂的条件分支。

以太坊包含一种非常易用、类似 JavaScript 的语言，称为`Solidity`，用于开发能够表示与现实世界交易条款和条件相关的业务逻辑的智能合约。

`Solidity`的出现使以太坊被一些人称为可编程货币或可编程通证。使用以太坊开发的区块链应用有时被称为 dApp 或分布式应用。
