# 第 5 章 区块链实现概览：比特币、以太坊和超级账本

## 区块链实现概览：比特币、以太坊和超级账本

核心数据结构是一个账户，交易输出被整合在其中。超级账本内的底层数据结构是一个属性-值数据库。

比特币使用工作量证明共识机制。以太坊也使用工作量证明共识机制，并且他们正计划转向权益证明机制。正如我们在讨论超级账本时所看到的，超级账本内的共识机制是可插拔的。

比特币是一个无需许可的公共区块链。以太坊的实现可作为无需许可或许可型使用。在更为人熟知的场景中（构建分布式应用），我们使用以太坊的无需许可公共实现。如果你想构建企业级应用，可以使用名为企业版以太坊的以太坊实现，这可以是许可型公共或许可型私有。

超级账本是许可型的，并提供了可扩展性。比特币中有许多节点；然而，性能较低。在以太坊中，同样节点众多，性能也较低。通常，对于每个超级账本区块链应用实现，节点数量不多，但拥有非常高的性能和高吞吐量。

使用比特币脚本编程语言的比特币智能合约非常有限。在以太坊中，你拥有`Solidity`，它是一种图灵完备的编程语言。原生超级账本`Fabric`通过`Go`编程语言提供了图灵完备的智能合约语言。超级账本中的其他框架支持使用其他编程语言编写智能合约。

下表总结了比特币、以太坊和超级账本之间的差异。

|          | 比特币 | 以太坊 | 超级账本 |
| :------- | :----- | :----- | :------- |
| **目的** | 支付   | 支付与分布式应用 | 企业级分布式应用 |
| **治理** | 比特币改进提案 | 以太坊改进提案 | Linux 基金会 |
| **货币** | `BTC` | `ETH` | 无 |
| **挖矿奖励** | 6.25 `BTC` | 2 `ETH` | 无 |
| **数据结构** | 交易 | 账户 | 灵活（属性-值数据库） |
| **共识机制** | 工作量证明 | 工作量证明（计划中为权益证明） | 可插拔 |
| **智能合约** | 比特币脚本 | `Solidity` | `Go` |

### 新兴区块链实现

现在让我们看看区块链领域正在发生的一些令人兴奋的新进展。比特币和以太坊目前都面临网络拥堵、高交易费用以及可扩展性问题。

在过去 36 个月里，比特币发布了两个第 1 层改进。第 1 层更改是对区块链实现的核心区块链层进行的修改，而第 2 层更改则针对区块链实现的集成层（如第[4](https://doi.org/10.1007/978-1-4842-8164-2_4)章所述）。比特币对交易中数据的存储方式、交易中脚本或智能合约的存储方式，以及脚本在比特币网络上的哈希和通信方式进行了数据结构改进。第 2 层创新包括闪电网络等发展。闪电网络允许参与方离线交易，然后将离线交易价值的聚合提交到主区块链网络。这提高了可扩展性。

以太坊也有许多第 1 层改进正在进行中，这些改进主要用于提高可扩展性和安全性。其中包括：

-   **分片**：主以太坊链将被分割成 64 条链，以提高性能。
-   **质押**：即权益证明共识机制的实现。一个名为信标链的开发版本（采用权益证明）已经发布，计划是将这条链缓慢合并到主以太坊网络中。

以太坊也有第 2 层创新，例如`Arbitrum`，这些创新是在以太坊核心组之外完成的。`Arbitrum`允许你将交易汇总到以太坊之外，并将聚合结果写回以太坊。
