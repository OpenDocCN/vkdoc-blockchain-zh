# 第 3 章 区块链组件与架构

从区块链纯粹主义者的角度来看，像比特币和以太坊这样的公有无许可区块链是最分布式和去中心化的，参与节点之间的权力分配最为平等。比特币和以太坊在用于更改或演进区块链平台的治理机制以及指定交易条款的功能方面有所不同。我们将在第[5](https://doi.org/10.1007/978-1-4842-8164-2_5)章中更详细地讨论这一点。类似地，在分布式-去中心化的连续统中，使用超级账本和其他私有许可区块链平台开发的应用程序将是最中心化的。

在下一节中，我们将总结区块链的组件，并回顾第四个组件：智能合约。

## 智能合约与区块链组件总结

在本节中，我们从业务层面总结区块链的四个关键组件。这四个组件是分布式账本、共识、隐私和智能合约，如图 3-3 所示。

***图 3-3.** 区块链组件总结*

到目前为止，在本章中，我们已经回顾了区块链的分布式账本、隐私和共识组件。

分布式账本是区块链网络中所有交易的系统或记录。区块链网络中的所有节点都拥有分布式账本的精确且真实的副本——当然，这受限于 BASE 语义的最终一致性属性。分布式账本是一个仅可追加的记录系统；也就是说，包含交易的区块可以被添加到账本中，但区块和交易不能被更改或删除。

区块链的隐私组件使得分布式账本成为一个保护隐私的分布式账本，确保交易中参与方的身份仅能被有权访问的人看到。隐私组件还确保所有交易都被签名，从而使每笔交易都是可归属且经过身份验证的。最后，隐私组件验证所有交易，以确保交易涉及的各方有权进行该交易。

区块链的共识组件负责创建区块，并确保区块中的所有交易都经过验证和确认。它还确保区块遵循所有

`rules required for a block to be valid`，最后，它确保创建该区块的节点能够证明其有权创建区块。节点证明这一点的一种方式是，表明它们已经找到了能够产生符合区块链规则所规定模式的哈希值的组件。共识组件还确保，当一个节点收到一个有效区块时，它会将该区块附加到其区块链上。

共识机制确保所有节点上的分布式账本包含的数据均是所有参与方已同意、被认定为正确且真实的，并代表区块链的真实状态。

区块链的最后一个组成部分是智能合约。智能合约以可执行代码的形式嵌入了完成交易所需的条款和条件。例如，当坎迪斯将她的漫画卖给马可时，坎迪斯可以花费马可给予她的资金的条款被编纂成代码，只有当这些条款得到满足时，坎迪斯才能使用这笔资金。这些条款可能很简单，比如坎迪斯能够访问她的私钥，也可能更复杂，比如需要确认坎迪斯已将漫画寄给马可。

一个针对银行的智能合约示例可以包括存款产生利息的条款；该智能合约将确保一个自动化交易以约定的利率和约定的频率（每月、每季度、每年）将利息金额存入客户账户。不同的区块链实现，在智能合约所能表示的逻辑类型方面，具有不同程度的灵活性。智能合约内置于交易中；所有条款都由计算机执行，因此不能含糊不清或需要人工解读。从这个意义上说，智能合约有潜力为区块链上的交易创造透明度。智能合约也带来成本和风险。对于每一个交易，区块链上的所有节点都需要执行智能合约，这会产生必须由交易所有者承担的计算成本。一旦智能合约被提交到区块链，就无法更改，这在软件缺陷方面造成了不可预见的风险。

总之，区块链的四个关键概念组成部分是分布式账本、隐私、共识和智能合约。这四个组成部分共同确保了分布式账本中每个区块中每个交易的隐私性、区块链上每个节点上数据的真实状态，以及在交易结算之前所有合约条款的履行。所有这一切都是在没有任何中心化第三方参与的情况下实现的。这就是区块链强大的原因；这四个组成部分协同工作，为区块链提供了有可能为个人、行业、整个经济乃至整个社会创造利益的功能。

在下一节中，我们将讨论区块链的局限性。

**区块链的局限性**

到目前为止，本书已详细描述了区块链的潜在益处、构成任何区块链实现基础的核技术，以及四个关键概念组成部分如何协同工作以实现区块链的潜在益处。在本章以及第 [2](https://doi.org/10.1007/978-1-4842-8164-2_2) 章中，我们已暗示了区块链的潜在局限性，其中一些局限性与分布式系统设计所面临的挑战有关。在本节中，我们将讨论区块链的局限性，并将其分为五类：能源消耗、可扩展性、恶意活动风险、变更管理以及保密性风险。这些局限性产生了约束，并将驱动我们在构思、设计和实施区块链时应当理解的设计权衡。
