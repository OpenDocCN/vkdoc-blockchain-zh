# 第 3 章：区块链组件与架构

从事漫画销售的朋友想出售她的*阿奇*系列漫画。

现在我们将重复同样的过程。法蒂玛要出售她的漫画，她的朋友们会购买这些漫画，我们需要证明法蒂玛确实有权出售她的漫画，所有购买漫画的个人都获得了授权使用资金进行购买，每笔销售交易都由所有参与方各自记录在一张纸上，并经过核实验证所有人是否正确记录了所有销售交易。一旦记录完十笔销售交易，所有参与方将再次尝试通过解谜来达成共识。

不过这次有一个不同之处。谜题解出后产生的水印将包含第一个水印的信息。这一次，假设陈（坎迪斯和法蒂玛的另一个朋友）是第一个解出谜题的人。陈将宣布自己解出了谜题，复制自己的纸张，并将这些副本发送给所有参与漫画销售的人。当人们收到陈的纸片时，他们能快速验证陈是否正确解出了谜题；他们还会注意到，陈发送的纸片上的水印中包含了来自马尔科第一张纸片水印的信息。这将成为他们的线索：陈发送的纸片需要与马尔科的那张纸片连接起来。

陈的纸片就是我们的第二个区块，它被链接到马尔科的前一个区块上。现在我们有了区块链的前两个区块！请注意，如果有人试图篡改马尔科的那张纸片，导致该纸片上的水印发生变化，那么他们也必须篡改陈的纸片，因为这张纸片包含了前一个水印的信息。所以现在修改第一个区块变得更加困难。

现在我们可以想象先前的过程重复多次。每完成十笔销售交易，就会附加一张新的纸片，其水印包含前一张纸片的信息。换句话说，更多的区块被添加到我们的区块链中。随着越来越多区块被链接在一起，修改先前区块的难度就会逐步增加。我们可以相当确信地说，区块链中的每个区块都是不可篡改的，任何区块中的所有交易都无法更改。我们还可以看到所有销售交易都有一条审计轨迹。我们将这条审计轨迹称为`provenance`（溯源）。

你可能会问，那些参与方作弊的纸片会怎样？嗯，如果有人在这种纸片上解出了谜题并将其发送给所有人，可能会发生两种情况：（1）人们可能会拒绝这张纸片，因为他们无法验证该纸片上的水印或交易；（2）他们无法将这张纸片链接到其他纸片上，因为他们没有一张水印能与此纸片上要求的前一张纸片水印信息相匹配的纸片。

这就是区块链如何维护其所有规则和数据的准确性与完整性，即使我们无法保证区块链中的所有参与者都是值得信赖的。这是一个非常重要的观点，你一定已经联想到了[第 2 章中的拜占庭容错机制与我们经济体系中的低效问题](https://doi.org/10.1007/978-1-4842-8164-2_2)，因为我们的系统无法确保[第 1 章](https://doi.org/10.1007/978-1-4842-8164-2_1)中价值交换参与方之间的信任关系。

在本节中，我们使用了一个假设性的例子，还附带了花哨的插图……但请耐心听我讲完。

通过故事叙述的方式，我们从概念上解释了区块链的两个重要组成部分——**分布式账本**和**共识机制**——的工作原理及其作用。我们同时也理解了区块链如何获得**不可篡改性**和**可追溯性**这两个重要特性。为了使这些组件的工作原理易于理解，我们假设所有参与方的身份都是已知的，从而忽略了区块链的另一个重要组成部分，即**隐私性**。

在下一节中，我们将回顾中心化账本与分布式账本之间的区别，并从技术角度描述交易是如何形成并传达给区块链网络中的参与者的。这将帮助我们重新审视隐私的概念，并解释其在区块链中是如何运作的。

## 分布式账本与区块链技术概述

在本节中，我们将回顾中心化账本与分布式账本之间的区别。然后，我们将以此为基础，讨论价值交换交易是如何在区块链网络的参与者（也称为节点）之间创建和传递的。这些机制将帮助我们理解如何保护所有参与者的隐私。

为了理解中心化账本的概念，让我们看一个银行和一家作为你的投资资产（如股票）托管人的公司的例子。

当银行的客户与银行进行交易时，银行会为他们记录这些交易。银行还会将交易记录的副本发送给你，你可以选择以某种格式保存这些记录。你可以通过银行每月发送的对账单获取副本，或者如果银行提供网站（几乎所有银行都提供），你可以登录账户查看交易记录、搜索满足特定条件的交易，甚至下载电子版交易记录。可以保证的是，在你下载交易记录时，它们与银行掌握的信息完全一致。在我们描述的这种情况下，只有银行拥有其所有客户全部交易的记录。每个客户都将这份记录视为所有交易的权威来源——可以说银行拥有金本位记录！我们可以说银行拥有所有银行交易的**中心化账本**。

除了银行之外，没有任何其他人拥有或能够访问所有客户全部交易的记录。我几乎可以肯定你会问：“这难道不是一件好事吗？”嗯，这取决于情况。如果银行被黑客攻击并丢失了所有数据，那么问题就来了。如果银行出于恶意原因决定删除你所有的存款交易（当然，他们永远不会这样做，对吧？），那么问题也来了。银行也可能因自身的不称职或疏忽而导致数据丢失。如果银行发生安全漏洞，那么其所有客户的隐私都会受到影响。银行可以单方面决定向你提供多少历史交易记录。如果你需要比银行单方面保留的历史记录更久远的信息，那么问题就来了。如果出于某种原因，你对银行记录的一笔交易有异议，那么银行永远是正确的，而你永远是错误的。当然，我们信任银行。他们怎么可能出错呢？如果银行因技术困难而导致其中心化账本不可用，那么这意味着我们不仅无法查看交易记录，也无法进行任何交易；也就是说，我们既不能存钱也不能取钱。在中心化账本中，拥有该账本的当事方拥有不成比例的权力。

我们可以为充当托管人的公司提供类似的说法。

我们的投资资产，例如股票。当我们说“拥有股票”时，很少会持有任何证明所有权的实物[⁵]，股票所有权仅仅是富达投资（Fidelity Investments）或嘉信理财（Charles Schwab）等托管公司数据库中的一条记录。上一段概述的挑战同样适用：如果这些公司遭受黑客攻击并发生数据丢失，就会出问题；如果这些公司恶意篡改其数据库，也会出问题；等等。

然而，中心化账本也有其优势。它们能够比分布式账本更轻松、更廉价地实现我们在[第 2 章](https://doi.org/10.1007/978-1-4842-8164-2_2)讨论的`ACID`语义。现在，让我们将注意力转向分布式账本。

[⁵]: 在过去，股票所有权由持有实体股票所有权证书来证明。

如果一家银行使用分布式账本运作，那么该银行的所有客户（如果他们愿意）都可以拥有该银行与所有客户之间所有交易的完整副本。每个客户还将得到保证，他们持有的账本与银行持有的副本以及其他每个客户持有的副本完全一致。此外，未经至少大多数客户同意，银行将无法单方面更改有关账本维护的规则。这些规则可以包括，例如，账本中保留的历史记录量、交易的格式以及每笔交易应包含的数据。银行或任何客户也都无法篡改任何已提交到账本中的交易。客户还将了解将交易提交到账本的规则，并且他们都将遵守这些规则。我们在此描述的分布式账本，将能够缓解之前描述的与中心化账本相关的许多挑战，例如银行单方面行事、银行恶意操作、银行遭受数据丢失或银行账本无法访问等问题。

不过，你可能会问，为什么让客户访问其他客户的交易是个好主意？这难道不会侵犯客户的隐私吗？在区块链分布式账本中，相关方的隐私通过确保交易不包含易于识别的相关方信息来维护。只有参与交易的一方才知道另一方的身份。即使所有客户都拥有分布式账本的副本，并且可以访问分布式账本中的所有交易，他们也无法确定交易参与方的身份。交易中使用属于某一方的加密密钥（公钥）来标识该方。该公钥与相关方身份之间的映射关系并非公开可得。通过这种方式，区块链中使用的分布式账本具有**隐私保护**功能。与中心化账本的银行可能遭受的安全漏洞相比，隐私保护的分布式账本能够减轻其带来的影响。
