# 区块链的限制与可扩展性

`applications.8`

让我们从第一个限制开始——能源消耗。区块链中过度能源消耗有三个驱动因素，这些因素对于构建拜占庭容错系统是必要的。这三个驱动因素是：冗余计算、共识机制以及大量节点的需求。

为了实现拜占庭容错，每个节点需要独立运行，因此每个节点都会重复其他节点已经完成的计算。当节点收到交易时，它会验证并确认该交易，同时执行智能合约中的代码；当节点收到区块时，它会验证该区块，并重新验证区块中包含的交易。这些计算是必要的，以确保每个节点是自包含的，这提高了区块链网络的整体可信度。

共识机制在设计上计算成本高昂，因为这增加了区块（以及区块中提交的交易）不可篡改性的信心。共识机制还被设计成一种竞争机制，因为这样能创建一个更可信的区块链网络，确保没有任何一个节点（或一组节点）有机会滥用创建区块的权力。大量节点（的需求）也服务于增强区块链网络可信度的目的。正如我们在[第 2 章](https://doi.org/10.1007/978-1-4842-8164-2_2)中看到的，大量的节点使得大多数节点合谋变得极其困难（如果不是不可能的话）。

虽然我们承认所有导致能源消耗增加的驱动因素都是必要的，但这并不能否定我们正在消耗过多能源的事实。能源是有成本的，不仅涉及参与方的经济成本，还包括过度能源消耗对环境的影响。包括中国在内的多个国家已经禁止在其境内的计算机上执行共识机制（挖矿）。许多在比特币网络上运行节点的参与方正试图转向可再生能源。面对气候变化对地球带来的风险，过度能源消耗是一个关键的局限性，有可能阻碍区块链的采用。

我们将讨论的区块链的第二个限制是可扩展性。我们将可扩展性定义为每秒提交到区块链网络的交易数量。每秒交易数量这个指标有时也被称为区块链网络的吞吐量。当一笔交易被包含在一个区块中，并且该区块已被区块链网络中的大多数节点添加到区块链上时，这笔交易就被认为已提交到区块链（这类似于说交易已完成结算）。区块链支持的吞吐量至少应等于其要解决的业务问题所需的交易量，包括这些交易量的预期增长。
