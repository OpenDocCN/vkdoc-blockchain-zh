# 代码阅读指南

以下七点将帮助初学者更轻松地阅读这份合约：

- 计算机与英语使用者一样，从上到下、从左到右读取代码。将一行代码放在另一行之前，通常意味着计算机会优先执行那条指令。

- 通常，程序接收输入并返回某种输出。`可计算`函数（可由计算机执行的数学函数）被定义为可写成算法的函数。

- `算法`接收数据，对其执行操作，并返回某种输出。`程序`则是嵌套了其他算法的算法。

- 算法就像一台机器：你可以多次重复使用它。因此，编写算法指令——即`编程`——会让你感觉很像填写“疯狂填词”，计算机会在之后根据用户（或在以太坊中，是合约）通过交易或消息调用提供的信息来自动补全。有时这些信息只是一个数字（例如，5 ether）。

- `运算符`是英文单词之间的符号，例如等号、加号和减号。这些符号的用法大多符合你的预期，但也有少数例外。你可以在表 4-1 中看到 Solidity 运算符。

- `类型`是计算机编程中的名词。所以当你看到一个类型时，你就知道在“疯狂填词”的对应位置允许填入什么内容。Solidity 中的一个常见类型是 `address`。

计算机最初的用途是快速进行数学计算。几十年来，使用计算机的人大多是物理学家，他们希望解决棘手的数学问题，以找出诸如此类问题的答案：发射阿波罗 11 号的最佳日期和时间是什么，才能使飞往月球的距离最短？

EVM 更接近这类最初的计算机，但它更适合处理复杂的会计和财务对账——就像你在商学院学习使用 Microsoft Excel 编写电子表格程序时可能会学到的那样。回想一下，数据库本身就是电子表格，而计算机程序则操作这些数据库。因此，当你`声明`某物时，你是在告诉计算机把它放进电子表格中——具体来说，是放进`堆栈`里。

计算机会自行判断需要准备多少内存来存储任何`临时`（即所谓`动态`）计算中的值——这些是用于计算 if-then 等偶然事件的小型关键逻辑语句。（定义堆栈和堆很重要，这样才能看出内存密集型程序的危险所在：即要求计算机使用超出其可用范围的动态内存。）

## Solidity 中的语句和表达式

正如你将看到的，Solidity 中到处都是函数。然而，它们的用法各不相同。

某些函数会产生一个`值`，例如一个数字，或者一个真/假问题的答案。这个值具体是什么，由前面提到的 Solidity 的`类型`决定；真/假值被称为`布尔值`。

### 什么是表达式？

产生值的函数被称为`表达式函数`。由于表达式会求值为某种类型的值，因此在编程中，它们可以替代值来使用。

另一些函数是`声明性的`，它们会在计算机内存中创建一个专用空间，这个空间会在每次运行该例程时被使用。这些声明性函数非常重要，因为它们对于编写语句至关重要。

### 什么是语句？

粗略地讲，`语句`告诉计算机执行一个动作。计算机利用表达式来确定如何执行这个动作以及何时执行。因此，计算机程序由语句组成，而语句通常由表达式（或其他语句）组成。

### 函数：公共与私有

在 JavaScript 和 Solidity 中，你可以使用分号来连接语句，并告诉计算机代码中还有另一个语句：

```
function first(); function second()
```

在 Solidity 中，你还可以声明是否希望某些函数在该程序外部可用。这些指定方式如下：

- `public`：外部和内部均可见（会为存储/状态变量创建一个访问器函数）

- `private`：仅在当前合约中可见（默认情况）

#### 注意

用 Solidity 代码编写的函数默认不是 `public` 的。你在创建它们时必须显式地声明为 `public`，否则它们将无法被当前合约以外的其他合约访问。

虽然这只是代码读写能力的入门介绍，但它应该足以让你开始解读我们稍后将讨论的一些智能合约在做什么。

## 值类型

在编写 Solidity 代码时，你可以告诉计算机每条算法指令应预期什么类型的值。本节描述 EVM 可以解释的值类型。

### 布尔值

在代码中称为 `bool`，布尔值是求值为 `true` 或 `false` 的真/假表达式。

### 有符号和无符号整数

在代码中称为 `int` 和 `uint`，这些是数字。如果它们带有符号（即减号），则表示它们是有符号的，可以是负数。因此，无符号整数就是正数。

### 地址

`address` 类型保存一个 20 字节的值，这是以太坊地址的大小（40 个十六进制字符，或 160 位）。`address` 类型也有成员类型。

### 地址的成员

这两个成员允许你查询账户余额，或向账户转账。在智能合约中要小心使用 `transfer`。更好的模式是允许接收方自主提取资金，而不是由合约发起转账。

- `balance`

- `transfer`

### 地址相关的关键字

关键字随 Solidity 语言一同提供。可以说，它们是以预定义方式使用该语言的方法。你可以在代码中使用这些关键字来完成智能合约中常见的任务。其中包括：

- `<address>.balance` (`uint256`)：以 wei 为单位返回 `address` 的余额

- `<address>.send(uint256 amount)` 返回 (`bool`)：向 `address` 发送指定数量的 wei，失败时返回 `false`

- `this`（当前合约的类型）：显式转换为 `address` 类型

- `selfdestruct(address recipient)`：销毁当前合约，并将其资金发送到指定的 `address`

#### 注意

通过使用关键字 `this.balance` 可以查询当前合约的余额。

### 较少使用的值类型

如果你已经是一名高级或中级程序员，以下几种值类型可能很有用：

-   动态大小的字节数组
-   定点数
-   有理数和整数常量
-   字符串常量
-   十六进制常量
-   枚举

### 复杂（引用）类型

一般来说，Solidity 中的类型在 EVM 存储中分配有 256 位内存，即 2,048 个字符。超过这一长度的类型在移动时会产生更高的 gas 成本。在 EVM 栈中分配持久化存储时，你需要谨慎选择。以下是超过 256 位的复杂类型：

-   数组
-   数组常量/内联数组
-   结构体
-   映射

`数组`、`结构体`和其他复杂类型拥有数据位置属性，Solidity 程序员可以利用它来控制这些类型是动态存储在内存中还是持久化存储。这有助于管理费用。

## 全局特殊变量、单位和函数

`全局`特殊变量可以被 EVM 上的任何 Solidity 智能合约调用；它们是该语言内置的。其中大部分返回关于以太坊链的信息。时间和以太币的单位也是全局可用的。数字常量可以带有 `wei`、`finney`、`szabo` 或 `ether` 后缀，并会在以太币的不同子面额之间自动转换。不带后缀的以太币货币数字将被视为 `Wei`。

时间相关的后缀可以用于数字常量之后，以在不同时间单位之间进行转换。这里，秒是基本单位，并且单位被视为通用单位。由于闰年的存在，使用这些后缀计算时间时要小心，因为并非所有年份都有 365 天，也并非所有天都有 24 小时。`1 == 1 seconds`，`1 minutes == 60 seconds`，`1 hours == 60 minutes`，`1 days == 24 hours`，`1 weeks = 7 days`，`1 years = 365 days`。

### 区块和交易属性

注意，这些全局变量仅适用于 Solidity 智能合约。不要将它们与你将在 Geth 中调用的 JavaScript Dapp API（将在第 6 章中学习）混淆。

-   `block.blockhash(uint blockNumber) returns (bytes32)`：给定区块的哈希，仅适用于最近 256 个区块
-   `block.coinbase (address)`：当前区块矿工地址
-   `block.difficulty (uint)`：当前区块难度
-   `block.gaslimit (uint)`：当前区块 gas 上限
-   `block.number (uint)`：当前区块号
-   `block.timestamp (uint)`：当前区块时间戳
-   `msg.data (bytes)`：完整的调用数据
-   `msg.gas (uint)`：剩余 gas
-   `msg.sender (address)`：消息的发送者（当前调用）
-   `msg.sig (bytes4)`：调用数据的前 4 个字节（函数标识符）
-   `msg.value (uint)`：随消息发送的 wei 数量
-   `now (uint)`：当前区块时间戳（`block.timestamp` 的别名）
-   `tx.gasprice (uint)`：交易的 gas 价格
-   `tx.origin (address)`：交易的发送者（完整调用链）

请注意，`msg` 的所有成员（即 `msg.sender` 和 `msg.value`）的值在每个外部函数调用中都可能改变，即使它们是库函数。如果你希望实现对 `msg.sender` 使用有访问限制的库函数，那么你需要手动将 `msg.sender` 的值作为参数提供。

### 全局函数

通常，在 Solidity 中，特殊函数主要用于提供关于区块链的信息，但有些也可以执行数学和加密函数。它们如下所示：

-   `keccak256(...) returns (bytes32)`：计算（紧打包的）参数的以太坊-SHA-3（Keccak-256）哈希
-   `sha3(...) returns (bytes32)`：`keccak256()` 的别名
-   `sha256(...) returns (bytes32)`：计算（紧打包的）参数的 SHA-256 哈希。“紧打包”意味着参数被连接起来，没有填充。要了解如何为参数添加填充，请参阅以下 URL：[`solidity.readthedocs.io/en/develop/units-and-global-variables.html#mathematical-and-cryptographic-functions`](http://solidity.readthedocs.io/en/develop/units-and-global-variables.html#mathematical-and-cryptographic-functions)。
-   `ripemd160(...) returns (bytes20)`：计算（紧打包）参数的 RIPEMD-160 哈希
-   `ecrecover(bytes32 hash, uint8 v, bytes32 r, bytes32 s) returns (address)`：从椭圆曲线签名中恢复与公钥关联的地址，出错时返回 `0`
-   `addmod(uint x, uint y, uint k) returns (uint)`：计算 `(x + y) % k`，其中加法以任意精度执行，不会在 `2**256` 处回绕
-   `mulmod(uint x, uint y, uint k) returns (uint)`：计算 `(x * y) % k`，其中乘法以任意精度执行，不会在 `2**256` 处回绕
-   `this`（当前合约的类型）：当前合约，可显式转换为其地址

此外，还值得一提的是在编写 Solidity 合约时可能很有用的合约相关变量：

-   `super`：继承层次结构中高一级的合约。关于继承的更多信息，请参阅下面章节中的链接。
-   `selfdestruct(address recipient)`：销毁当前合约，将其资金发送到指定地址
-   `assert(bool condition)`：如果条件不满足则抛出异常。
-   `revert()`：中止执行并回退状态更改

### 异常和继承

某些情况会自动导致异常。要查看所有这些情况，请访问 [`exceptions.eth.guide`](http://exceptions.eth.guide)。Solidity 语言也支持多重继承。即使一个合约从多个其他合约继承，区块链上也只会创建一个合约；基合约的代码始终被复制到最终合约中。关于通用继承系统的详细信息，请访问 [`solidity.readthedocs.io/en/develop/contracts.html#inheritance`](http://solidity.readthedocs.io/en/develop/contracts.html#inheritance)。
