# 哈希

*哈希*比加密更安全，至少从某种意义上说，不存在任何私钥能够将哈希“逆向”还原为其原始的可读形式。因此，如果一台机器不需要知道数据集的内容，就应该向其提供数据集的哈希值。

*哈希算法*与加密算法一样接收数据，但会生成固定长度的字符串或数字。在一个大型数据集中更改一个字符，就会导致哈希值变得完全不同。将哈希后的数据恢复为原始形式几乎是不可能的。流行的哈希算法包括`MD5`、`SHA-1`和`SHA-2`。以太坊和比特币协议都使用了`SHA-256`，这是目前最强大的哈希算法。

## 哈希的用途

如果你认出那些以`SHA`开头的哈希算法的名称，很可能是因为你在智能手机或电脑连接 Wi-Fi 并输入密码时，在网络界面中见过它们。由于哈希本质上具有单向性，它们非常适合在不泄露两个秘密值的情况下进行比较。因此，如果你的电脑对 Wi-Fi 密码进行哈希处理，并将其交给已知密码的 Wi-Fi 路由器——路由器本身也应计算该密码的哈希值——并得到相同的结果。这就能确认你拥有正确的密码，并被允许连接。这样做的好处是，任何在网络中窥探的人都无法看到密码本身，只能看到哈希值。
