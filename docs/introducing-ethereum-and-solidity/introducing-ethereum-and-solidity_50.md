# 8. 去中心化应用部署

## 如你所见，部署去中心化应用是新计算范式前沿的一场冒险

Chris Dannen (1)

(1) 美国纽约州布鲁克林

*去中心化应用*（或称*dapp*）与 EVM 协议的其他部分有着相同的理念：不可篡改的承诺。去中心化应用由智能合约组成，正如本书多次提到的，这些合约由以太坊网络上的所有节点几乎同时执行。

实际上，去中心化应用就像运行在 EVM 上、全球可用的网络服务，但用户可以通过普通的`HTML`/`CSS`/`JavaScript`前端来访问——这些前端可通过网页浏览器、智能手机应用或以太坊浏览器（如`Mist`）进行访问。

> **注意：** 本章讨论的主题面向具备一定开发技能的开发者。如果你是编程新手，请与第 9 章一同仔细阅读本章。然后，找一本`JavaScript`入门书籍来提高你的脚本编写能力。接着，访问[`solidity.eth.guide`](http://solidity.eth.guide)获取更多 Solidity 语言教程。

运行基于区块链的应用客户端，远比在云端托管的模式下管理客户端要简单得多。中心辐射型网络应用是垂直扩展的，反映了它们所运行的各个服务器。相比之下，以太坊应用是水平扩展的——这*正是你期望*云应用所具有的扩展方式。

尽管如今加密网络在交易处理能力上确实受到很大限制，但随着协议的其他组件日益成熟，它们将变得更快。

## 关于智能合约的七种思考方式

每个去中心化应用背后都有一系列智能合约。智能合约在以下场景中非常有用，这些场景可能成为原型开发中有趣的问题领域：

- 为现实世界中的某些事物或其他合约维护一个会计系统
- 创建转发合约，例如一个将收入自动重新发送到另一个独立账户的储蓄账户
- 管理多方之间的关系，例如自由职业者协议或薪资发放
- 充当其他合约的软件库
- 充当其他系统或合约集合的控制器
- 为公共网络服务提供特定于应用的逻辑
- 充当开发者可按需使用的实用工具，例如随机数生成器

去中心化应用的开发为应用开发者带来了各种新的关注点，同时也需要理解`Web3 JavaScript` API 和`Solidity`编程语言。希望你读完本书大部分内容后，已准备好直接使用这些工具！

要更好地了解当前正在构建的各种去中心化应用，请访问由 EtherCasts 运营的[`dapps.ethercasts.com`](http://dapps.ethercasts.com)。
