# 理解状态机

正如我们迄今为止多次讨论过的，EVM 是一个状态机。与其简单地定义这个概念然后继续，不如让我们花点时间讨论一下计算机到底是什么，然后再探讨以太坊如何推进这一概念。

## 数字与模拟

有状态计算机概念的基础是开关可以开或关的想法。那些常被称为机器通用语言的 1 和 0，可以理解为一系列隐喻性的开关，它们被设置成某种特定配置，以编码特定的字母、数字或其他键盘符号。键盘上的所有符号（甚至更多）仅需八个开关就能表示，这就是计算机内存以八的倍数堆叠的原因。例如，逗号所谓的字符代码是 `0010 1100`。

在计算机编程中，字母和数字可用于编写通俗称为代码的机器指令。图 3-1 所示的美国研究员兼海军少将格蕾丝·霍珀发明了第一个编译器，它能自动将人类可读的代码转换为机器代码（如 EVM 的字节码），这种代码抽象程度更低，因此更接近我们常听到的 1 和 0。²

### “语句（State-ments）”

单独的代码片段，如果单独来看，大致可以分为两类：表达式和语句。表达式（Expressions）用于求值特定条件；语句（Statements）（注意词根！）用于将信息写入计算机内存。表达式和语句共同作用，让计算机在满足特定条件时以可预测的方式修改数据库。这是自动化的核心，也是我们觉得计算机如此有用的原因！

语句可以求值为 `true` 或 `false`，根据代码的不同，这种二元结果可能导致信息被添加、删除或修改到计算机众多内存地址中的某一个。（由于 Solidity 语言是强类型的，它不像 JavaScript 那样存在“真值”和“假值”语句。）`true` 和 `false`、`yes` 和 `no`、`on` 和 `off` 之间的清晰区分，使得计算机能够安全地代替人类做出决策。

### 数据在状态中的作用

每次你更改计算机内存中的数据时，都可以想象它那数不清的内部开关（其中大部分像本章前面讨论的那样被虚拟化了）处于略微不同的配置状态。状态（State）通常指系统的当前状况：信息在机器不同内存地址间发生的客观系列变化，这些变化导致了其内存的当前内容。

区分属性和状态很重要。状态是可以轻松且可预测地改变的东西。我们以汽车为例。

给汽车重新喷漆是项艰巨的工作，但确实可以做到。油漆颜色就是一个属性的例子。在伪代码中，你可能会这样描述一辆汽车：

```
bodyColor = red
```

在计算机编程中，这被称为键/值对。键 `bodyColor` 被赋予了一个值 `red`。要更改此键的值，你的代码会对该值做出新的声明：

```
bodyColor = green
```

现在你的车被重新喷漆了。它有了一个新的颜色值。

现在假设你指示计算机，这辆车的颜色会频繁变化。换句话说，你将车的颜色设为一个变量。那么可以说，这个变量（本例中即颜色）可以有一个状态，即一个会变化的值。但像 `green` 这样的单个值是没有状态的；`green` 就只是 `green`。

里程表提供了另一个具有可变状态变量的例子。里程表的值可能是 `1000`，这个数字本身没有状态；它只是一个数字。不久，里程表的状态会变为一个新值（`1001`），但这只有在汽车座舱表达出指令，使发动机和变速箱从空档状态变为一档状态时才会发生。

对状态转换概念的熟悉掌握，将帮助非程序员深入理解去中心化系统设计中固有的真正难题。本章接下来的几节将提供一个速成课程。

## EVM 的内部工作机制

如果这是你第一次接触计算机内部原理，重要的是要记住，只要计算机处于开机状态，它就从未真正“休息”。计算机本身在运行一个状态函数，不断检查其状态的变化。它就像一个过于热心的实习生，每秒数千次地想知道是否有新工作落到他的桌上。

当新指令被触发时，计算机执行代码，并可能将新数据写入其内存。重要的是要注意，每次状态变化都必须基于上一次状态变化；计算机不会随意地把信息扔进内存地址。

如果出了什么问题——比如说这些指令中有某条在数学上不可行——机器的状态就会变得无效，程序会退出或停止。事实上，整个系统可能会崩溃。

在编程中，不断检查特定条件的程序被称为循环，因为它们会持续运行（循环）直到满足指定条件。EVM 持续运行一个循环，试图执行当前程序计数器指向的任何指令（即任何“待处理”的程序）。程序计数器的工作方式类似于熟食店的排队系统：每个程序拿一个号码，然后等待轮到它。

这个循环有几项任务：它计算每条指令的 Gas 费用；并且，如果前置计算成功，则在必要时使用内存来执行交易。这个循环会重复进行，直到虚拟机运行完所有待处理代码，或者抛出异常（即错误），此时该交易被回滚。

到目前为止，我们轻松地走过了一个世纪的计算机科学史，才追赶上 EVM。现在我们将开始慢下来，看看其中一些部分是如何实际工作的。

### EVM 持续检查交易

状态机（带内存的机器）可以被视为永不睡眠的存在。作为一个状态机，EVM 在其内存库中拥有所有交易的持续历史记录，一直追溯到第一笔交易。与需要处理不完美记忆的人类不同，计算机的状态（如同它今天存在的那样）是自机器首次启动以来，其内部发生的每一次状态变更的特定结果。

机器最新版本的状态可以被视为这台机器关于当前现实的规范“真理”。在以太坊中，这个真理涉及账户余额，以及使你的余额变成今天这样子的一系列交易。

### 创建关于所发生事件的通用机器叙事

因此，交易代表了一种机器叙事——一种从一个状态到另一个状态的计算上有效的弧线。正如 Gavin Wood 的以太坊黄皮书所述：

> 存在的无效状态变更远多于有效状态变更。无效的状态变更可能包括，例如，减少一个账户的余额而没有在其他地方进行相等且相反的增加。有效的状态转换是通过交易实现的。 ³

随着时间的推移，系统（如比特币）寻求创建一个可信的历史记录，以确保每个后续的状体变更都是合法的，而非由恶意行为者插入的指令。

### 密码学哈希

下一节将解释区块：它们的内容、工作原理以及如何形成链。为了正确理解该讨论，你首先需要了解密码学哈希算法及其用途。

### 哈希函数（或哈希算法）的作用

一般来说，在区块链的背景下，哈希函数的目的是快速比较大型数据集并评估其内容是否相似。一种单向算法将整个区块的交易处理成 32 字节的数据——一个哈希值，或由字母和数字组成的字符串，其中不包含任何关于区块内交易的可识别信息。哈希值为一个区块创建了独一无二的签名，使得下一个区块可以构建在其之上。与加密产生的密文（可以解密）不同，哈希的结果无法被“反哈希”。

**注意**

给定数据集的哈希值总是相同的。两个数据集产生相似哈希值在计算上是不可行的。即使是更改数据集中的一个字符，也会完全打乱哈希值。
