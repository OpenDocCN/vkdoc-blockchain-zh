# 以太坊与比特币如何运用树结构

在数学中，树是一种有序的数据结构，用于存储键值对关联数组。基数树是其压缩变体，可减少内存占用。在普通基数树中，键中的每个字符都描述了通往对应值的数据结构路径，就像一组方向指示。

创建默克尔树需要对大量交易数据“块”进行哈希运算，直到最终生成唯一一个根哈希。在以太坊和比特币中，默克尔树结构用于记录每个区块中的交易账本。默克尔树的根哈希会与其他元数据一起被哈希运算，并纳入后续区块的区块头。因此，可以说每个额外交易（每个区块内）都会不可逆转地改变默克尔根；即使一个错误的交易也会使根哈希变得完全不同，从而显露出错误。这就是区块能够向区块验证算法证明其合法来源的方式，该算法是整体区块处理流程的一部分。

对于比特币客户端而言，确定单个交易的状态只需查看主链最新区块的区块头即可。在区块头中，客户端应能找到默克尔证明，表明该区块的根哈希包含其某个默克尔树中的该交易。默克尔根是所有交易的指纹，这些交易按正确顺序排列，发生在该区块之前的区块链中。

## 默克尔-帕特里夏树

得益于区块头，节点可以快速便捷地查找、读取或验证区块数据。在比特币中，区块头是一个 80 字节的数据块，包含默克尔根以及其他五项内容。比特币区块头包含：

* 上一个区块头的哈希
* 时间戳
* 挖矿难度值
* 工作量证明随机数
* 包含该区块交易的默克尔树根哈希

默克尔树非常适合存储交易账本，但仅此而已。从以太坊虚拟机（EVM）的角度看，默克尔树的一个局限性是，虽然它能证明或证伪根哈希中包含某笔交易，但无法证明或查询网络的当前状态，例如某个用户的账户余额。

## 以太坊区块头的内容

为了弥补这一缺陷并让以太坊虚拟机能够运行有状态合约，以太坊的每个区块头不仅包含一个默克尔（交易）树，而是包含三个分别对应三类对象的树：

* 交易树
* 收据树（展示每笔交易结果的数据）
* 状态树

为了实现这一点，以太坊协议将默克尔树与我们前面描述的另一种树结构——帕特里夏树相结合。这种树结构是完全确定性的：两个具有相同（键/值）绑定的帕特里夏树总会产生相同的根哈希，从而为常见的数据库操作（如插入、查找和删除）提供更高效率。因此，以太坊客户端能够对其向网络提出的各类查询获得可验证的答案，例如：

* 交易`X`是否已被包含在区块中？（由交易树处理。）
* 告诉我过去 30 天内事件`Y`的所有实例。（由收据树处理。）
* 合约账户`Z`的当前余额是多少？（由状态树处理。）

更多关于这些树结构的工作原理及其选择原因，请查阅 [`trees.eth.guide`](http://trees.eth.guide)。

## 分叉

如本章前面所述，如果矿工们无法就最长、最重的链达成一致，矿工网络可能会一分为二。在加密货币社区中，分叉总是引起诸多争议，因为它似乎意味着人类社区的分裂以及机器网络共识的丧失。

实际上，初生分叉一直在发生。有时一个分支消亡，有时两个分支都消亡，有时一个分支存活下来并传播出一个获胜的叔块。分叉发生在两个有效区块指向同一个父区块时，但部分矿工看到其中一个，其余矿工看到另一个。这实际上创造了两个版本的“真相”，确保这两个群体不再能被认为处于同一网络。

**注意：** 状态分叉比协议分叉严重得多。在协议分叉中，没有数据被更改，但矿工可能会调整参数或更新其节点上的代码，使其按照社区已同意是整体改进的修改后规范运行。因此可以说协议分叉是自愿的，而状态分叉则未必如此。

在以太坊中，这些不断萌生的分叉会在四个区块内得到解决，作为数学必然性，因为一条链会找到胜利者、变得更长，并开始通过激励措施将其他节点“拉”向它——不仅包括找到并执行正确区块的矿工费，还包括收集叔块奖励的所有额外激励。

有时，一个节点在已经收到大约一到三个区块的奖励后，才会找到“正确”的链。一旦节点跳转到一条更好、更长、更可能的获胜链，该挖矿奖励可能会消失。然而，这一切都发生在四个区块内——即一分钟内——因此这些小的差错被认为无关紧要。

蓄意分叉通常由攻击者部署，目的是进行双重支付：通过同时将一笔余额发送到多个账户来凭空制造资金。

事实上，任何拥有超过 50%算力的人都可以引发所谓的“敌对”蓄意分叉。在双重支付攻击中，攻击者运营着一批拥有大量算力的矿工，发送一笔以太币交易来购买某产品。在获得产品后，攻击者拼凑出一个包含第二笔交易的错误区块。这第二笔交易试图将相同的资金发送回攻击者。然后，他或她创建一个与包含原始交易区块处于同一级别的区块，但其中包含的是第二笔交易，并将所有可能的算力投入到该分叉上挖矿。如果攻击者拥有超过 50%的算力，那么无论区块深度如何，双重支付最终都必然成功。低于 50%则成功可能性大大降低。但这类攻击仍然令人担忧，以至于在实践中，大多数使用以太币的交易所和其他机构在认为转账完成之前，都会等待多次确认。

## 挖矿教程

挖矿是尝试 Geth 的绝佳理由。由于 Geth 是一个极好的学习工具，而且安装相对简单，本节提供 macOS、Windows 和 Ubuntu 的安装说明。

**注意：** 除了此处的安装说明外，以下练习假设你处于 *nix 环境——即在 macOS 或 Ubuntu 14.04 (Trusty) 中运行终端应用程序。关于 Geth 的文档和教程链接，以及所有平台上所有以太坊客户端的说明，请访问 [`clients.eth.guide`](http://clients.eth.guide)。

### 在 macOS 上安装 Geth

首先，在你的 Mac 上打开位于“应用程序”文件夹中的“终端”。然后，在命令行中输入以下内容：

```
brew update
brew upgrade
```

更新完成后，命令行返回提示符，输入以下内容：

```
brew tap ethereum/ethereum
brew install ethereum
```

### 在 Windows 上安装 Geth

下载最新的稳定版二进制文件。从压缩包中解压出 `geth.exe`，打开命令终端并输入以下内容：

```
chdir <解压出的二进制文件路径>
open geth.exe
```
