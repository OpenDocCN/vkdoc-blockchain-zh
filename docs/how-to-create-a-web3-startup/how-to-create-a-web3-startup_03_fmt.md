# 3. Web3 技术栈

## 初创公司的平台

2017 年夏天，尼基尔·维斯瓦纳坦（Nikil Viswanathan）和乔·刘（Joe Lau）共同创立了 Alchemy。这家公司实际上诞生于他们在旧金山的公寓里。

维斯瓦纳坦和刘都曾是斯坦福大学计算机科学专业的毕业生。他们曾在谷歌、微软、Facebook 和 Pinterest 等公司工作过。^(⁵⁶) 他们还创立过几家公司。

但有一家公司会一飞冲天——那就是 Alchemy。从一开始，这家初创公司就专注于为区块链创建一个开发平台。

2021 年，增长以狂热的速度加速。4 月，该公司在 B 轮融资中筹集了 8000 万美元，估值为 5.05 亿美元。^(⁵⁷) 随后在 10 月，Alchemy 进行了另一轮融资。这一次，它以 35 亿美元的估值筹集了 2.5 亿美元。

但这还没完。2022 年 2 月，Alchemy 以 102 亿美元的估值筹集了 2 亿美元。

此外，该公司当时已经实现盈利，并每月从其区块链平台产生超过 10 亿美元的价值。其一些标志性客户包括 OpenSea、Dapper Labs 和 Adobe。Alchemy 的商业模式是根据其计算资源的使用量收费。

这家公司也非常精干，并未动用太多资金。例如，员工人数不到 30 人。毫无疑问，该公司在招聘方面非常挑剔——而这被证明对于打造一个出色的平台至关重要。

Alchemy 的案例凸显了 Web3 的巨大增长潜力。但它也展示了开发工具的重要性。由于区块链的复杂性，需要不同的方法。

在本章中，我们将探讨创始人们需要了解哪些可用工具以及如何评估它们。

## Web3 技术栈

技术栈描述了用于产品开发的工具。这些工具包括库、平台、集成开发环境（IDE）、数据库、网络服务等。

在创业界，某些类型的技术栈已经变得相当普遍。例如，有一种名为 MERN 的技术栈，它包含以下系统：

- `MongoDb`：这是一个 NoSQL 数据库。
- `Express`：这是一个 `Node.js` 网页框架。它有助于处理服务器端活动，如 URL 路由以及 HTTP 请求和响应。
- `React`：这让你能够使用 JavaScript 创建动态的前端界面。
- `Node`：这是一个用于运行应用的 JavaScript 网页服务器。

使用 MERN 技术栈，你可以创建高度复杂的网络应用程序。使用这项技术的公司包括 UberEats、Instagram 和沃尔玛。^(⁵⁸)

那么，Web3 技术栈又是什么呢？嗯，这仍处于演变之中。可能还需要几年的时间，才能出现像 MERN 这样的标准。

但当然，创业者们等不及了。他们需要就使用哪种技术做出决策。是的，这意味着存在风险。首先，需要选择区块链的类型。接下来，你要寻找那些最能实现你目标的工具。

“具体的项目可能需要特定的工具或编程语言，但我鼓励围绕分布式系统架构和加密协议进行第一性原理的学习；或者对于前端开发人员或设计师来说，要培养如何直观地表达分布式网络幕后发生的事情，以及这如何为用户带来价值的能力，”数据经济公司（The Data Economics Company）的董事总经理詹妮弗·欣克尔（Jennifer Hinkel）说。“因为在某些情况下，这些思维方式对于‘Web2’平台和应用的架构来说，是一种彻底的范式转换，我认为开发者花时间牢固掌握分布式架构范式及其与经济价值的联系是非常重要的。”^(⁵⁹)

欣克尔还建议你提出以下问题：

- 我是在构建一个应用吗？
- 我是在构建一个平台吗？
- 我是在构建一个基础设施工具吗？

当然，这些问题的答案会引导你走向不同的技术栈。“根据你旨在构建的目标，你将依赖技术栈的不同层级，因此必须在你构建的基础之上、与你一同构建的东西，以及需要兼容的内容之间做出选择，”欣克尔说。“如果有人构建了一个基础设施工具，那么他们也可能构建原型平台和应用来证明其用途，这是很正常的。”

在 Web3 中，技术栈的不同组成部分被划分为不同的层级。我们接下来将逐一了解这些层级。

### 第一层

第一层是底层区块链的基础。它使得区块链能够运作，从而支撑起比特币、以太坊和各种山寨币。

这是底层技术，包括复杂的网络基础设施、硬件和连接。它使得交易得以进行，并能够创建和运行智能合约。例如，正是在第一层，存在着工作量证明（PoW）、权益证明（PoS）、声誉证明（PoR）等共识算法。加密处理以及以太坊虚拟机（EVM）的运行也在此层级。

对于一家基础设施初创公司来说，创始人可能需要与第一层打交道。如果你正在创建自己的代币或加密货币，情况也是如此。但如果你是在构建一个应用，那么你实际上只需要对这项技术有一个基本的了解即可。

### 第二层

第二层涉及有助于提高底层第一层协议性能和效率的解决方案。再次强调，这主要针对基础设施初创公司。

毫不意外，第二层类别已经看到了大量的增长。这与 Web1 早期发生的情况并无不同。原因在于，需要创建一个坚实的技术基础设施，以支持消费者和企业级的应用。

对于第二层，已有多种创新帮助改进了网络，例如：

- `PoS`：这比最初的 `PoW` 协议速度更快，也更环保。
- `分片`：这对于实现更大的扩展至关重要。无需按顺序处理交易，而是可以在不同的数据块或分片中并行处理。此外，节点也不需要保留区块链的完整副本。
- `区块大小`：这允许在每个区块中处理更多的交易。一个相关的第二层系统例子是比特币的 `SegWit`。通过排除数字签名数据，每个区块有了更多的空间。

即使有了这些增强功能，用户社区也可能存在分歧。进行更改可能需要时间——并且在涉及第二层平台时可能会面临阻力。因此，最终可能会出现硬分叉，从而产生新的平台。

谈到第二层，无疑有很多选择，这对企业家来说是一个挑战。这可能意味着你可能会在一个平台上投入大量资源，而该平台最终可能被边缘化甚至变得过时。

事实上，核心基础设施可能会按照幂律法则运作。这是一个统计学概念，指的是一项数量的变化会导致另一项数量发生更大的变化。例如，将正方形的边长加倍，其面积将变为原来的四倍。

这与技术有什么关系呢？这实际上关乎标准的重要性。当一种标准出现时，它通常会占据主导地位。这就是操作系统领域发生的事情，无论是安卓（Android）、iOS 还是 Windows。

核心区块链平台也很可能发生同样的情况。根本没有足够的资源来同时基于多个平台进行构建。尽管到目前为止，Web3 生态系统中还没有出现像苹果、谷歌或微软这样的巨头。

为了构建规模，第二层运营商正在提供经济激励。这通常涉及向第三方付费，让他们在其平台上创建应用和解决方案。诚然，对于企业家来说，这可能意味着快速获利。但从长远来看，这可能是错误的方法。例如，微软在其移动操作系统上就曾这样做过。但最终它失败了，因为苹果和谷歌成为了这一类别的主导者。

请记住，Web1 提供了一个历史视角。当时对互联网的电信基础设施进行了大量投资。但这做得有些过头了，导致了资源供应过剩。结果，许多基础设施领域的参与者失败了。第二层领域很可能也会出现这种情况。

另外需要考虑的是：在 Web1 阶段，最大的赢家并非基础设施参与者。相反，它们来自应用端，例如亚马逊。

尽管如此，Web3 基础设施仍有大量工作要做。以下是来自安德森·霍洛维茨基金（Andressen Horowitz）博客的观点：“在第二层区块链上，扩展交易数量和速度以满足不断增长的需求一直是一个持续的挑战。应用的快速增长——这是行业成功的标志——只会加剧对动态且可靠的基础设施的需求。此外，可编程性是区块链的核心优势之一，但智能合约语言可能存在意想不到的漏洞，从而阻碍服务、用户资产（如 NFT）的可互操作性和安全性。”^(⁶⁰)

## Layer 2 与区块链三难困境

`Layer 2` 确实容易与 `Layer 1` 混淆。但为了更好地理解，让我们先看看所谓的区块链三难困境。这涉及到以下因素之间的权衡：

- **安全性**：确实，这是所有技术都存在的问题。但在区块链系统中，安全风险可能更为严重。原因是这类系统通常依赖于金融资产。
- **去中心化**：这关乎如何实现低摩擦的点对点交易。
- **可扩展性**：对于建立一个真正的全球性 Web3 系统而言，这一点至关重要。但当前的区块链系统常常存在拥堵和费用问题。结果就是性能可能不足——这意味着 Web3 技术的采用率可能会降低。

对于以太坊而言，它在前两个因素上做得不错。但正如我们在本书中所见，第三个因素常常是个问题。原因可能在于以太坊的创造者当初并未预料到这项技术会如此受欢迎。

那么，这一切与 `Layer 2` 有何关系呢？总的来说，它主要涉及 Web3 平台上的应用。但事实上，这已经用于创建旨在帮助提升可扩展性的技术——同时又不牺牲安全性和去中心化。实际上，这已经变成了一门非常大的生意。而且这种增长趋势可能还会持续一段时间。

在大多数情况下，`Layer 2` 旨在寻找方法来提升交易速度（通常称为更快的最终确定性）以及更高的交易吞吐量（即每秒交易数）。请记住，有许多初创公司正在为此努力。还有许多顶级风险投资公司投入了大量资金。

在本章的后续几个小节中，我们将探讨 `Layer 1` 和 `Layer 2` 类别中的几家顶尖初创公司。

### IoTeX

物联网（IoT）是一个由使用传感器和软件的物理节点构成的网络。凭借这一点，可以实现与互联网的连接，从而进行信息交换。物联网设备的例子包括家用电器、婴儿监视器、工业工具，甚至自动驾驶汽车。

物联网市场规模巨大，预计将快速增长。根据麦肯锡的数据，到 2025 年，该网络将拥有超过 750 亿个智能设备，产生 80 万亿 GB 的数据。^(⁶¹) 其价值估计为 11 万亿美元。

尽管如此，物联网仍然存在一些棘手的问题，例如安全性、隐私性和集成性。但 Web3 和区块链可以提供帮助。

这正是初创公司 `IoTeX` 所押注的方向。该公司成立于 2017 年，正在开发一个去中心化的 `Layer 2` 物联网平台。它允许用户控制其设备生成的数据。他们甚至可以通过其 `MachineFi` 技术将数据变现。请注意，其中涉及 `IOTX` 代币。

`IoTeX` 也自行制造设备。例如，有 `Ucam`，这是一款精密的家用安防摄像头。还有 `Pebble Tracker`，这是一个智能 GPS 系统，可以实时收集温度、湿度和空气质量数据。

`IoTeX` 使用兼容 `EVM` 的区块链，支持 `dApps` 和智能合约。该平台自 2019 年初开始运行，一直保持着高度稳定性。^(⁶²)

### Mina Ecosystem

`Mina Ecosystem` 于 2017 年推出，体量非常轻。其区块链大小仅为 22 KB，大约相当于几条推文的体积。正因为如此，它已被证明对智能手机和其他设备非常有用。

`Layer 2` 的 `Mina Protocol` 同样构建了强大的安全和隐私功能。为此，它使用了零知识证明应用和智能合约。这意味着系统可以在用户无需交出数据的情况下验证数据。`Mina Ecosystem` 利用这项技术创建了其独特的智能合约，称为 `zkApps`。另一个优势是编写这些应用程序非常容易。^(⁶³)

2022 年 3 月，`Mina Ecosystem` 筹集了 9200 万美元。^(⁶⁴) 这无疑是对零知识智能合约类别的重要肯定。

FTX Ventures 的合伙人 Brian Lee 表示：“Mina 的轻量级区块链架构使协议能够达到一个新的去中心化水平，让每个人都能轻松参与网络。这一点，再加上其原生的数据隐私能力，使其成为一个独特的 L1，在塑造 Web3 未来方面具有巨大潜力。”^(⁶⁵)

### Mysten Labs

2021 年，Evan Cheng、Sam Blackshear、Adeniyi Abiodun 和 George Danezis 共同创立了 `Mysten Labs`。在此之前，该团队曾在 Meta 的加密部门 Novi 工作。这些联合创始人曾帮助创建了 `Move` 编程语言以及 `Diem` 区块链。

尽管 Meta 拥有充足的资源，但推进工作仍然困难重重。主要原因之一是严苛的监管审查。

因此，对于联合创始人来说，创建一个初创公司似乎是更好的道路。`Mysten Labs` 专注于为名为 `Sui` 的 `Layer 1` 区块链构建先进的基础设施技术。这是 `Diem` 的下一代版本，使用无需许可的 `PoS` 协议。您还可以使用 `Move` 语言来创建智能合约。

那么，主要的区别在哪里呢？它拥有几乎无限的吞吐量（即每秒交易数）。^(⁶⁶)

至于商业模式，它与许多基础设施运营商的方法类似。也就是说，公司通过从不同区块链的代币中获取一定比例来产生收入。这本质上是一种所有权模式。^(⁶⁷)

### StarkWare

Uri Kolodny 是一位连续创业者。他共同创立了 OmniGuide，该公司开发了用于微创手术的精密光学激光手术刀。随后他又创办了 TimnaTimna，该公司为心血管问题创建了一种超声治疗产品。^(⁶⁸)

但在这之后，他进入了软件行业。例如，他创立了 Mondria Technologies，专注于一种用于数据可视化的新型声明式编程语言。

这段经历对他进入 Web3 领域创办 `StarkWare` 至关重要。该公司成立于 2017 年，是以太坊扩容解决方案的顶级开发者。其中两个主要解决方案包括 `StarkEx` 和 `StarkNet`。它们让人们更容易在以太坊网络上创建 `dApps`。

`StarkWare` 创建了自己的语言，称为 `Cairo`。该语言自 2020 年 6 月起已投入生产。不过，对于 `StarkWare`，您并不必须使用这种语言。该公司一直在为其他系统构建转译器。

`StarkWare` 使用零知识卷叠技术。这意味着验证基于合并交易，从而提升性能并降低费用。`StarkWare` 还对其安全基础设施进行了大量投资。至于客户，其中包括许多顶级 Web3 提供商，如 Immutable、dYdX 和 ConsenSys。^(⁶⁹)

自成立以来，`StarkWare` 已筹集了 1.73 亿美元。^(⁷⁰) 部分投资者包括红杉资本和 Paradigm。

### Matter Labs

当 Alex Gluchowski 于 2018 年底推出其 Web3 初创公司 Matter Labs 时，他并没有加密货币背景。他之前创办的两家公司分别是一个房车租赁市场和一家倡导整体生活方式的网站。然而，他从这些创业经历中获得了宝贵的技能。^(⁷¹) 他学会了如何构建现代化且可扩展的 Web 平台、管理远程团队、实施支付和记账系统、创建合规自动化流程，以及在国际市场开展业务。

关于 Matter Labs，其技术基础是名为 `zkSync` 的 Layer 2 协议。为了提高效率，智能合约的资金存放在主链上，而计算和存储则在链下处理。这基本上意味着交易被汇总到一个区块中。事实上，以太坊联合创始人 Vitalik Buterin 在他的博客中写道，从中长期来看，零知识汇总“将在所有用例中胜出……”^(⁷²)

Matter Labs 不仅构建了直观的用户体验，在安全特性方面也表现出色。这有助于其赢得客户青睐，尤其是在 DeFi 领域。

Matter Labs 平台使用 `Zinc`，这是一个用于创建智能合约的框架。这一点很重要，因为零知识证明协议框架通常缺乏这些能力。`Zinc` 的语法与 `Rust` 语言类似。不过，Matter Labs 也在开发一个 `Solidity` 实现。

2021 年底，该公司筹集了 5000 万美元。参与本轮融资的 Andreessen Horowitz 在博客中写道：“以太坊 L1 上的活动，涉及 DeFi、治理、NFT 交易、DAO 创建和游戏，其在创造力和进步方面已经超出了我们最疯狂的想象。解释我们所目睹的独创性的一种方式，可以追溯到密码学固有的自我主权的核心理念，这是 `web3` 精神所必需的。`zkSync` 正在为下一代开发者构建（并且正在积极招聘！），并将创新的涟漪浪潮释放到已然充满活力的以太坊生态系统中。”^(⁷³)

### Optimism

Jinglan Wang 是 Layer 2 基础设施领域的一个罕见人物。她是该行业顶级初创公司 Optimism 为数不多的女性 CEO 和联合创始人之一。

从大学开始，她就对区块链着迷。她后来成为麻省理工学院比特币俱乐部的联合主席，在那里她极大地扩展了她的行业人脉网络。

她没有完成学业，而是创办了一家名为 `Eximchain` 的公司。随后，她去了 `Zcash` 和 `Nasdaq`，担任区块链项目的产品经理。

她身上有道疤，实际上是因为以太坊。怎么回事呢？当 ETH 涨到 300 美元时，她正和朋友们在酒吧。她太兴奋了，以至于从椅子上摔了下来，脚踝裂开了一道口子。^(⁷⁴)

但最终是 Optimism 成为了她的突破性事业。2022 年 3 月，她从 Andreessen Horowitz 和 Paradigm 筹集了 1.5 亿美元的 B 轮融资，估值达到 16.5 亿美元。^(⁷⁵)

Optimism 的核心技术不同于零知识汇总。相反，它采用了“乐观汇总”。这指的是使用一组验证者来验证交易，而非密码学。在融资时，Optimism 平台锁定的总价值已超过 5 亿美元。^(⁷⁶)

Optimism 成功的关键之一在于其基本坚持了以太坊的方法论。^(⁷⁷) 这既利用了庞大的生态系统，又使得入门变得更加容易。

> **注意**
> Layer 2 和侧链之间常常存在混淆。但两者截然不同。Layer 2 系统构建在以太坊平台之上。而侧链则与之并行运行。它是一个独立的区块链，拥有自身的安全机制和共识方法。一个顶级的侧链是 `Polygon`。

## Offchain Labs

Ed Felton 今年 59 岁。然而，年龄并未成为他作为炙手可热的 Web3 初创公司 Offchain Labs 联合创始人兼首席科学家的障碍。事实上，Felton 拥有深厚的学术背景。他于 1985 年毕业于加州理工学院，获得物理学学位，并在华盛顿大学获得博士学位。他的博士论文是关于并行处理器的通信协议。

1993 年，他加入普林斯顿大学计算机科学系任教。此后，他曾担任联邦贸易委员会首席技术官，以及奥巴马总统的副首席技术官。^(⁷⁸)

Offchain Labs 的起源始于 Felton 在普林斯顿大学的研究工作。他在那里遇到了他的联合创始人 – Steven Goldfeder 和 Harry Kalodner – 并开始创建一个名为`Arbitrum`的 Layer 2 平台。这始于 2015 年，目标是让任何人都能使用以太坊。^(⁷⁹)

Offchain 的平台基于乐观汇总，并且已经获得了很大的发展势头。2021 年，有超过 350 个团队在使用它。一些合作伙伴包括`Uniswap`和`Chainlink`等公司。

该公司以 1 亿美元完成了 B 轮融资，领投方是 Lightspeed Venture Partners，估值达到 12 亿美元。^(⁸⁰)

在一篇博文中，Lightspeed 投资人 Amy Wu 这样说道：“关于 L2 如何运作、乐观汇总与零知识汇总之间的区别以及理论上哪种更好，已经有很多文章讨论。在决定领投 Offchain Labs 的 B 轮融资时，我们关注的是最重要的一点：什么对开发者来说更容易使用、体验更好。”^(⁸¹)

## Layer 3

Layer 3 被称为应用层。这是 dApps 等程序以及为其提供支持的协议所在的地方。这些应用几乎可以是任何东西。然而，大部分开发工作集中在 NFT、游戏和 DeFi 等领域。

Layer 3 有两个主要部分。首先是应用本身，即用户与之交互的部分。这通常涉及脚本和 API。

其次是执行部分。这关乎管理区块链上智能合约的规则和条件。本质上，这是底层代码。因此，在应用和执行之间需要协调行动。

在第 7 章和第 8 章中，我们将更深入地探讨 Layer 3 程序和系统。至于本章的其余部分，我们将介绍不同类型的 Web3 框架和库以及编程语言。

## Web3 框架与库

人们常常将“库”和“框架”这两个词混为一谈，但实际上它们之间存在重要区别。

诚然，两者都是代码库，都提供常用功能，例如身份验证、用户界面或安全功能。而且其中许多都基于 JavaScript 或 TypeScript。库和框架确实能极大地加快开发项目的进度。

那么，区别究竟在哪里呢？关键在于`控制反转`这一概念。基本上，使用库时，你对程序的工作流程拥有更多控制权。而框架则提供了它自己的路径。这与 API 的使用类似。

对于 Web3 来说，存在大量的库和框架。让我们来看一下：

-   `Web3.js`（库）：它提供了一套用于链上交易的函数以及与以太坊节点交互的组件。这基本上简化了处理数据传输的 `JSON RPC` 机制（这是一种标准格式）。这是通过使用 JavaScript 代码实现的。`Web3.js` 还能与 `Node.js` Web 应用程序和 Electron 桌面程序无缝协作。开发者通常会将 `Metamask` 浏览器扩展（一种浏览器内的以太坊钱包）与此库结合使用。
-   `Ethers.js`（库）：它使用 JavaScript 和 TypeScript 提供四个主要模块来与以太坊交互。`Ethers.provider` 负责处理交易；`ethers.contract` 帮助部署和管理智能合约；`ethers.utils` 为 dApp 提供数据格式化和处理功能；`ethers.wallet` 则用于连接以太坊地址。
-   `oo7.js`（库）：是的，它是以詹姆斯·邦德命名的。从该库的技术层面来看，它主要涉及响应式表达式或称`绑定`。这本质上使得判断以太坊智能合约上何时触发了值成为可能。
-   `Truffle`：Consensys 于 2017 年创建了它。`Truffle` 是一个全面的框架，拥有易于使用的工具，用于创建、调试和部署以太坊智能合约。它还与流行的 Web 框架（如 `React`、`Angular` 和 `Vue`）进行了集成。使用 `Truffle`，你可以使用 `Solidity` 或 `JavaScript` 进行编码。另一个有用的特性是这个框架能管理网络工件。`Truffle` 还包含了 `Ganache`，这是一套丰富的 GUI 组件。
-   `Hardhat`（框架）：它允许你用很少的工作量创建一个市场。它处理了管理数字资产的所有复杂事务。该框架支持 `Django` 和 `Ruby on Rails` 等 Web 框架。`Hardhat` 还拥有强大的 `Solidity` 调试能力，这些能力通过丰富的插件生态系统得到了增强。
-   `OpenZeppelin SDK`（框架）：该框架于 2018 年推出，基于 `Python` 语言构建，为 NFT 和其他代币提供组件。其一个主要优势是内置了强大的安全特性。`OpenZeppelin SDK` 带有自己的开发环境，称为 `Jesta`。但该框架最强大的功能或许是能够升级现有的智能合约。
-   `Brownie` 框架（框架）：它基于 `Python`，用于在以太坊上创建智能合约。在开发过程中，你可以使用 `Solidity` 或 `Vyper`。`Brownie` 框架使用了 `web3.py` 库，该库已被证明能在密集型环境中良好运行。

**注意**

`预言机`可以实现混合智能合约。这些合约同时处理链上和链下数据。此类别中的一个顶级解决方案是 `Chainlink`。它建立在多种技术之上，如 `Java`、`C++`、`Python` 和 `Node.js`。

### Web3 编程语言

正如我们所见，Web3 依赖于多种编程语言。有些语言已经存在多年，比如 `Java`、`JavaScript`、`C++` 和 `Python`。但也有一些是专门为区块链构建的语言。其中一些顶级的语言包括 `Solidity` 和 `Vyper`。

无论如何，Web3 开发有一些常用工具。通常会用到 `Node.js` 和一个代码编辑器。编辑器可以是一个用于命令行编码的纯文本应用程序，或者你也可以使用一个复杂的 IDE（流行的有 `Remix IDE`）。然后通常会用到 Web3 框架和库。最后，开发者会使用钱包来访问节点。

从宏观层面来看，通常有两种类型的开发者。一种是后端开发者，他们擅长处理区块链的复杂细节。另一种是前端开发者，他们专注于用户界面、工作流程和设计，并且通常不需要对底层技术有深入理解。

请注意，Web3 开发仍然面临挑战。具体如下：
-   **不成熟**：开发工具仍处于早期阶段。结果是用户界面可能比较笨拙。开发环境中常常缺少核心功能。
-   **成本**：Gas 费和其他费用可能是一个主要障碍。在大多数情况下，编码人员可能会对部署应用程序犹豫不决。这意味着活动将主要以实验为主，例如在测试网上进行。
-   **碎片化**：市场上已经有了许多开发工具。这使得编码人员难以找到最佳工具。同时还存在风险：大量投资于某个最终可能失去人气的工具。
-   **不可变性**：当智能合约被添加到以太坊区块链后，你通常无法更改它。这无疑很困难，因为代码往往远非完美。不过，有一些变通方法，比如使用代理来指向另一个智能合约。

但这些问题是可以预料的。其他新平台，如 `iOS` 或 `Android`，也经历过类似阶段。

好消息是，在开发工具方面有大量投资。未来也将更加关注低代码和无代码解决方案，这将有助于推动 Web3 开发的民主化。

现在让我们来看看其中一些语言。

### Rust

`Rust` 的语法与 `C++` 相似。但它能更轻松地处理内存泄漏等问题。其性能也很高，尤其体现在处理海量数据时。

请注意，2021 年 Stack Overflow 开发者调查（涉及超过 80,000 名开发者）将 `Rust` 评为最“受欢迎”的语言。⁸² 这已经是它连续六年获此殊荣了。使用该语言的公司包括 `Amazon`、`Meta`、`Google` 和 `Microsoft` 等。

但 Web3 正成为 `Rust` 发展的催化剂。例如，像 `Polkadot` 和 `Solana` 这样的平台使用该语言来创建智能合约。

诚然，学习曲线比较陡峭。但如果你想编写具有强大安全性的高性能 Web3 应用程序，`Rust` 是一个相当不错的选择。

**注意**

2021 年底，杰克·多西在推特上写道：“`rust` 是一种完美的编程语言。”⁸³

### Solidity

2014 年，加文·伍德开始开发 Solidity 语言，其目标是让为以太坊区块链创建去中心化应用变得更加容易。当时，该语言的开发工作使用 C++完成。虽然 C++功能极其强大，但也很复杂。

到 2015 年，伍德完成了 Solidity 的第一个版本。他还得到了其他顶尖开发者的帮助，例如克里斯蒂安·赖特维斯纳、亚历克斯·贝雷格萨斯齐、莉安娜·胡西克扬和平井洋一。

如今，Solidity 继承了 C++的重要特性。请注意，这两种语言都是面向对象的。这意味着你可以将应用程序组织成不同的组件，这些组件支持代码和逻辑的可重用性以及跨程序其他部分的继承。面向对象的方法往往能带来更高的效率和可扩展性，在多开发者协作时也更为便捷。

Solidity 在以太坊开发中依然非常流行。但你也可以将这门语言用于许多其他区块链平台，例如波场、Hedera Hashgraph、Avalanche 和币安智能链。

在语法方面，它类似于 JavaScript。这使得程序员更容易学习该语言。

最常用的 Solidity 集成开发环境（IDE）是 Remix。该环境旨在支持在区块链上编写、运行、调试和部署智能合约。请记住，必要的库已包含在发行版中。此外，它还与以太坊虚拟机（EVM）实现了无缝集成。

### Vyper

Vyper 用于为以太坊虚拟机开发智能合约，在该类别中排名第二，仅次于 Solidity。这门语言的创造者是维塔利克·布特林（最初命名为 Viper）。随后其他贡献者接手了这个项目。

Vyper 的特点之一是其安全特性。与 Solidity 相比，这些特性更自然地融入了开发过程。其编译器实现也更为简单。

其次是可审计性。这意味着非技术人员也可以查看 Vyper 代码，并对工作流程有一个大致的了解。

其语法类似于 Python。Vyper 拥有一些面向对象的特性，如继承。

然而，由于这些优点，与 Solidity 相比，其功能特性较少。这门语言的自由度也相对较低。

> **注意：** 为提高 Solidity 的安全性，每年都会举办“暗藏心机 Solidity 编程竞赛”。其目标是编写“看似无辜且直截了当的 Solidity 代码，但实际上包含恶意行为或后门”。^(【84】^) 2022 年的竞赛主题是构建一个 DeFi 平台，一等奖是参加波哥大 Devcon VII 的门票。

### QuickNode

QuickNode 是一个区块链基础设施平台。其一些知名客户包括 PayPal、Adobe、OpenSea 和 Fireblocks。该公司已获得超过 4000 万美元的风险投资支持。

借助 QuickNode，开发者可以快速搭建并集成区块链系统。其成功的关键之一在于速度。你可以在几毫秒或更短的时间内访问以太坊、比特币、Polygon 和其他节点。该公司成功的另一个关键因素是其开发团队的敏捷性。2021 年，它大约每个月就新增对一个新区块链的支持。

其增长尤为强劲。在 2021 年第一季度，QuickNode 每天为近两万名开发者的社区处理超过 20 亿次请求。^【(85)^】为了扩展规模，该公司已在四大洲的 15 个以上区域建立了多云基础设施。

但这种增长可能还会持续一段时间。市场与市场研究公司的一项预测表明，区块链基础设施将以 67%的年复合增长率增长，到 2025 年将达到 400 亿美元。^【(86)^】

QuickNode 通过一个易于使用的面板进行运营。通过该面板，您可以获得以下功能：
- **丰富的分析功能：** 您可以获取去中心化应用性能的指标，例如哪些智能合约方法被调用得最多以及何时被调用。
- **监控功能：** 您可以按小时、天、周或月查看流入您节点的流量趋势的详细信息。
- **调试功能：** 提供了有用的工具来检测和修复您的去中心化应用的问题。
- **NFT 获取工具：** 您无需再手动抓取日志。QuickNode 有一个聚合钱包中 NFT 的 API。
- **归档功能：** 系统会跟踪智能合约的所有状态并将其记录以备将来使用。
- **追踪功能：** 您可以使用不同类型的数据重新执行一笔交易。
- **集成功能：** 集成了许多工具，例如 MetaMask、Augur 和 MyCrypto。

### 开发者观点

那么 Web3 开发者有哪些方法和观点呢？以下是一家名为 GeodeFinance 的质押技术开发公司对此的见解。

首先，来看`0xCypher`^((87))给出的要点：
*   **以太坊**：“目前，在我们的应用中，每个钱包所属的数十个数据集记录都应保存在以太坊上。这是因为将它们保存在中心化数据库中，完全迫使用户信任我们。因此，我们界面上出现的每个网页组件都来自去中心化数据库以太坊。这样一来，包括我在内的任何人都不能随意更改数字。所有更新都在智能合约允许的范围内进行，这就是为什么我建议每个 DeFi 和 Web3 用户确保合约编写安全且无错误。”
*   **工具**：“我们首先需要找到一个 RPC（远程过程调用）端点。如果节点在你的本地机器上运行，那么你计算机的某个端口就已连接到以太坊。你发送到诸如`127.0.0.1/8000`之类的 URL 的每次调用，现在都可以从以太坊提取或发送你的余额和合约信息。连接供应商后，你还需要对所有合约交互进行编码。幸运的是，Web3 库在 Javascript 和 Python 中都能非常高效地使用。”
*   **权益证明**：“基于权益证明的以太坊第 2 版，即‘信标链’，基于存款机制。我使用 Web3 库的主要方式是将 API 调用发送到这些链，并通过预言机服务将我获取的数据加载到我们自己的智能合约中。通常，你可以通过简单地调用 Python 和 Javascript 中的函数来完成此操作。但我们使用一个名为 Gnosis 的多重签名系统，通过安全的共识来确保数据安全，这意味着我们要对输入数据进行编码和签名，然后发送给 web3。”

以下是来自 Pacific 的观点：
*   **Web3 体验**：“关于 Web3 所有这些卓越优势的主要问题和难点在于随之而来的责任。50 亿人可以访问互联网，但其中大多数人甚至不习惯使用它。此外，使用 Web3 并不像使用传统的 Web2 那样舒适。Web3 建立在诸如以太坊、Avalanche、Solana 等智能合约区块链之上。要在这些区块链上自由操作，你需要对自己的钱包承担全部责任，需要与各种不同类型的去中心化应用交互，并且大多数情况下，在基础层面上，你需要知道自己在这些去中心化应用内部和之上到底在做什么。所有这些因素起初对大多数人来说可能令人沮丧，而且老实说，考虑到大多数人，这确实令人沮丧。要让 Web3 成为新标准，需要一整套范式的转变，在我看来，这还需要一些时间。”
*   **在以太坊上开发**：“以太坊已经存在近十年了。它是第一个通过智能合约实现可编程货币概念的区块链。此外，以太坊基础扎实，拥有能力出众的团队和社区。这个坚实的基础使得以太坊颇具吸引力。从开发者的角度来看，互联网上关于以太坊的资源比任何其他区块链都多。你可以通过几次搜索引擎查询就找到答案。而大多数其他区块链，即使是兼容 EVM 的，情况也并非如此。技术围绕以太坊聚集，从而形成了更大的社区、更强的开发者支持，进而带来了更高的适应性。与此吸引力相关的是，围绕以太坊构建的去中心化应用数量超过了其他区块链的总和。虽然差距正在慢慢缩小，但以太坊在 Web3 基础设施上的主导地位仍将持续一段时间。”
*   **初创公司**：“Web3 的主要问题在于它仍可以被视为处于起步阶段，并且存在适应性问题。所以对我来说，一个有趣的初创公司应该着手解决这些关于 Web3 和以太坊的问题。OpenSea 和 NFT 生态系统在激发非技术人员的兴趣并将其带入这个领域方面做得非常出色。通过这个展示，我们已经看到任何人都可以稍加努力就能与区块链交互。尽管 OpenSea 的理念仍然与 Web2 公司类似，但为了让大多数人理解 Web3 的真正运作方式，需要一个从中中心化到去中心化的过渡形式。”
*   **超金融化**：“用户在 Web3 中所做的一切都通过他们的钱包进行，这与当前 Web2 的模式不同——人们认为他们免费使用大多数平台。当然，事实并非如此，但人们是这么感知的。从行为经济学中我们知道，大多数人都是损失厌恶的。在人们眼中，-10 美元的绝对值远大于 +10 美元的绝对值。因此，对于面向大众的 Web3 应用来说，一个有趣的初创公司应当构建一种架构，让人们不会觉得在区块链或 Web3 平台上操作时是在亏钱。”
*   **以太坊的替代品**：“对我来说，Avalanche 是一个很好的以太坊替代品，尤其是如果你是新用户或小型投资者，只有少量闲钱进行试验和学习。Avalanche 通过其 C 链提供 EVM 兼容性，并且由于其共识协议的概率特性，能以更低的交易成本实现更高的吞吐量。此外，Avalanche 区块链可以通过其子网在可扩展性方面具备一些优势。在一个 Web3 成为新标准的未来，像子网这样的跨越式技术仍然可以在消除可扩展性问题负担的同时确保去中心化。”

## 结论

在本章中，我们概览了 Web3 技术栈的全貌。它仍在不断发展，这给创业者带来了开发上的挑战。选择错误的平台和工具可能会阻碍初创公司的进展。但对于任何新兴行业来说，这始终是一种风险。

尽管如此，技术栈的结构已初现端倪。基础层是第一层（Layer 1），涉及区块链的核心技术。接下来是第二层（Layer 2），主要关乎提升性能和效率的系统。在过去几年中，风险投资对第二层表现出了浓厚的兴趣。

最后是第三层（Layer 3），涉及各类应用，例如 DeFi、NFT、游戏等。

在本章中，我们还了解了一些 Web3 库和框架，如 `Web3.js`、`Truffle` 和 `Ethers.js`。我们还概览了诸如 `Solidity`、`Vyper` 和 `Rust` 等编程语言。
